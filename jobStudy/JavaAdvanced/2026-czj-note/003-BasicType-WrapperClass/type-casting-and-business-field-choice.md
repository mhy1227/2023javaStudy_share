# 强制类型转换与业务字段类型选择

## 1. 什么是类型转换

类型转换就是：

- 把一种类型的数据变成另一种类型

在 Java 里常见两种：

- 自动类型提升
- 强制类型转换

## 2. 自动类型提升

小范围类型参与运算时，通常会自动提升到更大的类型。

例如：

```java
int a = 10;
double b = a;
```

这里不需要手动写强转。

## 3. 强制类型转换

从大范围类型转成小范围类型时，通常需要强转。

例如：

```java
double d = 12.9;
int x = (int) d;
```

这里结果是：

- `12`

因为小数部分会被截掉。

## 4. 强转的两个风险

### 4.1 精度丢失

```java
double d = 12.9;
int x = (int) d;
```

结果：

- `12`

### 4.2 数据溢出

```java
byte b = (byte) 128;
```

结果不会是你直觉里的 `128`，因为 `byte` 范围有限，会发生溢出。

## 5. `char` 参与运算的注意点

例如：

```java
char c = 'A';
int x = c + 1;
System.out.println(x);
```

这里结果是数值，不再是 `char` 本身。

说明：

- `char` 在运算时会涉及编码值

## 6. 为什么手机号通常用 String

这是一个很典型的业务问题。

手机号虽然看起来都是数字，但通常应该用：

- `String`

而不是：

- `long`

### 原因 1：手机号不是用来做数学运算的

手机号的本质更像：

- 标识符

不是拿来加减乘除的数值。

### 原因 2：可能有前导 0

如果某类编号前面有 `0`，用数值类型可能会丢失格式信息。

### 原因 3：可能包含区号或符号

例如：

- `+86`
- `-`
- 空格

这些都更适合字符串表示。

## 7. 不只是手机号，很多业务字段都该用 String

例如：

- 身份证号
- 学号
- 订单号
- 邮编
- 银行卡号

它们虽然“看起来像数字”，但本质更接近：

- 编号
- 标识
- 文本字段

所以常常应该用 `String`。

## 8. 什么时候才更适合用数值类型

如果字段是拿来做数值计算的，才更适合用：

- `int`
- `long`
- `double`
- `BigDecimal`

例如：

- 年龄
- 数量
- 金额
- 分数

## 9. 当前阶段最该掌握的结论

1. 大转小通常需要强转
2. 强转可能丢精度，也可能溢出
3. 手机号不是数值，应优先用 `String`
4. 业务里很多“纯数字样子”的字段，其实都更适合 `String`

## 10. 一句话总结

类型选择不能只看“长得像不像数字”，要看它到底是拿来计算，还是拿来标识。
