# long、float、double、char 高频快问快答

## 1. 为什么 `long` 常写 `L`

因为整数字面量默认是：

- `int`

如果数值超出 `int` 范围，或者你明确就是要表示 `long`，通常应写：

```java
long num = 123L;
```

## 2. 为什么 `float` 常写 `F`

因为小数字面量默认是：

- `double`

所以：

```java
float score = 12.5F;
```

更标准。

## 3. `double` 为什么更常用

因为：

- 小数字面量默认就是 `double`
- 范围和精度通常比 `float` 更常用

所以日常开发里，若没有特殊要求，小数更常见用：

- `double`

## 4. `char` 是字符还是数字

`char` 本质上表示：

- 单个字符

但它和编码值有关，所以参与运算时会表现出数值特征。

例如：

```java
char c = 'A';
System.out.println(c + 1);
```

这里输出的是数值结果，不是直接输出字符。

## 5. `char` 为什么只能放一个字符

因为 `char` 的设计就是：

- 表示一个字符

所以合法写法是：

```java
char c = 'A';
```

而下面是不合法的：

```java
char c = 'AB';
```

## 6. `char` 用单引号还是双引号

- `char` 用单引号
- `String` 用双引号

例如：

```java
char c = 'A';
String s = "A";
```

## 7. `'A' + 1` 的结果是什么

结果会是一个数值运算结果。

因为：

- `char` 在运算时会提升为整数类型参与计算

所以它考察的不是字符串拼接，而是字符编码相关的运算。

## 8. `boolean` 为什么不能写成 0 和 1

因为 Java 中：

- `boolean` 只有 `true` 和 `false`

例如：

```java
boolean flag = true;
```

而下面这种是不允许的：

```java
boolean flag = 1;
```

## 9. `byte b = 127; b = b + 1;` 为什么容易报错

因为：

- 表达式运算时，小范围整数类型通常会提升为 `int`

所以：

```java
byte b = 127;
b = b + 1;
```

右边表达式结果是 `int`，不能直接赋给 `byte`。

如果强转，还可能溢出。

## 10. `byte b = (byte)128;` 为什么结果怪

因为：

- `byte` 范围有限
- 超出范围后会发生溢出

这类题经常用来考你是否知道：

- 强转不只是“截断”
- 也可能因为范围不够而溢出

## 11. 面试里这类题主要考什么

本质上考的是：

1. 字面量默认类型
2. 小类型自动提升
3. 强制类型转换风险
4. `char` 和 `String` 的区别
5. `boolean` 的基本规则

## 12. 当前阶段最该记住的结论

1. 默认整数是 `int`
2. 默认小数是 `double`
3. `long` 常写 `L`
4. `float` 常写 `F`
5. `char` 是单字符，用单引号
6. `char` 参与运算时会体现编码值
7. `boolean` 不能直接拿 `0/1` 替代

## 13. 一句话总结

这类小题看着零碎，其实都围绕一件事：Java 对基本类型有明确的默认规则、提升规则和边界规则。
