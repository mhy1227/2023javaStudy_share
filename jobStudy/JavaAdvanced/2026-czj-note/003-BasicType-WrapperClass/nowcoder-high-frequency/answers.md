# 牛客高频面试题：参考回答版

## 1. String、StringBuilder、StringBuffer 的区别

- `String` 不可变
- `StringBuffer` 可变，线程安全
- `StringBuilder` 可变，非线程安全，但单线程下通常效率更高

## 2. 为什么 String 不可变

- `String` 的内容创建后不能直接改
- 所有“修改”操作本质上都返回新字符串
- 这样更安全，也便于常量池复用和哈希稳定

## 3. 为什么 String 适合做 HashMap 的 key

- `String` 不可变
- 内容稳定时 `hashCode()` 才稳定
- 放进哈希容器后不会因为内容变化导致定位出问题

## 4. `String s1 = "123"` 和 `new String("123")` 的区别

- `"123"` 更偏向直接使用常量池中的对象
- `new String("123")` 会额外创建新的堆对象
- 所以二者在 `==` 判断时表现可能不同

## 5. `==` 和 `equals()` 的区别

- `==` 比较地址或引用是否相同
- `equals()` 比较内容是否相同

对 `String` 和包装类这类对象来说，平时比较内容应优先用 `equals()`。

## 6. 什么是字符串常量池

- 字符串常量池是专门存放可复用字符串字面量的一块区域
- 相同字面量通常会复用同一个对象

## 7. `intern()` 是什么

- `intern()` 和字符串常量池有关
- 它返回的是常量池中对应内容的那个引用

## 8. 为什么要引入包装类

- Java 有很多场景只能使用对象
- 集合和泛型不能直接放基本类型
- 所以需要包装类作为桥梁

## 9. `int` 和 `Integer` 的区别

- `int` 是基本类型
- `Integer` 是包装类对象
- `Integer` 可以是 `null`，`int` 不行

## 10. 自动装箱和自动拆箱

- 自动装箱：基本类型转包装类
- 自动拆箱：包装类转基本类型

例如：

```java
Integer a = 10;
int b = a;
```

## 11. 包装类默认值

- 基本类型成员变量有默认值
- 包装类成员变量默认值是 `null`

例如：

- `int` 默认 `0`
- `Integer` 默认 `null`

## 12. `parseXxx()`、`valueOf()`、`xxxValue()` 的区别

- `parseInt("123")`：`String -> int`
- `Integer.valueOf("123")`：`String -> Integer`
- `num.intValue()`：`Integer -> int`

## 13. 为什么 Integer 会出现 `100 == 100` 为 true，`200 == 200` 为 false

- 因为 `Integer.valueOf()` 存在缓存机制
- 常见缓存范围是 `[-128,127]`
- 范围内可能复用同一个对象，范围外通常不会

## 14. 什么是自动类型提升

- 小范围类型参与运算时，通常会自动提升到更大的类型

## 15. 什么场景下需要强转

- 从大范围类型转成小范围类型时，通常需要强制类型转换

例如：

```java
double d = 12.9;
int x = (int)d;
```

## 16. 强转可能带来什么问题

- 精度丢失
- 数据溢出

## 17. 为什么手机号用 String

- 手机号更像标识，不是运算值
- 可能有前导 `0`
- 可能包含 `+86`、空格、连接符
- 所以更适合用 `String`

## 18. final 修饰基本类型和引用类型的区别

- `final` 修饰基本类型变量，值不能再改
- `final` 修饰引用类型变量，引用不能再指向别的对象

## 19. 为什么 `final` 不等于对象不可变

- `final` 约束的是变量绑定
- 对象内部内容是否能改，要看这个类本身的设计

例如：

```java
final StringBuilder sb = new StringBuilder("abc");
sb.append("def");
```

这里对象内容还是能改。

## 20. String 不可变和 final 的关系

- `String` 不可变，不是因为你写了 `final String s`
- 而是因为 `String` 这个类本身就是按不可变对象设计的

## 21. 一句话总结

这组题的主线其实就三条：

- 先分清值和对象
- 再分清引用和内容
- 最后再看包装类、常量池和业务字段类型选择
