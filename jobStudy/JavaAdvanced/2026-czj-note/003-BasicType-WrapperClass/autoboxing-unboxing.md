# 自动装箱与拆箱

## 1. 什么是自动装箱

自动装箱指的是：

- 基本数据类型自动转换成对应的包装类对象

例如：

```java
Integer a = 10;
```

这里本质上发生了：

- `int` -> `Integer`

## 2. 什么是自动拆箱

自动拆箱指的是：

- 包装类对象自动转换成对应的基本数据类型

例如：

```java
Integer a = 10;
int b = a;
```

这里本质上发生了：

- `Integer` -> `int`

## 3. 为什么这两个机制有用

因为写代码会更方便。

你不需要总是手动写：

```java
Integer.valueOf(10);
a.intValue();
```

编译器会帮你做一部分转换。

## 4. 示例

```java
Integer num1 = 100;  // 自动装箱
int num2 = num1;     // 自动拆箱
```

## 5. 需要注意的问题

### 5.1 拆箱时可能空指针

```java
Integer num = null;
int x = num;
```

这里会报：

- `NullPointerException`

因为拆箱需要拿到具体的基本类型值，而 `null` 没有值。

### 5.2 不要忘了包装类本质上是对象

虽然写法像基本类型，但它本质还是引用类型。

## 6. 当前阶段最该掌握的结论

1. 自动装箱：基本类型 -> 包装类
2. 自动拆箱：包装类 -> 基本类型
3. 自动拆箱时要小心 `null`
4. `Integer` 虽然写起来像 `int`，但本质仍然是对象

## 7. 一句话总结

自动装箱拆箱让代码更方便，但别忘了包装类仍然是对象，尤其要防止拆箱时的空指针问题。
