# ConcurrentHashMap 单体笔记

## 1. ConcurrentHashMap 是什么

`ConcurrentHashMap` 是并发场景下非常重要的 `Map` 实现类。

当前阶段可以先把它理解成：

- 更现代的线程安全 `Map`

## 2. 核心特点

- 存 `key-value`
- 线程安全
- 更偏并发场景
- 经常被拿来替代早期线程安全方案

## 3. 为什么它会被单独拿出来讲

因为它在面试里出场频率很高，常常和：

- `HashMap`
- `Hashtable`

一起对比。

## 4. 它适合什么场景

适合这些情况：

- 多线程环境下共享 `Map`
- 并发访问比较明显

## 5. 当前阶段注意点

- 现在先记定位，不急着深挖底层并发实现
- 后面如果进入并发章节，再专门展开

## 6. 一句话总结

`ConcurrentHashMap` 是“并发场景下最值得优先认识的线程安全 Map”。
