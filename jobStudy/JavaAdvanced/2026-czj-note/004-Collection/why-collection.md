# 为什么需要集合

## 1. 为什么数组不够

在学习集合之前，先要回答一个问题：

- 既然已经有数组，为什么还要集合？

数组当然能存多个数据，但它有几个明显限制。

## 2. 数组的局限

### 2.1 长度一旦确定就不方便改

例如：

```java
int[] arr = new int[10];
```

这里长度定了就是 10。

如果你后面发现：

- 元素不够放
- 元素用不完

都不够灵活。

### 2.2 插入和删除不方便

如果要在数组中间插入一个元素，往往需要：

- 挪动后面的数据

删除也类似。

### 2.3 不知道实际存了多少有效元素

数组长度是固定的，但其中不一定都是真正有效的数据。

例如：

```java
String[] arr = new String[100];
```

你可能只用了前 12 个位置。

### 2.4 功能不丰富

数组更像一个最基础的容器。

它不擅长直接解决这些问题：

- 去重
- 按 key 查找 value
- 动态增长
- 方便遍历和操作

## 3. 集合解决什么问题

集合本质上是在说：

- 如何更灵活地管理一组对象

它的优势通常有：

- 容量更灵活
- 插入删除更方便
- 提供更多现成方法
- 针对不同业务场景有不同实现

## 4. 集合最核心的使用场景

### 场景 1：存放一批对象

例如：

- 一组学生
- 一组商品
- 一组字符串

### 场景 2：要频繁增删改查

如果数据不是固定死的，集合通常比数组更合适。

### 场景 3：需要去重

例如：

- 用户标签不能重复
- 某个字段集合不允许重复

### 场景 4：需要 key-value 存储

例如：

- 用户名 -> 年龄
- 商品 id -> 商品对象

这时就会用到 `Map`。

## 5. 集合不是只有一种

Java 里的集合不是一个单独的类，而是一整套体系。

大体上分成两块：

- `Collection`
- `Map`

其中：

- `Collection` 下面常见的是 `List` 和 `Set`
- `Map` 用来存键值对

## 6. 当前阶段先记什么

现在先别急着背所有实现类，先把这个总认识建立起来：

1. 数组能存多个数据，但不够灵活
2. 集合是为了更方便地管理一组对象
3. `List`、`Set`、`Map` 分别解决不同问题

## 7. 一句话总结

集合出现的根本原因，就是数组在“动态管理一批数据”这件事上不够灵活。
