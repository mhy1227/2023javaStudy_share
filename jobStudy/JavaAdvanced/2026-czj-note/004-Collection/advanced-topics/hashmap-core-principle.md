# HashMap 核心原理

## 1. 为什么 HashMap 是集合进阶里的重点

`HashMap` 在 Java 面试里出现频率非常高。

因为它同时能考察：

- 集合
- 哈希思想
- `equals()` / `hashCode()`
- 扩容
- 线程安全问题

所以这篇文档的目标不是带你抠源码，而是先把主线讲顺。

## 2. HashMap 本质上在做什么

`HashMap` 本质上是：

- 通过 key 快速找到 value 的容器

核心思想是：

- 先把 key 转成一个哈希值
- 再根据哈希值决定元素大概放哪

所以你可以先把它理解成：

- “key 先算地址，再去对应位置找 value”

## 3. 底层结构可以先怎么理解

面试里最常见的说法是：

- 数组 + 链表

进一步一点会说：

- 在 JDK 8 里，冲突严重时还会涉及红黑树

所以你现在可以先记：

- `HashMap` 底层不是单一结构
- 它是以数组为主，再结合链表/树来处理冲突

## 4. 为什么会有哈希冲突

因为不同的 key 经过哈希计算后，可能会落到同一个位置。

这就叫：

- 哈希冲突

冲突并不代表数据丢了，而是：

- 同一个位置上要想办法继续存多个元素

## 5. 冲突时怎么处理

当前阶段先用最基础的面试说法：

- 早期和常见理解：链表处理冲突
- JDK 8 中冲突较严重时，可能转成红黑树优化查找

你现在不需要立刻背具体阈值，先把：

- “为什么不是只有数组”

这条线讲清就够了。

## 6. 为什么 key 的 equals 和 hashCode 很重要

这是 `HashMap` 最核心的面试点之一。

因为 `HashMap` 处理 key 时，通常会结合：

- `hashCode()`
- `equals()`

来判断：

- key 落在哪个位置
- 当前 key 是新 key，还是和旧 key 逻辑相同

所以 Java 才强调：

- 如果两个对象 `equals()` 为 `true`
- 那它们的 `hashCode()` 也必须相同

否则 `HashMap` 的行为就可能不符合预期。

## 7. 为什么 String 适合作为 HashMap 的 key

因为：

- `String` 不可变
- `String` 已经正确重写了 `equals()` 和 `hashCode()`

所以它既稳定，又适合参与哈希计算。

## 8. put() 大致做了什么

当前阶段可以先这样讲：

1. 对 key 做哈希计算
2. 确定大概落在哪个位置
3. 如果位置空，就直接放
4. 如果位置不空，就处理冲突
5. 如果遇到逻辑上相同的 key，就覆盖旧 value

这套流程已经足够应对大多数基础面试问题。

## 9. get() 大致做了什么

也可以先这样理解：

1. 先对 key 做哈希计算
2. 找到对应位置
3. 再根据 `equals()` 等规则找到真正匹配的 key
4. 返回对应 value

## 10. 为什么 HashMap 会扩容

因为数据一直增加，原来的容量不可能无限够用。

所以当元素数量达到一定程度时，`HashMap` 会：

- 扩容

扩容的目的就是：

- 降低冲突压力
- 提高整体访问效率

## 11. 为什么扩容会被拿来问

因为扩容不是白来的，它会带来：

- 重新组织数据的成本

所以面试官问扩容，往往是在看你知不知道：

- `HashMap` 性能不是凭空来的
- 它和容量、冲突、重排有关

## 12. 为什么 HashMap 线程不安全

当前阶段先记住结论即可：

- `HashMap` 一般不强调线程安全
- 多线程共享修改时可能出问题

这也是为什么并发场景下会更常想到：

- `ConcurrentHashMap`

## 13. 面试里怎么答更稳

如果问：

- 说一下 HashMap 的底层原理

你可以按这个顺序答：

1. `HashMap` 用来存 key-value
2. 核心思想是哈希定位
3. 底层可以理解为数组 + 链表，JDK 8 中冲突严重时还涉及红黑树
4. key 的 `hashCode()` 和 `equals()` 非常重要
5. 元素增多时会扩容
6. `HashMap` 一般不强调线程安全

## 14. 当前阶段最该记住的结论

1. `HashMap` 核心是哈希定位
2. 底层不是单一结构
3. 冲突处理是它设计的关键
4. `equals()` / `hashCode()` 和它强相关
5. 扩容是高频考点
6. 并发场景不要直接把它当线程安全容器

## 15. 一句话总结

`HashMap` 的主线可以浓缩成一句话：用哈希快速定位，用链表/树处理冲突，用扩容平衡效率。
