# Set 基础笔记

## 1. 什么是 Set

`Set` 是 `Collection` 下面的一类集合。

它最核心的特点是：

- 不可重复

也就是说，`Set` 更适合“去重”场景。

## 2. Set 和 List 的区别

### List

- 有序
- 可重复

### Set

- 重点在不可重复

所以如果需求是：

- 保证元素不重复

第一反应通常就是 `Set`。

## 3. Set 常见实现类

当前阶段先记这 3 个：

- `HashSet`
- `LinkedHashSet`
- `TreeSet`

## 4. HashSet

最常见的 `Set` 实现类。

当前阶段先记：

- 不可重复
- 无序
- 底层和哈希思想相关

这里的“无序”不是说数据会随机消失，而是：

- 迭代输出顺序通常不等于插入顺序

## 5. LinkedHashSet

可以先记成：

- 在 `HashSet` 基础上，迭代顺序表现更稳定

所以它兼顾：

- 不可重复
- 顺序表现更可预期

## 6. TreeSet

可以先记成：

- 更偏排序

也就是说：

- 元素放进去后，会按一定规则组织

所以 `TreeSet` 经常和：

- 排序
- 比较规则

联系在一起。

## 7. 为什么 Set 能去重

这就是 `Set` 最容易延伸出的重点。

当前阶段先记：

- `Set` 判断元素是否重复，和对象比较规则有关

特别是：

- `HashSet` 会和 `equals()`、`hashCode()` 强相关

所以后面学集合时，你会发现：

- `equals()` 和 `hashCode()` 不只是对象章节的内容
- 它们和 `HashSet`、`HashMap` 直接有关

## 8. Set 常用方法

因为 `Set` 继承自 `Collection`，所以很多方法你会觉得很熟悉。

常用的有：

- `add()`
- `remove()`
- `contains()`
- `size()`
- `isEmpty()`

## 9. Set 的典型使用场景

### 场景 1：去重

例如：

- 用户标签去重
- 单词集合去重
- id 集合去重

### 场景 2：判断某个元素是否存在

例如：

- 某个值是否已经出现过

## 10. 当前阶段最该记住的结论

1. `Set` 属于 `Collection`
2. `Set` 的核心特点是不可重复
3. 最常见实现类是 `HashSet`
4. `LinkedHashSet` 更强调顺序表现
5. `TreeSet` 更偏排序

## 11. 一句话总结

如果你最关心的是“不要重复”，那就优先想到 `Set`。
