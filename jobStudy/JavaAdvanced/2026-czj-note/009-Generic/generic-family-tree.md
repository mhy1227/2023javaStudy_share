# 泛型体系族谱

## 1. 为什么泛型也要先看族谱

泛型这一章和前面的集合、异常、IO 有点像：

- 单看一个例子好像不难
- 但放到整章里就容易混

很多人会写：

```java
List<String> list = new ArrayList<>();
```

但一旦被问到：

- 为什么要有泛型
- 泛型类和泛型方法有什么区别
- `T`、`E`、`K`、`V` 是什么意思
- 通配符是干什么的

就容易讲乱。

所以这里先用“族谱”的方式把大结构理出来。

## 2. 先说最短结论

可以先这样记：

1. 泛型的核心价值是“把类型信息提前到编译期”
2. 泛型最常见的位置是类、方法、接口
3. 通配符是泛型进一步使用时的扩展
4. 泛型和类型擦除强相关

## 3. 为什么要有泛型

在没有泛型时，很多集合更像：

- 什么都能放
- 但取出来时不够安全

例如可以先粗理解成：

- 编译期约束不够强
- 强制类型转换更多
- 出错更容易拖到运行时

所以泛型的核心作用可以先记成：

1. 提高类型安全
2. 减少强制类型转换
3. 让代码表达更清楚

## 4. 一个最基础的例子

```java
List<String> list = new ArrayList<>();
list.add("Tom");
```

这里最关键的不是语法长什么样，而是：

- 你已经明确告诉编译器，这个列表里放的是 `String`

所以后面如果你试图放别的类型，就更容易在编译期暴露问题。

## 5. 泛型最核心的大分法

泛型最常见可以先分成三类：

1. 泛型类
2. 泛型方法
3. 泛型接口

## 6. 一个基础版泛型族谱

```text
泛型体系
├─ 泛型类
├─ 泛型方法
└─ 泛型接口
```

这一版虽然简单，但已经够你先建立主线。

## 7. 一个更详细一点的族谱

```text
泛型体系
├─ 基础价值
│  ├─ 类型安全
│  ├─ 减少强制类型转换
│  └─ 提高代码表达力
│
├─ 基本形态
│  ├─ 泛型类
│  ├─ 泛型方法
│  └─ 泛型接口
│
├─ 常见类型参数名
│  ├─ T
│  ├─ E
│  ├─ K
│  └─ V
│
├─ 进阶使用
│  ├─ 通配符 ?
│  ├─ ? extends
│  └─ ? super
│
└─ 底层特点
   └─ 类型擦除
```

这张图不用一口气全背，但结构要有印象。

## 8. 泛型类是什么

可以先理解成：

- 类在定义时就把类型参数留出来

最基础的感觉就是：

- 这个类操作的数据类型不是写死的
- 使用时再具体指定

## 9. 泛型方法是什么

可以先理解成：

- 不是整个类泛型，而是某个方法自己带泛型能力

所以这里要先有一个印象：

- 泛型不一定非要写在类上

## 10. 泛型接口是什么

可以先理解成：

- 接口也可以把类型参数留出来

这样实现类或者使用方在后续再具体指定类型。

## 11. T、E、K、V 是什么意思

这个点非常常见。

它们本质上都是：

- 类型参数名

只是大家约定俗成地更常这样写：

### T

- Type，表示某种类型

### E

- Element，常见于集合元素

### K

- Key，常见于 `Map`

### V

- Value，常见于 `Map`

所以可以先记：

- 它们不是关键字
- 是约定俗成的命名习惯

## 12. 通配符为什么后面还要单独讲

因为真正开始使用泛型时，经常会遇到：

- `?`
- `? extends`
- `? super`

这已经不只是“声明一个泛型类”那么简单了，而是在说：

- 泛型类型范围怎么约束

所以它很适合后面单独展开。

## 13. 类型擦除为什么总被一起提

因为 Java 泛型并不是把类型信息无限保留到底层。

面试里很常会继续追问：

- 泛型是不是运行时还完整存在
- 为什么不能直接 `new T()`
- 为什么有些地方泛型有限制

这些问题往后都和：

- 类型擦除

联系起来。

当前阶段先有这个印象就够了。

## 14. 泛型这章最容易混的点

最容易混的是：

1. 泛型的意义不是“写法高级”，而是编译期类型安全
2. 泛型不只存在于类，也能出现在方法和接口
3. `T`、`E`、`K`、`V` 不是特殊关键字
4. 通配符不是和泛型完全分开的另一章
5. 泛型和类型擦除强相关

## 15. 当前阶段最该记住的结论

1. 泛型的核心价值是类型安全
2. 泛型最常见的位置是类、方法、接口
3. `T`、`E`、`K`、`V` 是常见类型参数命名
4. 通配符和类型擦除是后面两条重要主线

## 16. 一句话总结

泛型这一章最核心的主线就是：为什么要有泛型，以及泛型类、泛型方法、泛型接口、通配符、类型擦除之间是什么关系。
