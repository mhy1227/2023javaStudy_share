# 泛型方法基础笔记

## 1. 为什么泛型方法也要单独讲

很多人一看到泛型，第一反应就是：

- 泛型类

但泛型其实不一定非要写在类上。

很多时候，一个类本身不是泛型类，但某个方法仍然可以：

- 单独拥有泛型能力

这就是：

- 泛型方法

## 2. 什么是泛型方法

泛型方法可以先理解成：

- 方法自己声明类型参数

也就是说：

- 不是整个类泛型
- 而是这个方法本身有泛型

## 3. 一个最基础的例子

```java
public class Util {
    public <T> void printValue(T value) {
        System.out.println(value);
    }
}
```

这里最关键的是：

- `<T>` 写在返回值前面

这表示：

- 这个方法是泛型方法

## 4. 泛型方法怎么使用

例如：

```java
Util util = new Util();
util.printValue("hello");
util.printValue(100);
```

可以先这样理解：

- 同一个方法
- 可以处理不同类型的数据

## 5. 一个更常见的返回值例子

```java
public class Util {
    public <T> T getFirst(T a, T b) {
        return a;
    }
}
```

使用时：

```java
Util util = new Util();
String s = util.getFirst("A", "B");
Integer n = util.getFirst(1, 2);
```

这个例子更能说明：

- 泛型方法不仅能接收泛型参数
- 还能返回泛型结果

## 6. 泛型方法和泛型类有什么区别

这个点非常容易被问。

### 泛型类

- 类型参数是类级别的

### 泛型方法

- 类型参数是方法级别的

一句话：

- 一个是“整个类都带泛型”
- 一个是“只有这个方法带泛型”

## 7. 普通类里能不能有泛型方法

可以，而且很常见。

例如：

```java
public class Util {
    public <T> void print(T value) {
        System.out.println(value);
    }
}
```

这里 `Util` 本身不是泛型类，但：

- `print` 是泛型方法

## 8. 泛型类里还能不能再写泛型方法

也可以。

例如：

```java
class Box<T> {
    private T value;

    public <E> void show(E data) {
        System.out.println(data);
    }
}
```

这里要先有一个印象：

- 类上的 `T`
- 方法上的 `E`

可以不是同一个东西。

## 9. 为什么静态方法里常见泛型方法

这个点后面很容易被问。

因为静态方法不依赖具体对象，所以很多工具类都会写成：

- 静态泛型方法

例如：

```java
public class Util {
    public static <T> void print(T value) {
        System.out.println(value);
    }
}
```

这类写法在工具方法里特别常见。

## 10. 泛型方法这块面试里容易问什么

常见问题有：

1. 什么是泛型方法
2. 泛型方法写在哪里
3. 泛型方法和泛型类有什么区别
4. 普通类里能不能有泛型方法
5. 静态方法里能不能用泛型

## 11. 当前阶段最容易混的点

1. `<T>` 写在返回值前面，不是写在参数列表后面
2. 泛型方法不要求类本身是泛型类
3. 类上的 `T` 和方法上的 `T` / `E` 不是一回事
4. 泛型类和泛型方法的作用范围不同

## 12. 当前阶段最该记住的结论

1. 泛型方法是方法自己声明类型参数
2. 泛型方法可以出现在普通类里
3. 泛型方法和泛型类不是同一个概念
4. 静态工具方法里很常见泛型方法

## 13. 一句话总结

泛型方法最核心的意义，就是让“某个方法”具备处理不同类型数据的能力，而不要求整个类都变成泛型类。
