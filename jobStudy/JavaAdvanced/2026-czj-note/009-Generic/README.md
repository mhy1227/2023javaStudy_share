# 泛型占位说明

## 1. 这一章准备讲什么

Java 泛型这一章，核心不是只记住几个尖括号，而是先分清：

- 为什么要有泛型
- 泛型类、泛型方法、泛型接口是什么
- `T`、`E`、`K`、`V` 这些到底表示什么
- 通配符是怎么回事
- 泛型和编译期检查有什么关系

也就是说，这一章最容易混的不是语法本身，而是：

- 泛型存在的意义
- 泛型在不同位置上的写法
- 泛型的边界和限制

## 2. 为什么这一章放在 IO 后面

因为前面已经补到了：

- 集合
- 数据结构
- 异常
- IO

这些内容里其实一直都在接触：

- `List<String>`
- `Map<String, Integer>`
- `Class<T>`

也就是说，泛型其实你早就在用，只是还没单独系统梳理。

所以现在把它拉出来，时机是比较自然的。

## 4. 一句话总结

泛型这一章的核心，不是背尖括号，而是先分清：它为什么存在、写在什么位置、能帮你避免什么问题、又有哪些边界。
