# 泛型牛客高频面试题

## 1. 这一份是做什么的

这一份单独只放问题，不展开答案。

适合用法：

- 快速过题
- 自测能不能口述出来
- 面试前最后一轮扫题

如果你想同时看标答，可以配合本目录下另一份：

- `generic-nowcoder-high-frequency-answers.md`

## 2. 第一梯队：最常被直接问

1. 为什么 Java 要引入泛型？
2. 泛型的核心作用是什么？
3. 泛型为什么能提高类型安全？
4. 泛型为什么能减少强制类型转换？
5. 泛型类、泛型方法、泛型接口分别是什么？
6. 泛型类和泛型方法有什么区别？
7. `T`、`E`、`K`、`V`、`R` 通常分别表示什么？

## 3. 第二梯队：很容易连着追问

1. 什么是通配符 `?`？
2. `List<?>` 能不能添加元素？为什么？
3. `? extends T` 和 `? super T` 有什么区别？
4. 为什么经常说 `extends` 偏读、`super` 偏写？
5. `List<Object>` 和 `List<String>` 是父子关系吗？为什么？
6. `List<?>` 和 `List<Object>` 一样吗？
7. 泛型中为什么会出现“只读”“只写”这种说法？

## 4. 第三梯队：高频延伸题

1. 什么是类型擦除？
2. Java 泛型是编译期机制还是运行期机制？
3. 为什么运行时拿不到完整的泛型类型信息？
4. 为什么不能直接 `new T()`？
5. 为什么不能直接创建泛型数组，比如 `new T[10]`？
6. 为什么类上的泛型参数不能直接用于静态成员？
7. 为什么异常类不能直接定义成泛型类？

## 5. 第四梯队：结合集合一起问

1. 为什么 `List`、`Set`、`Map` 这类集合要和泛型一起使用？
2. 为什么不推荐使用原生类型，比如裸 `List`？
3. `Map<K, V>` 里的 `K` 和 `V` 为什么要分开设计？
4. `ArrayList<String>` 和 `ArrayList<Object>` 为什么不能互相赋值？
5. 为什么说泛型是集合题的基础前置知识？

## 6. 可能出现的场景题

1. 如果一个方法只负责读取数据，参数应该优先考虑 `extends` 还是 `super`？
2. 如果一个方法主要往集合里放数据，参数应该优先考虑 `extends` 还是 `super`？
3. 如果你要写一个通用工具方法，应该定义成泛型类还是泛型方法？
4. 面试官问“你项目里泛型最常出现在哪”，你怎么回答？
5. 如果让你解释 `Collections.copy(List<? super T>, List<? extends T>)`，你会怎么拆开讲？

## 7. 一轮复习优先级

如果时间不够，优先口述这几题：

1. 为什么要有泛型？
2. 泛型类 / 泛型方法 / 泛型接口区别
3. `?`、`? extends`、`? super`
4. `List<?>` 为什么不能随便 `add`
5. 什么是类型擦除
6. 为什么不能 `new T()`
7. 为什么不能直接创建泛型数组
