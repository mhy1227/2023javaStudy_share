## 2. 泛型这块高频到底在问什么

## 3. 第一梯队：几乎必问

### 3.1 为什么要有泛型

回答主线：

- 提高类型安全
- 减少强制类型转换
- 把类型检查尽量提前到编译期

### 3.2 泛型类、泛型方法、泛型接口有什么区别

回答主线：

- 泛型类是类级别带泛型参数
- 泛型方法是方法自己声明泛型参数
- 泛型接口是接口把类型参数留出来

### 3.3 `T`、`E`、`K`、`V` 是什么意思

回答主线：

- 它们本质上都是类型参数名
- `T` 常表示 Type
- `E` 常表示 Element
- `K` / `V` 常表示 Key / Value

### 3.4 为什么泛型能提高类型安全

回答主线：

- 因为它把很多原本可能拖到运行时的类型问题，提前到了编译期

## 4. 第二梯队：非常爱追问

### 4.1 `?`、`? extends`、`? super` 的区别

回答主线：

- `?` 表示未知类型
- `? extends` 更偏读
- `? super` 更偏写

### 4.2 为什么 `List<?>` 不能随便 add

回答主线：

- 因为你只知道它是某种列表
- 但不知道具体允许放什么类型

### 4.3 为什么说 `extends` 偏读、`super` 偏写

回答主线：

- `? extends Number` 更适合把元素当 `Number` 读出来
- `? super Integer` 更适合往里写 `Integer`

## 5. 第三梯队：常作为延伸问题

### 5.1 什么是类型擦除

回答主线：

- Java 泛型很多类型信息主要服务于编译期
- 很多限制背后都和类型擦除有关

### 5.2 为什么不能 `new T()`

回答主线：

- 因为不能在这里直接按不确定的泛型类型创建对象
- 背后和类型擦除强相关

### 5.3 为什么不能直接创建泛型数组

回答主线：

- 泛型数组是高频限制点
- 背后和类型擦除、类型安全有关

### 5.4 为什么静态上下文不能直接用类上的泛型参数

回答主线：

- 类上的泛型参数和实例使用相关
- 静态成员属于类本身
- 两者层级不同

## 6. 最容易出的组合题

泛型这章经常不是单点问，而是一串问下来。

例如：

1. 为什么要有泛型
2. 泛型类和泛型方法区别
3. `? extends` 和 `? super`
4. 为什么不能 `new T()`
5. 最后追问类型擦除

所以复习时不要把这些点完全割裂开。

## 7. 如果只剩一轮时间，优先背什么

建议优先级：

1. 为什么要有泛型

3. `?`、`? extends`、`? super`
4. 为什么 `List<?>` 不能随便 add
5. 类型擦除
6. `new T()` 和泛型数组限制

## 8. 一段比较稳的总回答

可以这样答：

Java 泛型我一般先从作用去理解，它的核心是把类型检查尽量提前到编译期，提高类型安全并减少强制类型转换。泛型最常见的位置有类、方法和接口；使用时经常会继续遇到通配符 `?`、`? extends`、`? super`，其中 `extends` 更偏读，`super` 更偏写。再往后，很多泛型限制，比如不能直接 `new T()`、不能直接创建泛型数组，背后都和类型擦除有关。

## 9. 当前阶段最该记住的结论

1. 泛型高频点并不散，主要集中在少数几条主线
2. 通配符和类型擦除是两个最容易追问的点
3. `new T()`、泛型数组、静态泛型这些限制很常被问
4. 面试里要能把“为什么要有泛型”和“为什么有限制”连起来说

## 10. 一句话总结

泛型这一章的高频面试题，核心不是背尖括号写法，而是把“作用、分类、通配符、擦除、限制”这几条主线讲顺。
