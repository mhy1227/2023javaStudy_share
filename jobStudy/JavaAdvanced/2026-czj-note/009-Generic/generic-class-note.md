# 泛型类基础笔记

## 1. 为什么先学泛型类

泛型这一章最常见的入口，其实就是：

- 泛型类

因为平时最熟的例子大多都长这样：

```java
List<String>
Map<String, Integer>
```

这背后本质上都和“类把类型参数留出来”这件事有关。

所以先把泛型类立住，后面泛型方法、泛型接口、通配符会更顺。

## 2. 什么是泛型类

泛型类可以先理解成：

- 类在定义时，把某种类型先留成参数

也就是说：

- 类里要操作的数据类型不是写死的
- 使用这个类时，再具体指定

## 3. 一个最基础的例子

```java
class Box<T> {
    private T value;

    public void setValue(T value) {
        this.value = value;
    }

    public T getValue() {
        return value;
    }
}
```

这里可以先这样理解：

- `T` 不是某个具体类名
- `T` 表示“某种类型”

也就是说：

- `Box` 这个类本身还没决定装的是什么

## 4. 泛型类怎么使用

例如：

```java
Box<String> box = new Box<>();
box.setValue("hello");
String str = box.getValue();
```

这里最关键的是：

- 你在使用时把 `T` 指定成了 `String`

所以这个 `Box` 里装的就是：

- `String`

## 5. 为什么泛型类有意义

可以先总结成 3 点：

1. 类型更安全
2. 代码更清楚
3. 减少强制类型转换

如果没有泛型，很多时候就更容易变成：

- 什么都能放
- 但取出来时不够安全

## 6. 一个更直观的对比例子

### 没有泛型的感觉

可以粗理解成：

- 取出来时更容易需要强转
- 写错类型时更容易拖到运行时

### 有泛型的感觉

```java
Box<Integer> box = new Box<>();
box.setValue(100);
Integer num = box.getValue();
```

这里更直观地表达了：

- 这个盒子里装的是 `Integer`

## 7. 泛型类里的 T 能用在哪

最常见的是：

1. 成员变量类型
2. 方法参数类型
3. 方法返回值类型

例如：

```java
class Pair<T> {
    private T first;
    private T second;

    public T getFirst() {
        return first;
    }

    public void setFirst(T first) {
        this.first = first;
    }
}
```

## 8. 泛型类可以有多个类型参数吗

可以。

例如：

```java
class Pair<K, V> {
    private K key;
    private V value;
}
```

这个例子里：

- `K` 更像 key
- `V` 更像 value

这也是为什么你平时看到：

- `Map<K, V>`

不会觉得陌生。

## 9. 泛型类和普通类最大的区别是什么

普通类更像：

- 类型写死了

泛型类更像：

- 类型先留出来，使用时再指定

所以两者最大的区别，不是“写法花不花”，而是：

- 类型是否可参数化

## 10. 泛型类这块面试里容易问什么

常见问题有：

1. 什么是泛型类
2. 泛型类为什么有意义
3. `T` 是什么
4. 泛型类和普通类的区别
5. 一个泛型类能不能有多个类型参数

## 11. 当前阶段最容易混的点

1. `T` 不是具体类名
2. 泛型类的类型通常是在使用时指定
3. 泛型类和泛型方法不是一回事
4. `K`、`V`、`E`、`T` 都只是类型参数名

## 12. 当前阶段最该记住的结论

1. 泛型类是在类定义时把类型留成参数
2. 它能提高类型安全
3. 它能减少强制类型转换
4. 一个泛型类可以有一个或多个类型参数

## 13. 一句话总结

泛型类最核心的意义，就是让“类操作的数据类型”不再写死，而是在使用时再具体指定。
