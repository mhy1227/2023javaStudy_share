# 策略模式

## 1. 这份文档怎么用

策略模式是最适合和业务代码结合来讲的一个模式。

你至少要能回答：

- 什么是策略模式
- 它通常解决什么问题

## 2. 策略模式是什么

策略模式的核心是：

- 把一组可替换的算法或处理逻辑抽出来
- 在运行时根据不同条件选择不同策略

### 面试口语版

> 策略模式的核心是把不同处理逻辑封装成不同策略，在运行时按条件选择具体策略执行，从而避免大量 if else 堆在一起。

## 3. 为什么要用策略模式

如果一个业务场景里有很多分支：

- 支付方式不同
- 订单处理规则不同
- 优惠计算方式不同

最容易写成：

- 很长的 `if else`
- 很长的 `switch`

策略模式的价值就在于：

- 每个策略单独实现
- 调用方只负责选择

## 4. 项目里常见场景

比如：

- 不同支付渠道
- 不同优惠规则
- 不同登录方式
- 不同报表导出策略

这类都很适合用策略模式来讲。

## 5. 一句话收尾

> 策略模式最核心的价值，是把“不同业务规则”拆成不同策略，减少 if else，提升扩展性。
