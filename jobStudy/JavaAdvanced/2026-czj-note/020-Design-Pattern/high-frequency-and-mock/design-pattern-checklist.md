# 设计模式复习 Checklist

## 1. 基础认知

- 我能说清设计模式不是背模板代码，而是解决重复出现的设计问题
- 我能说清学习设计模式的核心价值是降耦合、提扩展、拆职责
- 我知道面试里不会只问定义，还会问场景、对比、项目落地

## 2. 单个模式是否能讲顺

- 我能讲清单例模式的核心目的、适用场景、线程安全追问
- 我能讲清工厂模式解决的是对象创建耦合问题
- 我能讲清代理模式为什么经常和 AOP 一起问
- 我能讲清策略模式为什么常用来消灭大段 if else
- 我能讲清模板方法为什么强调流程固定、步骤可变
- 我能讲清观察者模式为什么适合事件通知场景
- 我能讲清责任链模式为什么适合多节点串行处理
- 我能讲清适配器模式为什么解决接口不兼容
- 我能讲清装饰器模式为什么是动态扩展能力

## 3. 高频对比是否能答

- 我能区分策略模式和模板方法
- 我能区分代理模式和装饰器模式
- 我能区分工厂模式和简单直接 new 的差别
- 我能区分观察者模式和责任链模式
- 我能区分适配器模式和代理模式不是一回事

## 4. 项目表达是否到位

- 我不会上来就背定义，而是会先讲问题场景
- 我会把“为什么这样设计”放在“用了什么模式”前面
- 我能把业务例子和模式名称对应起来
- 我能顺手补一个 Spring 或框架里的例子

## 5. 面试现场是否稳定

- 我能用 30 秒说清一个模式的定义和场景
- 我能用 1 分钟把“问题 -> 方案 -> 模式”讲完整
- 我不会把所有能拆分类的代码都硬说成设计模式
- 我不会只背书面定义，导致追问一来就断掉

## 6. 复习顺序建议

1. 先过顶层主线文档，知道每个模式是干什么的
2. 再过高频问答，把问题和回答骨架对上
3. 再过口语版，把书面表达改成能说出口的版本
4. 再刷陷阱题和模拟题，专门练区分和追问

## 7. 一句话收尾

> 如果你现在只能记一条，那就记住：设计模式复习的重点不是记名字，而是记“这个模式解决什么问题、为什么要这么拆”。
