# 设计模式真实面经混合题

## 1. 这份文档怎么用

这一篇不是系统讲知识点，而是把公开面经里经常出现的问法压成题单。

适合：

- 面试前快速过一遍真实问法
- 看看自己对哪些问题最容易卡住
- 用来决定下一轮该补哪个模式

## 2. 真实面经里最常出现的问法

### 基础起手题

- 什么是设计模式
- 为什么要学设计模式
- 常见的设计模式你知道哪些
- 你最熟的两个设计模式是什么

### 单例相关

- 单例模式怎么写
- 懒汉和饿汉的区别是什么
- 双重检查锁为什么要配 `volatile`
- 静态内部类单例为什么常被认为更稳
- 枚举单例为什么安全

### 工厂相关

- 工厂模式解决什么问题
- 为什么不直接 `new`
- 简单工厂、工厂方法、抽象工厂怎么区分
- 项目里什么时候值得引入工厂

### 代理相关

- 静态代理和动态代理区别
- JDK 动态代理和 CGLIB 区别
- Spring AOP 为什么总和代理模式一起问
- 代理模式和装饰器模式怎么区分

### 策略和模板方法

- 策略模式适合什么场景
- 模板方法模式适合什么场景
- 为什么说策略模式适合替代大段 `if else`
- 策略模式和模板方法模式怎么区分

### 观察者和责任链

- 观察者模式适合什么场景
- 监听器机制为什么经常被归到观察者模式
- 责任链模式适合什么场景
- 过滤器链、拦截器链为什么经常拿责任链来解释

### 适配器和装饰器

- 适配器模式主要解决什么问题
- 接老系统接口时为什么容易想到适配器
- 装饰器模式和代理模式是不是一样
- Java IO 为什么常被用来举装饰器模式例子

## 3. 公开面经里常见的“框架结合题”

- Spring 中用到了哪些设计模式
- Spring MVC 中能看到哪些设计模式
- AOP 背后体现了什么设计模式
- `HandlerAdapter` 为什么常被说成适配器模式
- `Template` 系列为什么常被说成模板方法模式

## 4. 公开面经里常见的“项目表达题”

- 你项目里有没有用过设计模式
- 你怎么判断一个场景该用策略模式而不是 `if else`
- 你项目里哪里像责任链
- 你项目里哪里体现了工厂思想
- 如果项目里没有标准模式实现，你怎么回答更稳

## 5. 公开面经里常见的“对比题”

- 策略模式和模板方法模式区别
- 代理模式和装饰器模式区别
- 观察者模式和发布订阅区别
- 工厂模式和直接 `new` 的区别

## 6. 公开面经里常见的“反问式陷阱题”

- 只要拆掉 `if else` 就是策略模式吗
- 监听器就等于观察者模式吗
- 单例模式是不是就是全局变量
- 工厂模式是不是只是少写几个 `new`
- 代理模式和装饰器模式是不是本质一样

## 7. 这批题说明了什么

从公开面经看，设计模式不是让你把 23 种模式都背全，而是主要集中在：

- 单例
- 工厂
- 代理
- 策略
- 模板方法
- 观察者
- 责任链
- 适配器
- 装饰器

再往后就是：

- Spring 里的落地
- 项目里的真实场景
- 容易混的对比题

## 8. 建议复习顺序

1. 先把上面 9 个高频模式讲顺
2. 再把 Spring 映射题讲顺
3. 再练项目题和对比题
4. 最后用模拟题去压口语表达
