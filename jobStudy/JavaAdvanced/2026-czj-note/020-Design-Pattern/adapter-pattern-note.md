# 适配器模式

## 1. 这份文档怎么用

适配器模式在面试里很适合和这些词一起记：

- 老接口
- 新接口
- 不兼容
- 转换

## 2. 什么是适配器模式

适配器模式的核心是：

- 原来的接口和现在需要的接口不兼容
- 中间加一层适配器做转换

### 面试口语版

> 适配器模式的核心是把原本不兼容的接口通过中间一层转换成调用方能接受的形式。

## 3. 为什么要用适配器

现实开发里很常见的一种情况是：

- 老系统有一套接口
- 新系统想接入，但接口风格不一致

这时候如果直接大改原代码：

- 成本高
- 风险大

适配器模式的价值就在于：

- 尽量不动原有系统
- 通过一层转换把差异接住

## 4. 常见场景

比如：

- 接第三方 SDK
- 老接口升级到新接口
- 不同系统之间协议转换

## 5. 面试里怎么讲更稳

适配器模式最重要的不是说“像转接头”，而是讲清楚：

- 为什么原接口不能直接用
- 为什么要加这一层转换

### 展开答法

> 适配器模式适合那种原系统接口不能直接满足当前调用需求，但又不适合直接去改原接口的场景。它通过加一个中间转换层，把旧接口包装成新接口，让调用方和原系统解耦。

## 6. 一句话收尾

> 适配器模式最核心的价值，是在不大改原系统的前提下，把“不兼容”接成“可兼容”。
