# 观察者模式

## 1. 这份文档怎么用

观察者模式在面试里经常和“事件通知”“监听器”一起出现。

你至少要能回答：

- 观察者模式是什么
- 它通常适合什么场景

## 2. 观察者模式是什么

观察者模式的核心是：

- 一个对象状态发生变化
- 自动通知一组依赖它的对象

### 面试口语版

> 观察者模式的核心是建立一种一对多的通知关系，当被观察对象状态变化时，相关观察者会收到通知并做各自处理。

## 3. 为什么要用观察者模式

因为有些场景不是一个动作只对应一个处理。

比如：

- 用户注册成功之后
- 可能要发优惠券、发短信、记录日志、做积分初始化

如果全写在一个方法里，就容易越来越耦合。

观察者模式的价值就在于：

- 通知方和处理方解耦
- 一个事件可以挂多个处理逻辑

## 4. 常见理解方式

观察者模式很适合和这些词一起记：

- 事件
- 监听器
- 发布通知
- 多个响应方

所以你在面试里也可以顺手提：

- Spring 事件机制
- 各类监听器

## 5. 一句话收尾

> 观察者模式最核心的价值，是把“事件发生”和“后续响应”拆开，让通知方和处理方解耦。
