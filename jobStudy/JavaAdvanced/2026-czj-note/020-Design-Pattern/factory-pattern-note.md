# 工厂模式

## 1. 这份文档怎么用

工厂模式在 Java 面试里很常见，因为它特别容易和 Spring 串起来。

你至少要能回答：

- 工厂模式解决什么问题
- 为什么不直接 new

## 2. 工厂模式是什么

工厂模式的核心是：

- 把对象创建这件事单独封装起来
- 调用方不直接关心对象怎么创建

### 面试口语版

> 工厂模式的核心是把对象创建过程封装起来，让调用方更多关注“我要什么对象”，而不是“这个对象具体怎么 new 出来”。

## 3. 为什么要用工厂模式

如果代码里到处都是：

- `new A()`
- `new B()`
- `if else` 判断创建哪个对象

后面需求一变，创建逻辑就容易散得到处都是。

工厂模式的价值就在于：

- 统一对象创建入口
- 降低调用方和具体实现类的耦合

## 4. 面试里怎么理解它的价值

工厂模式本质上解决的是：

- 创建逻辑复杂
- 对象种类会扩展
- 调用方不该和具体类绑死

所以它很适合：

- 多实现类场景
- 插件化扩展场景
- 框架式对象管理

## 5. 和 Spring 怎么串起来

Spring 里很多地方都能看到工厂思想。

因为 Spring 本质上就在帮你做：

- 对象创建
- 生命周期管理
- 依赖注入

所以你在面试里可以顺手提一句：

> Spring 本身就大量体现了工厂思想，因为 Bean 的创建和管理本质上就不是业务代码自己直接 new。

## 6. 一句话收尾

> 工厂模式最核心的价值，是把“对象创建”从业务调用里抽出来，降低耦合、方便扩展。
