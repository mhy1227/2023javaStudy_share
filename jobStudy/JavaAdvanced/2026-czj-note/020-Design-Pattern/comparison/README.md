# 设计模式横向对比

## 1. 这个子目录是干什么的

这个子目录专门处理设计模式里最容易混的横向对比题。

它不是重新讲单个模式定义，而是重点解决：

- 两个模式到底差在哪
- 面试时怎么一句话区分
- 分别更适合什么场景
- 容易答混的点是什么

## 2. 当前目录结构

- `strategy-vs-template-method.md`
- `proxy-vs-decorator.md`
- `observer-vs-pubsub.md`
- `factory-vs-direct-new.md`

## 3. 怎么看最顺

建议顺序：

1. `strategy-vs-template-method.md`
2. `proxy-vs-decorator.md`
3. `observer-vs-pubsub.md`
4. `factory-vs-direct-new.md`

## 4. 一句话记住

> 对比题最稳的答法，不是背结构像不像，而是先说它们分别解决什么问题。
