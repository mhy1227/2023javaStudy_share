# 代理模式 vs 装饰器模式

## 1. 为什么这两个容易混

因为它们形式上都很像：

- 都可能包一层对象
- 都不一定改原类
- 都可能在原逻辑之外加行为

但它们的目标并不一样。

## 2. 核心区别

- 代理模式更偏控制访问、统一增强
- 装饰器模式更偏动态扩展能力

## 3. 一句话区分

> 代理模式重点是“代你访问并统一做控制”，装饰器模式重点是“在原能力上继续叠加新功能”。

## 4. 面试口语版

> 两者结构上都像包一层对象，但代理模式通常更强调权限、事务、日志、远程访问这类统一控制或增强；装饰器模式更强调在不改原类的前提下，一层层给对象叠加新能力。简单说，一个偏控制，一个偏扩展。

## 5. 典型场景

- 代理模式：Spring AOP、远程代理、权限校验、事务增强
- 装饰器模式：Java IO 流包装、功能链式增强

## 6. 常见追问

### 问题

Spring AOP 为什么更常说是代理，不说装饰器？

### 更稳的答法

> 因为 Spring AOP 的核心目标不是给对象动态叠加很多业务能力，而是统一织入事务、日志、权限这类横切逻辑，本质更接近代理模式里的访问控制和统一增强。

## 7. 容易翻车的点

- 因为“都包一层”就直接说成一样
- 只会讲结构，不会讲设计意图
- 说不出 Java IO 为什么常被拿来举装饰器例子

## 8. 一句话收尾

> 代理模式是“帮你管访问”，装饰器模式是“帮你加能力”，这就是面试里最稳的区分方式。
