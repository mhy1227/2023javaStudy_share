# 装饰器模式

## 1. 这份文档怎么用

装饰器模式在面试里经常容易和代理模式混。

所以这一篇至少要讲清楚：

- 装饰器模式是什么
- 它为什么和代理不完全一样

## 2. 什么是装饰器模式

装饰器模式的核心是：

- 不改原对象代码
- 通过包装对象的方式
- 给对象动态增加功能

### 面试口语版

> 装饰器模式的核心是通过包装对象的方式，在不改原类代码的情况下，给对象动态扩展功能。

## 3. 为什么要用装饰器

如果你想给一个对象不断加功能：

- 加日志
- 加缓存
- 加统计

如果每加一种功能就继承一层，很快类就会很多、很乱。

装饰器模式的价值就在于：

- 用组合代替过多继承
- 功能可以一层层叠加

## 4. 常见理解方式

你可以把装饰器理解成：

- 原对象不变
- 但外面一层一层包
- 每包一层就多一点能力

这类理解在面试里很好讲。

## 5. 和代理模式有什么区别

这是高频追问。

你可以先这样记：

- 代理更强调“控制访问、增强外部逻辑”
- 装饰器更强调“在原有能力上叠加新功能”

两者形式上都像包一层，但出发点不一样。

### 稳一点的答法

> 代理和装饰器表面上都像包一层对象，但代理更偏控制访问和统一增强，装饰器更偏在原有功能基础上动态叠加新能力。

## 6. 一句话收尾

> 装饰器模式最核心的价值，是通过组合包装的方式，灵活给对象叠加功能，而不是靠继承把类层级越堆越厚。
