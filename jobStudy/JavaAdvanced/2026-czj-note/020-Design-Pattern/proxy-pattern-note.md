# 代理模式

## 1. 这份文档怎么用

代理模式在 Java 面试里非常重要，因为它和 Spring AOP 基本是强关联的。

你至少要能回答：

- 代理模式是什么
- 为什么要做代理
- 它和 AOP 有什么关系

## 2. 代理模式是什么

代理模式的核心是：

- 不直接操作目标对象
- 而是通过一个代理对象去控制、增强或包装目标对象的行为

### 面试口语版

> 代理模式就是在不直接改目标对象代码的前提下，通过代理对象对原有行为做一层控制或增强。

## 3. 为什么要用代理

因为很多时候你想加的是“通用逻辑”，比如：

- 日志
- 权限校验
- 事务
- 统计

这些逻辑如果全写进业务代码里，业务类会越来越重。

代理模式的价值就是：

- 不改原业务逻辑
- 但可以在外面包一层增强

## 4. 和 Spring AOP 的关系

这是代理模式最常见的落点。

你可以这么理解：

- Spring AOP 的很多能力，本质上就是通过代理机制实现的

比如：

- 事务
- 日志切面
- 权限切面

所以面试里如果问代理模式，你基本都可以顺手提一句 Spring AOP。

## 5. 一句话收尾

> 代理模式的核心价值，是在不侵入原业务代码的前提下，对行为做统一增强，这也是 Spring AOP 能成立的关键思路之一。
