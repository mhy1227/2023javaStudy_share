# 二叉搜索树基础笔记

## 1. 什么是二叉搜索树

二叉搜索树可以先理解成：

- 带有有序规则的二叉树

前面的二叉树，只强调：

- 每个节点最多两个孩子

而二叉搜索树在此基础上进一步要求：

- 左边节点通常比当前节点小
- 右边节点通常比当前节点大

所以它不是“随便一棵二叉树”。

## 2. 为什么它重要

因为它同时兼顾了两件事：

- 树形结构
- 有序查找

所以它在“查找、插入、删除”这类问题里很经典。

## 3. 它和普通二叉树的区别

普通二叉树只要求结构像二叉树。

二叉搜索树除了结构，还要求：

- 节点值满足大小关系

所以你可以这样记：

- 二叉树强调形状
- 二叉搜索树强调形状加顺序

## 4. 最核心的规则

对任意一个节点来说，通常满足：

- 左子树的值比它小
- 右子树的值比它大

这个规则会递归地体现在整棵树里。

## 5. 为什么查找会更自然

因为有大小规则，所以找一个值时可以不断缩小范围。

例如你要找 `10`：

- 如果当前节点比 `10` 大，就往左找
- 如果当前节点比 `10` 小，就往右找

这就比“毫无规则地整棵树乱找”更高效。

## 6. 为什么中序遍历很关键

对于二叉搜索树来说，中序遍历结果通常是：

- 递增有序

这也是为什么前面在树遍历里强调：

- 中序遍历特别重要

## 7. 二叉搜索树的优点

可以先记这几个：

1. 结构清晰
2. 查找思路自然
3. 插入时也能根据大小找到位置
4. 和“有序集合”思想很接近

## 8. 二叉搜索树的局限

它不是永远都快。

如果树长得很不均匀，例如一边特别长，就会退化得很难看。

这时它看起来甚至会接近：

- 链表

所以后面才会继续学：

- 平衡树
- 红黑树

## 9. 和 Java 集合的关系

这里要先建立一个基础关联：

- `TreeSet`
- `TreeMap`

它们和“有序树结构”思想强相关。

当前阶段不必一上来背红黑树细节，但要知道：

- 树形有序结构不是凭空来的
- 背后和二叉搜索树这一类思想有关

## 10. 和 Hash 结构的区别

这一点以后很容易被问到。

可以先粗记：

- Hash 结构更偏快速定位
- 树结构更偏有序组织

所以如果题目强调：

- 有序
- 范围
- 最小值 / 最大值

树结构通常会更自然。

## 11. 当前阶段最该记住的结论

1. 二叉搜索树是带大小规则的二叉树
2. 左边通常比根小，右边通常比根大
3. 中序遍历结果通常有序
4. 它和 `TreeSet`、`TreeMap` 的底层思想强相关
5. 它可能退化，所以后面还会引出平衡树

## 12. 一句话总结

二叉搜索树的核心价值，是让树不仅能分层组织数据，还能按大小关系进行查找和维护。
