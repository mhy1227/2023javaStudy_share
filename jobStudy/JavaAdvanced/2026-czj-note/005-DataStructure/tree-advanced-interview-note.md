## 3. 二叉树题型还能继续怎么分

后面再刷树题时，建议直接按题型分类。

### 3.1 遍历类

代表思路：

- 前序
- 中序
- 后序
- 层序

代表问题：

- 遍历输出
- 右视图
- 之字形层序遍历

### 3.2 高度 / 深度 / 直径类

代表问题：

- 最大深度
- 最小深度
- 二叉树直径
- 平衡二叉树

这一类的共同点是：

- 递归函数往往要从子树拿高度信息

### 3.3 路径类

代表问题：

- 路径和
- 根到叶子路径
- 最大路径和

这一类通常在考：

- DFS
- 回溯
- 路径状态维护

### 3.4 结构判断类

代表问题：

- 对称二叉树
- 相同的树
- 平衡二叉树
- 完全二叉树
- 是否是 BST

这一类通常在考：

- 递归定义是否写稳
- 你能不能把“局部判断”和“全局约束”区分开

### 3.5 构造类

代表问题：

- 前序 + 中序重建二叉树
- 有序数组转 BST
- BST 转双向链表

这一类常暴露的问题是：

- 索引边界不清
- 左右子树区间划分不清

### 3.6 设计类

代表问题：

- 二叉树序列化与反序列化
- 迭代器设计

这一类更像：

- 树 + 代码设计

## 4. 面试官最爱在树题后面追问什么

树题通常不会只停在“你把代码写出来了没有”。

最常见追问有这些：

1. 为什么这里要先递归左子树再递归右子树
2. 为什么层序遍历要用队列
3. 判断 BST 为什么不能只比较当前节点和左右孩子
4. 这个递归函数的返回值到底表示什么
5. 时间复杂度和空间复杂度是多少
6. 能不能改成迭代写法
7. 递归会不会栈溢出

所以树这块不能只练“做题结果”，还要练“题后解释”。

## 5. 树题里几个很容易被忽略的关键点

### 5.1 BST 判断是全局约束，不是局部比较

很多人第一次写 `isValidBST` 时，只会判断：

- 左孩子比根小
- 右孩子比根大

这其实不够。

真正的 BST 判断要看整棵子树是不是都满足上下界约束。

### 5.2 LCA 的关键不在模板，在返回值含义

最近公共祖先题经常有人“背模板背出来”，但一追问就卡住。

你真正要说清的是：

- 左子树找到了什么
- 右子树找到了什么
- 当前节点为什么可以成为汇合点

### 5.3 平衡二叉树本质上是在做“高度信息回传”

判断平衡二叉树时，真正关键不是“会不会求高度”，而是：

- 能不能在递归回来的时候顺手把不平衡状态一起带上来

### 5.4 直径题通常不是单纯的遍历题

二叉树直径这类题经常考：

- 节点左侧高度
- 节点右侧高度
- 用左右高度更新全局答案

所以它更接近：

- 后序遍历 + 信息汇总

## 6. 还要不要补 N 叉树

要有印象，但优先级低于二叉树。

原因很简单：

- 大多数入门和实习面试，二叉树远高于 N 叉树

但 N 叉树仍然值得知道这些：

1. 一个节点可以有多个子节点
2. DFS 和 BFS 思路依然成立
3. 只是“left / right”变成“children 列表”

所以它更像：

- 二叉树遍历思想的扩展版

## 7. Trie 要不要放进树这一章

可以放，但建议放在“树进阶扩展”里，而不是放进第一轮主线。

因为 Trie 常见于：

- 字符串前缀匹配
- 搜索建议
- 单词查找

它不是树入门第一轮必须优先刷的内容，但面试中偶尔会问：

- 什么是前缀树
- 它适合解决什么问题

所以至少要会说：

1. Trie 是一种树形结构
2. 适合按字符逐层组织前缀
3. 适合前缀查询，不适合替代所有字符串查找场景

## 8. 平衡树 / 红黑树要补到什么程度

如果当前目标是：

- Java 基础复习
- 校招 / 实习 / 初中级后端面试

那建议补到“认知层”就够了。

至少要知道：

1. 普通 BST 可能退化
2. 所以需要平衡思想
3. AVL 更严格追求平衡
4. 红黑树是工程里更常见的平衡搜索树方案
5. Java 里的 `TreeMap` 是红黑树实现的 `NavigableMap`
6. `TreeSet` 底层依赖 `TreeMap`

这里通常不要求你现场手写红黑树旋转，但很可能会问：

- 为什么不是普通 BST
- 为什么 `TreeMap` 能保持有序

## 9. 树和 Java 集合这块还可以怎么补

如果后面从数据结构继续过渡到集合，那么树这里最自然的连接点是：

1. `TreeMap`
2. `TreeSet`
3. `PriorityQueue` 和堆的关系

这里建议把思路连起来：

- Hash 结构偏快速定位
- 树结构偏有序组织
- 堆结构偏优先级处理

这三类在面试里经常会被放在一起对比。
