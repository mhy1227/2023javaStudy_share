## 2. 为什么树题特别适合手撕

因为树题很适合现场考察这几件事：

1. 你会不会递归
2. 你会不会画树结构
3. 你会不会区分 DFS 和 BFS
4. 你会不会处理空节点
5. 你是不是只会背遍历顺序

数组题很多时候还能靠线性思路硬写。

但树题如果结构感不清楚，就很容易：

- 递归出口写错
- 左右子树逻辑写反
- 一层层处理时忘了队列

## 3. LeetCode 官方树卡片给出的核心方向

根据 LeetCode 官方树面试卡片和 Binary Tree / BST Explore，树这块最核心的训练点集中在：

1. `breadth-first` 和 `depth-first`
2. `Maximum Depth of Binary Tree`
3. `Validate Binary Search Tree`
4. `Binary Tree Level Order Traversal`
5. `Convert Sorted Array to Binary Search Tree`

这说明官方在强调三条主线：

- 遍历
- BST 判断
- 树的构造

也就是说，树的手撕题并不是散的，而是围绕这几条主线重复变形。

## 5. 面经里最常见的手撕树题

结合牛客讨论区和面经页里能看到的题型，树的手撕题最常出现的是下面这些。

### 第一梯队：最常被手写

1. 二叉树前中后序遍历
2. 二叉树层序遍历
3. 二叉树最大深度
4. 对称二叉树
5. 判断是不是二叉搜索树
6. 二叉树最近公共祖先

这一梯队的特点是：

- 题目短
- 很经典
- 非常能看出你递归和层序遍历是否扎实

### 第二梯队：进阶手撕

1. 重建二叉树
2. 翻转二叉树 / 二叉树镜像
3. 判断是不是平衡二叉树
4. 判断是不是完全二叉树
5. 路径和
6. 输出右视图

这一梯队的特点是：

- 不是只靠一个遍历模板
- 需要你把递归返回值和结构判断结合起来

### 第三梯队：综合构造题

1. 序列化 / 反序列化二叉树
2. BST 转双向链表
3. 有序数组转 BST

这一梯队更像：

- 树 + 设计
- 树 + 构造

## 6. 如果面试官说“手撕一道树题”，你最该优先练什么

建议优先级：

1. 前序 / 中序 / 后序遍历
2. 层序遍历
3. 二叉树最大深度
4. 对称二叉树
5. 判断是不是二叉搜索树
6. 最近公共祖先
7. 重建二叉树

这 7 组优先练的原因很简单：

- 已经覆盖了 DFS、BFS、BST、递归结构判断、构造题这几条主线

## 7. 树题面试里最爱追问什么

树题一般不会只停在“你写出来了没有”，还会继续追问这些：

1. 递归出口为什么这么写
2. 为什么层序遍历要用队列
3. 前中后序遍历分别什么时候有用
4. 判断 BST 为什么不能只比较当前节点和左右孩子
5. 最近公共祖先为什么能这样返回
6. 时间复杂度和空间复杂度是多少
7. 能不能改成迭代写法

所以复习树题时，不能只记结果，还要把这些解释也顺手练出来。

## 8. 手撕树题时最容易暴露的问题

1. 前中后序顺序写混
2. `root == null` 的出口写不稳
3. 层序遍历忘了按层处理
4. 判断 BST 只看局部，不看全局范围
5. LCA 只会背模板，不理解左右子树返回值
6. 重建二叉树时前序 / 中序索引关系理不清

## 9. 一轮速刷建议

如果你现在就在准备后端一面 / 二面，树题可以这么刷：

### 第一天

1. 前序 / 中序 / 后序遍历
2. 层序遍历
3. 最大深度

### 第二天

1. 对称二叉树
2. 判断 BST
3. 最近公共祖先

### 第三天

1. 翻转二叉树
2. 平衡二叉树
3. 重建二叉树

## 10. 一句话总结

树的手撕面试真正高频的，不是“树有多少题”，而是少数几类经典题不断变形反复考：遍历、层序、BST、LCA、结构判断、构造。
