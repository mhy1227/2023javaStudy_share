# 数组基础笔记

## 1. 什么是数组

数组可以先理解成：

- 用来存放一组相同类型数据的连续容器

例如：

```java
int[] arr = {1, 2, 3, 4};
```

这里就是一个整数数组。

## 2. 数组的核心特点

当前阶段先抓这几个：

1. 元素类型相同
2. 长度固定
3. 可以按下标访问

## 3. 为什么数组很重要

因为数组几乎是最基础的数据结构之一。

后面你会发现很多结构、很多题、很多集合实现，都和数组思想有关。

例如：

- `ArrayList`
- 哈希表中的数组部分
- 堆

## 4. 数组的优势

### 4.1 按下标访问直观

```java
System.out.println(arr[0]);
```

### 4.2 存储结构简单

它是很多更复杂结构的基础。

## 5. 数组的局限

### 5.1 长度固定

一旦创建，长度不方便改。

### 5.2 插入删除不灵活

如果在中间插入或删除元素，往往需要移动其他元素。

### 5.3 必须知道大概容量

如果你一开始估错大小，就会比较麻烦。

## 6. 什么时候会想到数组

适合这些场景：

- 数据量相对确定
- 需要按位置快速访问
- 问题本身带有“下标”语义

## 7. 和 ArrayList 的关系

可以先这样理解：

- 数组是底层结构
- `ArrayList` 是 Java 封装好的动态列表工具

也就是说：

- 数组更底层
- `ArrayList` 更好用、更灵活

## 8. 当前阶段最该记住的结论

1. 数组元素类型相同
2. 数组长度固定
3. 数组适合按下标访问
4. 数组在中间插入删除不够灵活

## 9. 一句话总结

数组是最基础的线性结构之一，它擅长按位置访问，但不擅长动态增删。
