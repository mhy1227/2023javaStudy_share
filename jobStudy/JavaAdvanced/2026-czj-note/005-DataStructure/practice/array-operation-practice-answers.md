# 数组操作练习题参考答案

## 1. 做数组题先抓住一个总原则

如果题目是基于普通数组而不是链表，那么很多操作的本质都是：

- 找位置
- 移动元素
- 维护有效长度

所以数组题的关键不只是“会写循环”，而是知道：

- 为什么要移
- 从哪开始移
- 朝哪个方向移

## 2. 基础操作题参考答案

### 题 1 删除第一个元素

核心思路：

- 从下标 `1` 开始，把后面的元素依次往前覆盖

逻辑过程：

- `arr[0] = arr[1]`
- `arr[1] = arr[2]`
- `arr[2] = arr[3]`
- `arr[3] = arr[4]`

结论：

- 删除首元素需要整体左移
- 时间复杂度通常看作 `O(n)`

### 题 2 删除最后一个元素

核心思路：

- 一般不需要整体移动
- 只要把有效长度减一即可

结论：

- 删除尾元素通常是数组删除里最自然的一种
- 如果题目允许只维护逻辑长度，它的代价更低

### 题 3 删除指定位置元素

假设删除下标 `2`：

- `30` 被删掉
- 从下标 `3` 开始的元素向前覆盖

逻辑过程：

- `arr[2] = arr[3]`
- `arr[3] = arr[4]`

结论：

- 删除位置之后的元素要左移
- 删除位置之前的元素不动

### 题 4 删除指定值的第一个匹配项

核心思路：

1. 先遍历找到值第一次出现的下标
2. 再复用“删除指定位置元素”的逻辑

如果值不存在：

- 直接返回原结果

结论：

- 这是“查找 + 删除”的组合题

### 题 5 在开头插入元素

核心思路：

- 所有元素都要向后移动一位
- 通常从后往前移动

为什么从后往前：

- 避免前面的值被提前覆盖

逻辑过程：

- `arr[4] = arr[3]`
- `arr[3] = arr[2]`
- `arr[2] = arr[1]`
- `arr[1] = arr[0]`
- 最后 `arr[0] = 5`

结论：

- 插入头部需要整体右移
- 右移通常从后往前

### 题 6 在指定位置插入元素

核心思路：

1. 先从最后一个有效元素开始右移
2. 给目标下标腾出位置
3. 再赋新值

结论：

- 插入本质是“先腾位置，再放元素”
- 删除更像“先覆盖，再缩短有效区间”

### 题 7 查找某个元素是否存在

核心思路：

- 从头到尾遍历，逐个比较

结论：

- 最直接的是顺序查找
- 普通无序数组里，时间复杂度通常看作 `O(n)`

### 题 8 找最大值和最小值

核心思路：

- 遍历数组
- 用变量记录当前最大值和最小值

常见写法：

- 初始最大值、最小值都先设为 `arr[0]`

结论：

- 一次遍历就可以完成

### 题 9 数组反转

核心思路：

- 头尾双指针交换

例如：

- `left = 0`
- `right = arr.length - 1`

每次交换：

- `arr[left]` 和 `arr[right]`

然后：

- `left++`
- `right--`

直到：

- `left >= right`

### 题 10 删除数组中所有等于某个值的元素

推荐思路：

- 使用双指针

可以这样理解：

- 快指针负责扫描原数组
- 慢指针负责维护“新数组有效区间”

如果当前元素不等于目标值：

- 就把它写到慢指针位置
- 然后慢指针后移

结论：

- 这种写法比“删一个挪一次”更干净
- 也是后面很多题的基础模板

## 3. 进阶题参考答案

### 题 11 合并两个有序数组

核心思路：

- 两个指针分别指向两个数组开头
- 比较当前元素，谁小就先放进结果数组

为什么容易出现在面试里：

- 它能同时考数组、双指针、有序性利用

### 题 12 删除重复元素

前提是：

- 数组有序

核心思路：

- 慢指针表示去重后最后一个有效元素位置
- 快指针往后扫描
- 遇到和慢指针位置不同的值，才保留

结论：

- “有序”让重复元素天然挨在一起
- 所以双指针会非常自然

## 4. 这组题真正想让你建立什么感觉

1. 删除通常对应左移
2. 插入通常对应右移
3. 删除尾部最省事
4. 删除头部和中间更容易触发整体移动
5. 数组很多题都可以转成“覆盖 + 有效长度”
6. 双指针是数组题非常高频的思路

## 5. 一句话总结

数组操作题最关键的不是代码有多花，而是你能不能一眼看出：哪些元素该动、哪些元素不该动、移动方向是什么。
