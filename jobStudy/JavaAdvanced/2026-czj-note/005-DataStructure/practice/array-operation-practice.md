# 数组操作练习题

## 1. 这一组练习主要练什么

这一组不是为了刷很难的算法题，而是为了把数组最基础的操作练熟。

重点放在：

- 按下标访问
- 插入元素
- 删除元素
- 元素移动
- 边界判断
- 时间复杂度感觉

如果这部分不熟，后面学：

- `ArrayList`
- 双指针
- 滑动窗口
- 排序题

都会有点虚。

## 2. 做题时先记住一个前提

数组的核心特点是：

- 长度固定
- 连续存储
- 支持按下标访问

所以一旦发生插入、删除，常见代价就是：

- 移动元素

## 3. 基础操作题

### 题 1 删除第一个元素

给定数组：

```java
int[] arr = {10, 20, 30, 40, 50};
```

要求：

- 删除第一个元素
- 删除后逻辑结果变成 `20, 30, 40, 50`

思考点：

1. 哪些元素需要左移
2. 左移从哪个下标开始
3. 时间复杂度大概是什么

### 题 2 删除最后一个元素

给定数组：

```java
int[] arr = {10, 20, 30, 40, 50};
```

要求：

- 删除最后一个元素
- 删除后逻辑结果变成 `10, 20, 30, 40`

思考点：

1. 需不需要整体移动元素
2. 为什么删除最后一个通常比删除第一个更自然

### 题 3 删除指定位置元素

给定数组：

```java
int[] arr = {10, 20, 30, 40, 50};
```

要求：

- 删除下标为 `2` 的元素
- 删除后逻辑结果变成 `10, 20, 40, 50`

思考点：

1. 删除的是值 `30`
2. 从哪个位置开始左移
3. 哪部分元素不需要动

### 题 4 删除指定值的第一个匹配项

给定数组：

```java
int[] arr = {10, 20, 30, 20, 50};
```

要求：

- 删除值为 `20` 的第一个元素
- 删除后逻辑结果变成 `10, 30, 20, 50`

思考点：

1. 先找下标还是直接删
2. 如果值不存在怎么办

### 题 5 在开头插入元素

给定数组：

```java
int[] arr = {10, 20, 30, 40};
```

要求：

- 在最前面插入 `5`
- 逻辑结果变成 `5, 10, 20, 30, 40`

思考点：

1. 哪些元素需要右移
2. 应该从前往后移还是从后往前移

### 题 6 在指定位置插入元素

给定数组：

```java
int[] arr = {10, 20, 40, 50};
```

要求：

- 在下标 `2` 的位置插入 `30`
- 逻辑结果变成 `10, 20, 30, 40, 50`

思考点：

1. 先腾位置还是先赋值
2. 元素右移时为什么通常从后往前

### 题 7 查找某个元素是否存在

给定数组：

```java
int[] arr = {3, 7, 9, 2, 8};
```

要求：

- 判断 `9` 是否存在
- 判断 `10` 是否存在

思考点：

1. 最直接的做法是什么
2. 时间复杂度大概是多少

### 题 8 找最大值和最小值

给定数组：

```java
int[] arr = {12, 5, 19, 3, 8};
```

要求：

- 找出最大值
- 找出最小值

思考点：

1. 需要几次遍历
2. 初始值应该怎么选

### 题 9 数组反转

给定数组：

```java
int[] arr = {1, 2, 3, 4, 5};
```

要求：

- 原地反转成 `5, 4, 3, 2, 1`

思考点：

1. 哪两个位置交换
2. 需要循环到哪里停止

### 题 10 删除数组中所有等于某个值的元素

给定数组：

```java
int[] arr = {1, 2, 2, 3, 2, 4};
```

要求：

- 删除所有值为 `2` 的元素
- 删除后逻辑结果变成 `1, 3, 4`

思考点：

1. 如果每删一个就整体左移，会不会麻烦
2. 能不能用一个“慢指针”表示新数组有效区间

## 4. 进阶一点的小题

### 题 11 合并两个有序数组

给定：

```java
int[] arr1 = {1, 3, 5};
int[] arr2 = {2, 4, 6};
```

要求：

- 合并成一个新的有序数组

思考点：

1. 为什么这个题经常和双指针一起出现
2. 为什么前提强调“有序”

### 题 12 删除重复元素

给定一个有序数组：

```java
int[] arr = {1, 1, 2, 2, 3, 4, 4};
```

要求：

- 删除重复元素
- 保留每个值一个
- 最终逻辑结果变成 `1, 2, 3, 4`

思考点：

1. 为什么“有序”会让问题简单很多
2. 慢指针表示什么，快指针表示什么

## 5. 做这一组题时最容易犯的错

1. 下标越界
2. 删除时左移范围写错
3. 插入时右移方向写反
4. 忘了考虑空数组
5. 忘了考虑删除值不存在
6. 把“物理长度”和“逻辑有效长度”混在一起

## 6. 一句话总结

数组练习的核心，不是背模板，而是彻底搞清楚：发生插入删除时，到底哪些元素需要移动、从哪开始移动、为什么这样移动。
