## 2. 为什么链表题总爱被手撕

因为链表题很适合现场考察这几件事：

1. 你会不会画图
2. 你会不会处理空指针
3. 你会不会稳定改指针
4. 你会不会写出边界完整的代码
5. 你是不是只会背题解

数组题很多时候还能硬写出来。

但链表题如果真没想清楚：

- 一改指针就容易断链

所以它很适合拿来区分“会背”还是“真的会写”。

## 3. LeetCode 官方链表卡片给出的核心方向

根据 LeetCode 官方链表面试卡片和 Linked List Explore，链表这块最核心的训练点集中在：

1. `Two-pointer technique`
2. `dummy node trick`
3. `Reverse Linked List`
4. `Merge Two Sorted Lists`
5. `Linked List Cycle`

这说明官方其实在强调两件事：

- 快慢指针
- 虚拟头结点

也就是说，链表手撕不是散题，而是围绕这两根主线展开。

## 5. 面经里最常见的手撕链表题

结合牛客面经讨论页里能直接看到的题型，链表手撕最常出现的是下面这几道：

### 第一梯队：最常被手写

1. 反转链表
2. 合并两个有序链表
3. 判断链表是否有环
4. 找环入口
5. 删除倒数第 n 个节点
6. 两个链表的第一个公共节点

这一梯队的特点是：

- 题目短
- 很经典
- 但非常能看出你指针是不是稳

### 第二梯队：进阶手撕

1. 回文链表
2. 区间反转
3. k 个一组翻转
4. 合并 k 个有序链表
5. 删除有序链表重复元素 II
6. 单链表排序

这一梯队的特点是：

- 不是只靠一个小技巧
- 往往是多个链表套路组合

### 第三梯队：综合设计题

1. 链表相加
2. 复制随机指针链表
3. LRU 缓存结构

这一梯队更像：

- 链表 + 哈希
- 链表 + 设计题

## 6. 如果面试官说“手撕一道链表题”，你最该优先练什么

建议优先级：

1. 反转链表
2. 合并两个有序链表
3. 判断链表是否有环
4. 找环入口
5. 删除倒数第 n 个节点
6. 找两个链表的第一个公共节点
7. 回文链表

这 7 个优先练的原因很简单：

- 覆盖了反转、合并、快慢指针、虚拟头结点、双指针换头这些核心技巧

## 7. 手撕链表时面试官最爱追问什么

链表题一般不会只停在“你写出来了没有”，还会追问这些：

1. 空链表怎么办
2. 单节点链表怎么办
3. 删除头结点怎么办
4. 为什么要用虚拟头结点
5. 为什么快慢指针这样走
6. 时间复杂度和空间复杂度是多少
7. 能不能递归写
8. 如果不允许额外空间怎么办

所以复习时不能只看主逻辑，还要顺手把这些边界说清楚。

## 8. 手撕链表时最容易暴露的问题

1. `next` 没保存就改指针
2. 删除节点时没找到前驱
3. 忘了返回新头结点
4. 回文链表只会说，不会恢复结构
5. 环入口原理说不清
6. 相交链表只会哈希，不会双指针换头

## 10. 一句话总结

链表手撕面试真正高频的，不是“链表有多少题”，而是少数几道经典题不断变形反复考：反转、合并、快慢指针、虚拟头结点、相交、环。
