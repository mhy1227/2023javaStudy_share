# 树基础笔记

## 1. 什么是树

树是一种非常重要的非线性数据结构。

和数组、链表、栈、队列这种“更像一条线”的结构不同，树更像：

- 层级结构
- 分叉结构

所以你可以先把它想成：

- 从一个根开始，往下不断分支

## 2. 为什么树很重要

因为很多实际问题本身就是层级关系。

例如：

- 文件夹结构
- 组织架构
- 分类目录
- 家谱

这些都更像树，而不是像数组或链表那样一条线。

## 3. 树里的常见概念

当前阶段先认识这些词：

- 根节点
- 父节点
- 子节点
- 叶子节点
- 高度
- 层

这些术语以后会经常出现。

## 4. 根节点

树最上面的那个起点，叫：

- 根节点

它没有父节点。

## 5. 父节点和子节点

如果一个节点下面连着别的节点，那么：

- 上面的是父节点
- 下面的是子节点

## 6. 叶子节点

没有子节点的节点，叫：

- 叶子节点

可以把它理解成：

- 树枝的最末端

## 7. 树为什么属于非线性结构

因为它不是简单排成一条线，而是一个节点可能连出多个分支。

这和：

- 数组
- 链表

这种一维线性思路完全不同。

## 8. 二叉树

当前阶段最常见、最重要的树是：

- 二叉树

可以先简单理解成：

- 每个节点最多有两个子节点

也就是：

- 左孩子
- 右孩子

## 9. 为什么二叉树常见

因为它既足够简单，又能承载很多重要扩展：

- 二叉搜索树
- 平衡树
- 红黑树
- 堆

所以你后面学到很多高级结构，本质上都和二叉树有关。

## 10. 二叉搜索树

当前阶段可以先有个印象：

- 它是一类带有有序规则的二叉树

这样后面你看到：

- `TreeSet`
- `TreeMap`

就不会觉得“排序结构”是凭空出现的。

## 11. 树和集合有什么关系

树不是 Java 集合，但集合里有些实现类会和树思想强相关。

最典型的就是：

- `TreeSet`
- `TreeMap`

所以这里要记住：

- 树是数据结构思想
- `TreeSet` / `TreeMap` 是 Java 中基于这类思想的具体实现

## 12. 当前阶段最该记住的结论

1. 树是非线性结构
2. 树更适合描述层级关系
3. 二叉树是最重要的入门树结构
4. `TreeSet`、`TreeMap` 和树思想有关

## 13. 一句话总结

树的核心不是“按顺序排一串数据”，而是“用分层分叉的方式组织数据”。
