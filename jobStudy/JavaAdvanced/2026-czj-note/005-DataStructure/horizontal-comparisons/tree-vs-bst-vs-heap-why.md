# 树 vs 二叉搜索树 vs 堆

## 1. 为什么这三个总容易混

因为它们都和“树形结构”有关。

很多人一开始会觉得：

- 都长得像树
- 都有根节点
- 都可能有左右孩子

但面试里真正要分清的是：

- 普通树在强调什么
- BST 在强调什么
- 堆又在强调什么

## 2. 先说最短结论

可以先记成：

- 树：看层级和结构
- BST：看有序查找
- 堆：看堆顶优先级

## 3. 普通树最关心什么

普通树最关心的是：

- 节点关系
- 层级结构
- 遍历方式

所以普通树题常考：

- 前中后序遍历
- 层序遍历
- 路径
- 最近公共祖先
- 结构判断

## 4. BST 最关心什么

BST 最关心的是：

- 有序规则

也就是通常要求：

- 左子树节点值小于根
- 右子树节点值大于根

并且这个规则要对整棵子树都成立。

所以 BST 常考：

- 搜索
- 插入
- 删除
- 校验是不是 BST
- 中序遍历是否有序

## 5. 堆最关心什么

堆最关心的是：

- 堆顶元素优先级

例如：

- 大顶堆根最大
- 小顶堆根最小

但它并不要求像 BST 那样整棵树有序。

所以堆更适合：

- TopK
- 第 K 大 / 第 K 小
- 优先队列
- 动态维护最值

## 6. BST 和堆最本质的区别

这一点最容易被问。

### BST

- 强调整体有序规则
- 中序遍历通常有序
- 更适合查找和范围判断

### 堆

- 只强调局部堆序
- 不保证整体有序
- 更适合快速拿最大值 / 最小值

所以不能把堆理解成：

- “另一种有序树”

它和 BST 的目标完全不同。

## 7. 为什么 BST 可能退化，而堆通常强调完全二叉树

BST 如果插入顺序不好，可能退化成链表。

堆通常会结合：

- 完全二叉树

这样更方便：

- 用数组存
- 快速定位父子节点

所以它们在结构维护上也不是同一路子。

## 8. 和 Java 集合怎么对应

可以先这样记：

- 树 / BST 思想 -> `TreeMap` / `TreeSet`
- 堆思想 -> `PriorityQueue`

这也说明：

- `TreeMap` 更偏有序
- `PriorityQueue` 更偏优先级

## 9. 面试里怎么快速回答

可以这样答：

普通树主要是描述层级和节点关系；二叉搜索树是在树结构上加了全局有序规则，所以适合查找和范围问题；堆虽然也是树形结构，但它只强调堆顶优先级，不追求整棵树有序，所以更适合 TopK、优先队列这类题。

## 10. 一句话总结

三者最大的区别不是“形状像不像”，而是“目标到底是什么”：树看结构，BST 看有序，堆看优先级。
