# 栈与队列基础笔记

## 1. 为什么这两个总被放在一起学

因为它们都是非常经典的线性结构，而且都不只是“存数据”，更重要的是：

- 它们规定了数据进出的规则

所以它们和数组、链表相比，更强调：

- 取数据的顺序

## 2. 什么是栈

栈可以先理解成：

- 后进先出

也就是常说的：

- LIFO

可以把它想成：

- 一摞盘子，最后放上去的最先拿下来

## 3. 栈的典型操作

当前阶段先记：

- 入栈
- 出栈
- 看栈顶元素

## 4. 什么是队列

队列可以先理解成：

- 先进先出

也就是：

- FIFO

可以把它想成：

- 排队打饭，先来先服务

## 5. 队列的典型操作

当前阶段先记：

- 入队
- 出队
- 查看队头

## 6. 栈和队列的核心区别

### 栈

- 后进先出

### 队列

- 先进先出

这就是两者最关键的区别。

## 7. 什么是双端队列

双端队列可以理解成：

- 两头都能进出

所以它比普通队列更灵活。

这也是为什么集合章节里：

- `Deque`

很重要。

## 8. 和 Java 集合的关系

在 Java 中：

- 栈曾经常和 `Stack` 联系在一起
- 队列常和 `Queue` 联系在一起
- 双端队列常和 `Deque`、`ArrayDeque`、`LinkedList` 联系在一起

## 9. 栈和队列适合什么场景

### 栈

适合：

- 需要“最后进去的先处理”的场景

### 队列

适合：

- 需要“先来的先处理”的场景

## 10. 当前阶段最该记住的结论

1. 栈是后进先出
2. 队列是先进先出
3. 双端队列更灵活
4. 栈和队列强调的是“访问顺序规则”

## 11. 一句话总结

栈和队列的本质不是“存一批数据”，而是“规定这批数据怎么进、怎么出”。
