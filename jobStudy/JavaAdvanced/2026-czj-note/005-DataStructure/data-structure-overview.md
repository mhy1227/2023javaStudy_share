# 数据结构总览

## 1. 什么是数据结构

数据结构可以先简单理解成：

- 组织数据和管理数据的一种方式

也就是说，重点不是“数据是什么”，而是：

- 数据怎么存
- 数据怎么组织
- 数据怎么查
- 数据怎么增删改

## 2. 为什么要学数据结构

因为同样一批数据，用不同结构去存，效果会很不一样。

例如：

- 有的结构适合按下标访问
- 有的结构适合头尾操作
- 有的结构适合查重
- 有的结构适合排序或层次关系

所以数据结构本质上是在解决：

- 什么问题适合用什么组织方式

## 3. 数据结构和集合是什么关系

这个点一定要分清。

### 数据结构

更偏底层概念。

例如：

- 数组
- 链表
- 栈
- 队列
- 树
- 堆

### 集合

更偏 Java 提供的现成框架和实现类。

例如：

- `ArrayList`
- `LinkedList`
- `HashMap`
- `HashSet`

所以可以这样理解：

- 数据结构是“思想和模型”
- 集合是“Java 里的具体实现和工具”

## 4. 常见分类方式

数据结构通常可以先分成两大类：

- 线性结构
- 非线性结构

### 4.1 线性结构

元素之间更像一条线。

常见有：

- 数组
- 链表
- 栈
- 队列

### 4.2 非线性结构

元素之间不是简单一条线，而是更复杂的层次或网状关系。

常见有：

- 树
- 图

## 5. 逻辑结构和存储结构

这个词以后会遇到，但当前阶段不用太重。

先简单理解：

- 逻辑结构：数据之间“看起来”是什么关系
- 存储结构：数据在内存里“怎么放”

例如：

- 逻辑上是队列
- 实现时可能用数组，也可能用链表

## 6. 当前阶段先学哪些

建议顺序：

1. 数组
2. 链表
3. 栈
4. 队列
5. 树
6. 堆

其中前四个属于第一批最基础内容。

## 7. 学数据结构时最重要的视角

每学一个结构，都建议问自己这几个问题：

1. 它长什么样
2. 它适合解决什么问题
3. 它最擅长什么操作
4. 它最不擅长什么操作
5. Java 里有哪些集合和它有关系

## 8. 当前阶段最该记住的结论

1. 数据结构是组织数据的方式
2. 数据结构不等于 Java 集合，但集合和数据结构强相关
3. 线性结构和非线性结构是第一层分类
4. 数组、链表、栈、队列是最先要掌握的一组

## 9. 一句话总结

学数据结构，核心不是背定义，而是弄清楚“为什么这种结构存在，它解决了什么问题”。
