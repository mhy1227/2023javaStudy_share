# 数据结构族谱

## 1. 为什么要看数据结构族谱

和集合一样，数据结构如果只零散地学，很容易混。

所以也需要一个总图，先知道：

- 哪些是线性结构
- 哪些是非线性结构
- 它们和 Java 集合的关系是什么

## 2. 一个基础版族谱

```text
基础数据结构
├─ 线性结构
│  ├─ 数组
│  ├─ 链表
│  ├─ 栈
│  └─ 队列
│
└─ 非线性结构
   ├─ 树
   ├─ 堆
   └─ 图
```

## 3. 线性结构怎么理解

线性结构的特点可以先理解成：

- 元素之间更像一条线

常见的有：

- 数组
- 链表
- 栈
- 队列

## 4. 非线性结构怎么理解

非线性结构的特点可以先理解成：

- 不是简单一条线，而是层级或网状关系

常见的有：

- 树
- 堆
- 图

## 5. 一个更详细一点的族谱

```text
数据结构
├─ 线性结构
│  ├─ 顺序存储
│  │  └─ 数组
│  └─ 链式存储
│     └─ 链表
│        ├─ 单链表
│        ├─ 双向链表
│        └─ 循环链表
│
├─ 受限线性结构
│  ├─ 栈
│  ├─ 队列
│  └─ 双端队列
│
└─ 非线性结构
   ├─ 树
   │  ├─ 二叉树
   │  ├─ 二叉搜索树
   │  └─ 平衡树 / 红黑树
   ├─ 堆
   │  ├─ 大顶堆
   │  └─ 小顶堆
   └─ 图
```

## 6. 为什么要把栈和队列单独拎出来

因为它们虽然也属于线性结构，但它们和普通数组、链表不同的地方在于：

- 它们对“进出规则”有限制

例如：

- 栈：后进先出
- 队列：先进先出

所以可以把它们看成：

- 受限线性结构

## 7. 为什么堆放在树下面又常被单独讲

因为堆本质上和树结构强相关，但它的使用语义又很特殊：

- 更强调优先级

所以实际学习时常常既把它看作树的一类，又单独拿出来讲。

## 8. 数据结构和集合的对应关系

这个点非常重要。

可以先这样对应：

- 数组 -> `ArrayList`
- 链表 -> `LinkedList`
- 栈 / 队列 / 双端队列 -> `Stack` / `Queue` / `Deque`
- 树 -> `TreeSet` / `TreeMap`
- 堆 -> `PriorityQueue`
- 哈希思想 -> `HashMap` / `HashSet`

这说明：

- 数据结构是底层思想
- 集合是 Java 给你的具体工具

## 9. 当前阶段最该记住的结论

1. 数据结构先分线性和非线性
2. 数组、链表、栈、队列是一组最基础的线性结构
3. 树、堆、图属于更偏非线性的结构
4. Java 集合和这些数据结构思想强相关

## 10. 一句话总结

数据结构族谱的意义，就是让你先看清“这些结构到底各自属于哪一类”，然后后面学集合或做题时能快速对应起来。
