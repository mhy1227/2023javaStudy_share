# 堆基础笔记

## 1. 这里的堆是什么

这里说的“堆”，指的是：

- 数据结构里的堆

不是 JVM 内存里的：

- 堆内存

这两个名字一样，但不是一回事。

所以复习时一定要先分清：

- 数据结构堆
- 内存中的堆

## 2. 堆是什么

堆可以先理解成：

- 一种特殊的树形结构

更准确地说，常见讨论里它通常和：

- 完全二叉树

联系在一起。

## 3. 为什么堆重要

因为它常用来解决：

- 优先级问题

也就是说，不是简单按谁先来谁先处理，而是：

- 谁优先级高，谁先处理

## 4. 大顶堆和小顶堆

当前阶段先记这两个概念：

### 大顶堆

根节点比较大。

### 小顶堆

根节点比较小。

你现在不必急着背严格定义，先知道：

- 堆更强调“堆顶元素”的优先级特征

## 5. 堆和排序、优先级的关系

堆最常出现的场景就是：

- 优先级队列
- 堆排序

所以以后你看到：

- `PriorityQueue`

就要想到它背后和堆思想有关。

## 6. 堆和普通树的区别

普通树强调的是：

- 层级结构

堆强调的是：

- 节点之间的优先级规则

所以堆不是“任何树都行”，而是有明确约束的一类树结构。

## 7. 堆和 PriorityQueue 的关系

可以先这样理解：

- 堆是数据结构思想
- `PriorityQueue` 是 Java 里的具体优先级队列实现

这也是为什么集合章节里提到 `PriorityQueue` 时，会自然连到这里。

## 8. 当前阶段最该记住的结论

1. 数据结构里的堆，不是 JVM 堆内存
2. 堆是一种特殊树结构
3. 堆最常和优先级问题一起出现
4. `PriorityQueue` 和堆思想强相关

## 9. 一句话总结

堆的核心不是“树长什么样”，而是“谁在堆顶、谁优先被处理”。
