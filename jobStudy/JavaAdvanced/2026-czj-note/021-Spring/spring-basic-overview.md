# Spring 基础总览

## 1. 这份文档怎么用

这一篇是 Spring 这一章的总入口。

它主要解决：

- 什么是 Spring
- 为什么 Java 后端项目离不开 Spring
- Spring 这一章到底该从哪些主线来理解

如果你一上来就扎进很多注解和配置，很容易越学越碎，所以先把总览打牢最重要。

## 2. 先记住最核心的 5 句

先把下面这 5 句记住：

1. Spring 本质上是一个帮助我们管理对象和整合基础能力的框架。
2. `IOC` 的核心是对象不再由程序员手动管理，而是交给容器。
3. `AOP` 的核心是在不改业务代码的前提下织入通用逻辑。
4. Spring 事务本质上和代理增强强相关。
5. Spring Boot 不是替代 Spring，而是帮助 Spring 项目更快启动和整合。

## 3. 什么是 Spring

最稳的回答：

> Spring 是 Java 生态里非常核心的一个轻量级开发框架，主要帮助我们做对象管理、依赖注入、AOP 扩展、事务管理以及和各种基础能力的整合。

更口语一点：

> 你可以把 Spring 理解成 Java 后端开发里的基础设施框架，很多对象管理和通用能力都可以交给它。

## 4. 为什么 Spring 这么重要

因为它不是只解决一个点，而是把很多后端项目最核心的基础能力都串起来了。

你可以简单理解成：

- 不用自己到处 `new` 对象
- 不用自己手写大量样板依赖管理
- 事务、日志、权限这类横切逻辑更容易统一处理
- Web 层和业务层更容易组织起来

一句话记忆：

> Spring 把 Java 后端里很多“通用但麻烦”的工作做成了框架能力。

## 5. Spring 这一章最核心的主线

如果你要把 Spring 讲顺，最稳的是抓住下面几条线：

1. `IOC` / 依赖注入
2. `AOP` / 代理增强
3. Bean 生命周期
4. 事务
5. Spring MVC
6. Spring Boot 自动装配

这几条线基本就是大多数 Spring 面试题的骨架。

## 6. IOC 这条线在解决什么

这一条主要在回答：

- 为什么对象不用自己手动创建
- 为什么依赖关系能自动串起来
- 容器为什么能帮你管理对象

一句话理解：

> `IOC` 让对象管理权从程序员手里转交给容器。

## 7. AOP 这条线在解决什么

这一条主要在回答：

- 为什么日志、事务、权限这些逻辑能统一织进去
- 为什么业务代码不用到处写重复模板

一句话理解：

> `AOP` 让通用横切逻辑可以在不侵入业务代码的前提下统一增强。

## 8. Bean 生命周期这条线在解决什么

这一条主要在回答：

- 一个 Bean 什么时候创建
- 初始化在什么时候发生
- 销毁又在什么时候发生

一句话理解：

> Bean 生命周期本质上是在讲一个对象从创建到销毁经历了什么。

## 9. 事务这条线在解决什么

这一条主要在回答：

- 为什么 `@Transactional` 能生效
- 为什么有时候会失效
- 事务传播行为在描述什么

一句话理解：

> Spring 事务本质上是 `AOP` 在事务场景下的一种典型应用。

## 10. Spring MVC 这条线在解决什么

这一条主要在回答：

- 一次请求怎么进来
- 为什么能找到对应 Controller
- 返回 JSON 或视图时 Spring 做了什么

一句话理解：

> Spring MVC 是 Spring 在 Web 层的标准解决方案。

## 11. Spring Boot 这条线在解决什么

这一条主要在回答：

- 为什么有了 Spring 还要 Spring Boot
- 自动装配到底在干什么
- 为什么项目启动会更方便

一句话理解：

> Spring Boot 让 Spring 项目更快搭起来、更少配置、更方便整合。

## 12. 一版适合面试的完整回答

> 我一般会把 Spring 分成几条主线来理解。第一条是 `IOC` 和依赖注入，也就是对象为什么能交给容器统一管理；第二条是 `AOP`，也就是为什么事务、日志、权限这些通用逻辑能在不改业务代码的情况下统一增强；第三条是 Bean 生命周期，解释一个 Bean 从创建到销毁经历了什么；第四条是 Spring MVC，解释一次请求为什么能走到 Controller；最后再延伸到 Spring Boot，也就是为什么项目可以通过自动装配更快启动和整合。

## 13. 最容易答错的点

- 把 Spring 讲成一堆注解罗列
- 只会说 Spring 很重要，但说不清它到底解决了什么
- 把 IOC、DI、AOP、事务完全拆开，讲不成体系

## 14. 一句话收尾

> Spring 这一章真正要讲顺的，不是注解名字，而是它为什么能帮你管理对象、织入增强、组织 Web 请求和整合项目能力。
