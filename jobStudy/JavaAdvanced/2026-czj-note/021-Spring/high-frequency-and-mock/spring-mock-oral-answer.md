# Spring 口语化回答模板

## 1. 这份文档怎么用

这一篇不是知识点原文，而是把 Spring / Spring Boot 面试里常见题目，整理成更接近真实一面口语的表达。

适合在下面两种场景使用：

1. 你已经会知识点了，但说出来很僵
2. 你想把“书面答案”压成“能直接开口答”的版本

## 2. 什么是 Spring

### 口语版

> 我一般会把 Spring 理解成一个基础设施框架。它最核心的事情，一个是帮我们统一管理对象，也就是 IOC 和依赖注入；另一个是帮我们把事务、日志、权限这些通用逻辑统一织进去，也就是 AOP 这条线。所以 Spring 不只是注解多，本质上是它把很多通用能力都接起来了。

## 3. 什么是 IOC

### 口语版

> IOC 我一般就理解成，对象不再由业务代码自己到处 `new`，而是交给 Spring 容器统一管理。这样对象之间的依赖也能由容器自动串起来，耦合度会低很多。

## 4. 什么是 AOP

### 口语版

> AOP 我理解成是把日志、事务、权限这种横切逻辑从业务代码里抽出去，统一增强。这样业务方法就只关心业务本身，不需要每个地方都重复写一样的模板代码。

## 5. `@Transactional` 为什么能生效

### 口语版

> `@Transactional` 能生效，核心不是这个注解自己有执行能力，而是 Spring 给目标对象做了代理。外部调用走到代理对象时，Spring 才能在方法前后统一织入事务开启、提交和回滚逻辑。

## 6. `@Transactional` 为什么会失效

### 口语版

> 这个我一般先看有没有走 Spring 代理，因为同类内部自调用是最经典的失效场景。再往后会看异常是不是被自己吃掉了，或者异常类型是不是没有触发默认回滚。如果还不对，再看是不是切到了异步线程，导致事务上下文没带过去。

## 7. Spring MVC 流程怎么讲

### 口语版

> Spring MVC 我一般会从请求链路讲。请求先进 `DispatcherServlet` 这个总入口，然后它去找对应的处理器，再把 Controller 方法调起来。业务执行完以后，如果是页面场景就走视图解析，如果是前后端分离场景就走消息转换，把对象转成 JSON 再返回。

## 8. 你项目里常用哪些 Spring 注解

### 口语版

> 我一般会分几类来说。组件注册类像 `@Service`、`@RestController`；依赖注入类像 `@Autowired`、`@Resource`；Web 层像 `@RequestBody`、`@RequestParam`、`@RequestMapping`；配置类像 `@Configuration`、`@Bean`；事务和增强相关像 `@Transactional`、`@Validated`、`@ControllerAdvice`。如果结合项目，最常用的其实还是 `@Service`、`@RestController`、`@Autowired`、`@RequestBody`、`@Transactional` 这些。

## 9. Spring 循环依赖怎么讲

### 口语版

> 循环依赖最经典的就是 A 依赖 B，B 又依赖 A。Spring 不是所有循环依赖都能解决，更常见能处理的是单例 Bean 的属性注入场景，因为它可以先把对象实例化出来，再提前暴露一个早期引用，让另一个 Bean 先拿去用。构造器注入那种一开始就得把依赖凑齐，所以通常更难解。

## 10. 什么是 Spring Boot

### 口语版

> Spring Boot 我一般不把它讲成另一个框架，它还是站在 Spring 这套体系上。它最核心的价值是把原来比较重的配置和整合工作简化掉，让项目能更快启动、更快接中间件。

## 11. 为什么有了 Spring 还要 Spring Boot

### 口语版

> 因为传统 Spring 虽然功能很强，但项目一搭起来配置会比较重。Spring Boot 的价值就是通过默认约定把这些样板配置收掉，让项目搭建和整合效率更高。

## 12. 什么是自动装配

### 口语版

> 自动装配我一般会强调一句，它不是无脑全配，而是按条件装配。也就是 Spring Boot 根据你项目里的依赖、配置和运行条件，决定哪些配置类和 Bean 应该生效。

## 13. `starter` 是什么

### 口语版

> `starter` 我理解成场景化依赖包。像 Web、Redis、MyBatis 这种常见能力，Spring Boot 先把这类场景常用的依赖组合好，你引进来以后，再配合自动装配把对应配置接起来。

## 14. 一句总收尾

> 如果让我整体讲 Spring 和 Spring Boot，我一般会先讲 Spring 解决对象管理和能力增强的问题，再讲 Spring MVC 负责 Web 请求处理，最后讲 Spring Boot 负责快速整合和自动装配，这样整条线会比较顺。
