# Spring / Spring Boot 面试前一晚

## 1. 这份文档怎么用

如果现在已经是面试前一晚，不适合再铺大量新知识。

这一篇的目标只有一个：

帮你把 Spring / Spring Boot 这章压到最后一轮能看的程度。

## 2. 今晚不要做什么

- 不要再看太多源码细节
- 不要再无限扩展新框架
- 不要试图把所有八股全背下来
- 不要把时间全耗在最难的细节题上

## 3. 今晚一定要过的主线

只过这 6 条：

1. IOC / 依赖注入
2. AOP
3. 事务生效与失效
4. Spring MVC
5. Spring Boot 自动装配
6. Spring / Spring Boot 项目表达

## 4. 今晚推荐阅读顺序

1. `spring-one-page-cheatsheet.md`
2. `springboot-one-page-cheatsheet.md`
3. `spring-checklist.md`
4. `springboot-checklist.md`
5. `spring-1min-3min-5min-answer-pack.md`
6. `springboot-1min-3min-answer-pack.md`
7. `spring-and-springboot-project-answer-pack.md`

## 5. 今晚必须口述的 8 题

1. 什么是 Spring
2. 什么是 IOC
3. 什么是 AOP
4. `@Transactional` 为什么能生效
5. `@Transactional` 为什么会失效
6. 什么是 Spring Boot
7. 什么是自动装配
8. Spring / Spring Boot 在你项目里分别承担什么角色

## 6. 今晚最容易翻车的点

- 把 Spring Boot 讲成“另一个框架”
- 说自动装配时答成“全自动全都配”
- 只会说事务失效是“同类调用”，却说不清为什么
- 只背三级缓存名字，说不清它解决什么问题
- 项目表达时只会重复“用得很多”

## 7. 今晚看牛客面经后最值得记住的补充点

- 近一年牛客面经里，`Spring Boot 启动流程` 出现频率明显不低
- `@Transactional + @Async` 组合题会被拿来追问
- `Spring Boot 3` 对自动配置文件的变化是加分项
- `三级缓存也解决不了所有循环依赖` 这类表述更像真实二面追问

## 8. 今晚最后 10 分钟做什么

只做三件事：

1. 看一遍两份 one-page
2. 看一遍两份 checklist
3. 自己口述 3 次“Spring + Spring Boot 串讲”

## 9. 一句话收尾

> 面试前一晚最重要的不是再学更多，而是把已经会的东西压缩成“明天真能说出来”的状态。
