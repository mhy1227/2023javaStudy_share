# Spring Boot 配置加载笔记

## 1. 这份文档怎么用

这一篇专门补 Spring Boot 项目里非常常见、但很多人面试时讲不清的一条线：

- 配置是怎么加载的
- `application.yml` / `application.properties` 怎么起作用
- Profile 怎么切换
- 配置覆盖顺序大概怎么理解

这类题特别像：

- 项目场景题
- 线上排障题
- Spring Boot 进阶追问

## 2. 先记住最核心的 6 句

1. Spring Boot 支持外部化配置。
2. 配置不只来自配置文件，还可能来自环境变量、命令行参数、系统属性等。
3. Profile 的作用是按环境切换配置。
4. 同一个配置项可能来自多个位置，所以面试里经常会问覆盖顺序。
5. 配置加载不是“只看 application.yml”，而是一整套 property source 机制。
6. 项目题里最值钱的是把“来源 + 覆盖 + Profile”三件事讲顺。

## 3. 什么是外部化配置

官方文档把这条线叫 Externalized Configuration。

面试里你可以这样答：

> Spring Boot 支持外部化配置，也就是应用配置不一定硬编码在代码里，而是可以从配置文件、环境变量、系统属性、命令行参数等不同来源加载进来。

一句话版：

> 配置和代码分离，而且支持多个来源。

## 4. 常见配置来源有哪些

按项目视角最值得先记的有：

- `application.properties`
- `application.yml`
- 环境变量
- JVM 系统属性
- 命令行参数

你不需要一上来背完整顺序，但要知道：

> Spring Boot 配置来源不止配置文件。

## 5. Profile 是什么

官方文档把 Profiles 单独列出来讲。

面试里最稳的说法是：

> Profile 的作用是按环境切换配置。比如开发、测试、生产环境可以使用不同配置组，Spring Boot 会根据激活的 Profile 去加载对应配置。

一句话版：

> Profile 本质上是环境分组。

## 6. 激活 Profile 怎么讲

你可以这样答：

> 常见做法是通过配置文件、环境变量、系统属性或者命令行参数来激活某个 Profile。项目里最常见的理解就是 dev、test、prod 这类环境切换。

## 7. 配置覆盖顺序怎么讲最稳

这是最容易被问但也最容易答乱的一题。

更稳的说法是：

> Spring Boot 有一整套 property source 顺序，不同来源的配置会按优先级叠加和覆盖。面试里我不会强背完整列表，但会知道命令行参数、环境变量、系统属性和配置文件之间存在覆盖关系，而且配置文件内部也会受 Profile 影响。

这句比硬背完整顺序更稳。

## 8. 如果面试官继续追问“最常见的覆盖关系”

你可以这样答：

> 最常见的理解是：同一个配置项如果同时出现在多个来源，优先级更高的来源会覆盖优先级低的来源。项目里最常见需要关注的是命令行参数、环境变量、系统属性以及不同 Profile 下的配置文件。

## 9. 配置文件格式怎么讲

可以这样答：

> Spring Boot 同时支持 `application.properties` 和 `application.yml`。项目里很多团队会更偏 `yml`，因为层级结构更清晰，但本质上两者都是配置来源。

## 10. 为什么这题在项目里特别重要

可以这样答：

> 因为线上很多“为什么配置没生效”“为什么环境不对”“为什么和本地不一样”的问题，最后都可能回到配置来源、覆盖顺序和 Profile 激活这三件事上。

## 11. 一版适合面试的标准回答

> Spring Boot 支持外部化配置，也就是说配置不只来自 `application.yml` 或 `application.properties`，还可能来自环境变量、系统属性、命令行参数等多个来源。Profile 的作用是按环境切换配置，比如 dev、test、prod。配置真正生效时，不同来源之间会按优先级叠加和覆盖，所以项目里很多配置问题本质上都和配置来源、覆盖顺序以及当前激活的 Profile 有关。面试里我一般会先把这三件事讲清，再根据需要继续展开。 

## 12. 最容易答错的点

- 以为 Spring Boot 配置只看 `application.yml`
- 只会说 Profile，不知道它和覆盖顺序的关系
- 强背一长串优先级顺序，反而主线讲乱
- 不能把配置加载和项目排障联系起来

## 13. 一句话收尾

> Spring Boot 配置加载题最重要的，不是背全优先级清单，而是知道配置从哪里来、怎么覆盖、怎么按环境切换。
