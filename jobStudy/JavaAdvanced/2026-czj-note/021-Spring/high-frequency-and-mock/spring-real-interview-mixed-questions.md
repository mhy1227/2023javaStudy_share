# Spring / Spring Boot 混合面试题

## 1. 这份文档怎么用

这一篇专门处理真实面试里很常见的一种情况：

面试官不会严格按“Spring 一章问完，再问 Spring Boot 一章”来分。

更真实的节奏通常是：

- 先从 Spring 起手
- 再追到事务或 MVC
- 然后很自然切到 Spring Boot
- 最后再落回项目场景

所以这里把 Spring 和 Spring Boot 混着问的高频题单独整理出来。

## 2. 混合高频题 1

### 问题

你怎么理解 Spring 和 Spring Boot？

### 回答方向

> Spring 更偏运行时基础能力，Spring Boot 更偏工程化整合和快速启动。

### 口语版

> 我一般会把 Spring 理解成项目运行时的基础设施，主要解决对象管理、依赖注入、AOP、事务这些基础能力；Spring Boot 则是在这套基础设施上继续往前走，把项目启动、配置管理和常见中间件整合做成更顺的工程化方案。

## 3. 混合高频题 2

### 问题

Spring、Spring MVC、Spring Boot 三者关系怎么讲？

### 口语版

> 我一般会理解成三层。Spring 是底座，主要负责 IOC、AOP、事务这些通用能力；Spring MVC 是 Web 层方案，负责把请求组织起来；Spring Boot 则是工程化整合层，让项目能更快启动、更自然接入这些能力。

## 4. 混合高频题 3

### 问题

`@Transactional` 为什么能生效，放到 Spring Boot 项目里是不是一样？

### 回答方向

> 本质还是 Spring AOP 和代理，不是因为项目用了 Spring Boot 就变了。

### 口语版

> `@Transactional` 能生效，本质上还是 Spring AOP 和代理机制在起作用。Spring Boot 只是把项目整合和启动做得更顺，但事务这条线的根还是 Spring 本体，不是换成 Spring Boot 就换了一套原理。

## 5. 混合高频题 4

### 问题

既然 Spring 已经能做很多事，为什么还需要 Spring Boot？

### 口语版

> 因为 Spring 解决的是基础能力，但传统项目把这些能力整合起来会比较重。Spring Boot 的价值就是把默认配置、启动入口和常见依赖整合工程化，让项目更快落地，而不是去替代 Spring 本身。

## 6. 混合高频题 5

### 问题

如果结合项目，你会怎么一起介绍 Spring 和 Spring Boot？

### 口语版

> 如果结合项目讲，我会说 Spring 更像运行时底座，负责对象管理、事务控制和 Web 请求处理；Spring Boot 则是在这套底座上继续做工程化整合，比如启动入口、配置管理、常见中间件接入。这样一来，项目从业务开发到能力扩展都会更顺。

## 7. 混合高频题 6

### 问题

Spring MVC 和 Spring Boot 有什么关系？

### 回答方向

> Spring MVC 是 Web 层技术，Spring Boot 会把它整合得更自然。

### 口语版

> Spring MVC 本身是 Spring 在 Web 层的解决方案，而 Spring Boot 做的是把 Spring MVC 这种能力更快接进项目。也就是说，Spring MVC 解决“怎么处理请求”，Spring Boot 解决“怎么把这套能力快速组织起来并启动起来”。

## 8. 混合高频题 7

### 问题

自动装配和 Spring IOC 是什么关系？

### 回答方向

> 自动装配不是替代 IOC，而是在 IOC 容器之上决定哪些配置类和 Bean 要被放进去。

### 口语版

> 我会把它理解成：IOC 解决的是对象交给容器管理，而自动装配解决的是“有哪些配置和 Bean 应该被放进这个容器里”。所以自动装配是在 Spring 容器能力之上的进一步工程化封装。

## 9. 混合高频题 8

### 问题

`starter` 和 Spring Bean / 自动装配是什么关系？

### 口语版

> starter 更偏依赖整合，它先把某类场景常用依赖带进来；自动装配再根据条件决定哪些配置类和 Bean 生效。也就是说，starter 带材料，容器和自动装配负责把材料组织成真正可运行的能力。

## 10. 混合高频题 9

### 问题

如果面试官先问你 Spring，再突然追到 Boot 3 版本差异，你怎么接？

### 回答方向

> 先稳住 Spring 主线，再说 Boot 版本差异是工程层变化，不是 Spring 本体概念全变了。

### 口语版

> 我一般会先把主线稳住，也就是 Spring 本体还是对象管理、AOP、事务这些底层能力。Boot 版本差异更多是工程层和自动配置层的变化，比如 Java 版本要求、Jakarta 命名空间、自动配置发现机制这些，不能把它和 Spring 本体基础概念混在一起。

## 11. 混合高频题 10

### 问题

如果让你最后用一段话把 Spring、Spring MVC、Spring Boot 串起来，你会怎么讲？

### 口语版

> 我会这样理解：Spring 是项目运行时的基础设施，主要解决对象管理、依赖注入、AOP 和事务；Spring MVC 是 Spring 在 Web 层的请求处理方案；Spring Boot 则是在这套生态上继续往前做工程化整合，让项目能更快启动、更自然配置、更方便接入常见能力。所以这三者不是并列替代关系，而是底座、Web 层和工程化整合层的关系。

## 12. 一句话收尾

> 真实面试里最容易失分的，不是单点不会，而是被 Spring 和 Spring Boot 混着问时主线突然乱掉。
