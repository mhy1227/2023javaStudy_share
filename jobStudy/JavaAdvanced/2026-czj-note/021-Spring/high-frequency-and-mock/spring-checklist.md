# Spring 考前 Checklist

## 1. 这份文档怎么用

这一篇是考前快速过一遍用的。

如果你没时间重看全部 Spring 文档，就拿这份 checklist 对着过：

- 会不会定义
- 会不会一句话解释
- 会不会接追问
- 会不会结合项目说

只要有一项明显不稳，就回去补对应主线文档。

## 2. Spring 本体

- 我能不能一句话说清什么是 Spring。
- 我能不能讲出 Spring 最核心的几条主线：IOC、AOP、事务、MVC、Spring Boot。
- 我能不能解释 Spring 为什么在 Java 后端里这么核心。

## 3. IOC / 依赖注入

- 我能不能解释什么是 IOC。
- 我能不能讲清为什么叫控制反转。
- 我能不能讲出依赖注入和 IOC 的关系。
- 我能不能说出常见注入方式。
- 我能不能解释为什么构造器注入更值得推荐。

## 4. AOP

- 我能不能讲清什么是 AOP。
- 我能不能解释什么叫横切逻辑。
- 我能不能举出事务、日志、权限这类典型场景。
- 我能不能说明 Spring 为什么需要 AOP。
- 我能不能顺着 AOP 接到事务。

## 5. Bean / 生命周期

- 我能不能一句话说清什么是 Bean。
- 我能不能讲出 Bean 生命周期的主干流程。
- 我能不能解释“从实例化到销毁”大概发生了什么。
- 我会不会把生命周期答成一堆接口名堆砌。

## 6. 事务

- 我能不能讲清 `@Transactional` 为什么能生效。
- 我能不能解释为什么事务一般放在 Service 层。
- 我能不能讲出事务传播行为在描述什么。
- 我能不能说清 `REQUIRED` 和 `REQUIRES_NEW`。
- 我知不知道不是所有异常都会默认回滚。

## 7. 事务失效

- 我能不能讲出同类内部自调用为什么会失效。
- 我能不能解释异常被吃掉为什么会影响回滚。
- 我能不能说出 Bean 不归 Spring 管也会有问题。
- 我能不能提到异步 / 多线程切线程带来的事务上下文问题。

## 8. Spring MVC

- 我能不能讲清 Spring MVC 的主流程。
- 我能不能解释 `DispatcherServlet` 是什么。
- 我能不能区分 `HandlerMapping` 和 `HandlerAdapter`。
- 我能不能说明返回页面和返回 JSON 的处理差别。

## 9. 常见注解

- 我能不能把常见注解按类别讲。
- 我能不能区分 `@Autowired` 和 `@Resource`。
- 我能不能区分 `@Component` 和 `@Bean`。
- 我能不能区分 `@RequestBody` 和 `@RequestParam`。
- 我能不能区分 `@ResponseBody` 和 `@RestController`。

## 10. 循环依赖

- 我能不能解释什么是循环依赖。
- 我能不能讲清为什么 Spring 不是所有循环依赖都能解决。
- 我能不能区分属性注入和构造器注入的处理差异。
- 我能不能用自己的话解释三级缓存到底在解决什么问题。

## 11. Spring Boot

- 我能不能讲清什么是 Spring Boot。
- 我能不能解释为什么有了 Spring 还要 Spring Boot。
- 我能不能讲清 Spring、Spring MVC、Spring Boot 的关系。
- 我能不能解释什么叫约定优于配置。

## 12. 自动装配

- 我能不能解释什么是自动装配。
- 我能不能强调“按条件装配”而不是“无脑全配”。
- 我能不能说清 `@SpringBootApplication` 是启动入口。
- 我能不能说出 `@EnableAutoConfiguration` 的作用。
- 我能不能讲出 `starter` 和自动装配的关系。

## 13. 开口表达

- 我回答时是不是还在背书，而不是在讲逻辑。
- 我能不能把 Spring 本体和 Spring Boot 连成一条线。
- 我能不能结合项目举一个 Controller / Service / 事务 / 注解使用场景。
- 我能不能在 30 秒、1 分钟、3 分钟三种长度下都讲得出来。

## 14. 如果时间只够最后过一遍

优先检查这 10 个：

1. 什么是 Spring
2. 什么是 IOC
3. 什么是 AOP
4. `@Transactional` 为什么能生效
5. `@Transactional` 为什么会失效
6. Spring MVC 流程
7. 常见注解分类
8. 循环依赖
9. 什么是 Spring Boot
10. 什么是自动装配

## 15. 一句话收尾

> Checklist 的意义不是让你全背下来，而是帮你快速发现“哪一条主线现在一问就卡壳”。
