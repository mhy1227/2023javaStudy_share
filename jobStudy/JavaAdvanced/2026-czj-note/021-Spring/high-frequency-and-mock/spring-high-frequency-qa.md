# Spring 高频问答

## 1. 这份文档怎么用

这一篇只做一件事：

把 Spring 本体里最常被问到的高频题，按“问题 + 回答骨架”的方式整理出来。

它不替代主线文档，而是帮助你在看完主线后，快速进入答题状态。

## 2. 什么是 Spring

### 问题

什么是 Spring？

### 回答骨架

> Spring 是 Java 生态里非常核心的一个轻量级开发框架，核心价值在于对象管理、依赖注入、AOP 扩展、事务管理以及和各种基础能力的整合。

### 追问点

- 为什么 Java 项目离不开 Spring
- Spring 最核心的两三条主线是什么

## 3. 什么是 IOC

### 问题

什么是 IOC？

### 回答骨架

> IOC 就是控制反转，核心思想是对象的创建和依赖管理不再由业务代码自己控制，而是交给 Spring 容器统一处理。

### 追问点

- 为什么叫控制反转
- IOC 和依赖注入是什么关系

## 4. 什么是依赖注入

### 问题

依赖注入怎么理解？

### 回答骨架

> 依赖注入可以理解成 IOC 的一种实现方式，Spring 在创建对象时，把它依赖的其他对象一并注入进去，这样业务代码就不需要自己到处 `new` 依赖对象。

### 追问点

- 依赖注入有哪些方式
- 为什么更推荐构造器注入

## 5. 什么是 AOP

### 问题

什么是 AOP？

### 回答骨架

> AOP 就是面向切面编程，核心思想是在不修改业务代码的前提下，把日志、事务、权限这类通用横切逻辑统一织入到目标方法前后。

### 追问点

- 什么叫横切逻辑
- Spring 事务和 AOP 有什么关系

## 6. Spring 为什么需要 AOP

### 问题

为什么 Spring 里要有 AOP？

### 回答骨架

> 因为项目里很多通用逻辑会在多个业务模块重复出现，比如事务、日志、权限、监控。如果这些逻辑全写进业务代码里，重复度会很高，维护成本也大，所以 Spring 通过 AOP 把它们统一抽离出来。

### 追问点

- 哪些场景适合 AOP
- 哪些场景不适合

Bean 生命周期大概怎么讲？

### 回答骨架

## 9. Spring 事务为什么能生效

### 问题

`@Transactional` 为什么能生效？

### 回答骨架

> Spring 事务本质上是基于 AOP 和代理机制做的。调用代理对象的方法时，Spring 才能在方法前后统一织入事务开启、提交和回滚逻辑。

### 追问点

- 为什么会失效
- 为什么一般放在 Service 层

## 10. `@Transactional` 为什么会失效

### 问题

事务注解为什么会失效？

### 回答骨架

> 常见原因包括没有经过 Spring 代理、同类内部自调用、异常被吃掉、异常类型没有触发默认回滚、对象没交给 Spring 管理，以及异步切线程导致事务上下文丢失。

### 追问点

- 同类内部调用为什么绕过代理
- 为什么不是所有异常都回滚

## 11. Spring MVC 执行流程怎么讲

### 问题

Spring MVC 请求流程是什么？

### 回答骨架

> 请求先进入 `DispatcherServlet`，它作为前端控制器统一接收请求；然后通过 `HandlerMapping` 找到处理器，再通过 `HandlerAdapter` 调用 Controller；Controller 返回结果后，Spring MVC 再做视图解析或消息转换，最后把结果返回给客户端。

### 追问点

- `DispatcherServlet` 为什么叫前端控制器
- `HandlerMapping` 和 `HandlerAdapter` 分别做什么

## 12. 常见注解怎么分类讲

### 问题

你项目里常用哪些 Spring 注解？

### 回答骨架

> 我一般分几类来讲。组件注册类有 `@Component`、`@Service`、`@Repository`、`@RestController`；依赖注入类有 `@Autowired`、`@Resource`；配置类有 `@Configuration`、`@Bean`、`@Value`、`@ConfigurationProperties`；Web 层有 `@RequestMapping`、`@RequestBody`、`@RequestParam`；事务和增强相关有 `@Transactional`、`@Validated`、`@ControllerAdvice`、`@ExceptionHandler`。

### 追问点

- `@Autowired` 和 `@Resource` 区别
- `@Component` 和 `@Bean` 区别

## 13. 循环依赖怎么讲

### 问题

Spring 循环依赖怎么理解？

### 回答骨架

> 循环依赖就是对象之间互相依赖形成闭环。Spring 不是所有循环依赖都能解决，更常见能处理的是单例 Bean 的属性注入循环依赖，因为 Spring 可以通过提前暴露早期引用来打破闭环。

### 追问点

- 为什么构造器循环依赖更难处理
- 为什么会扯到三级缓存
