# Spring Boot 模拟面试卷

## 1. 使用说明

这一份专门针对 Spring Boot。

练法建议：

1. 先按顺序答
2. 再随机抽题答
3. 每答完一轮，看自己到底是卡在“定位题”还是“自动装配题”

## 2. 第一组：基础定位题

1. 什么是 Spring Boot？
2. Spring Boot 和 Spring 是什么关系？
3. 为什么有了 Spring 还要 Spring Boot？
4. Spring、Spring MVC、Spring Boot 三者关系怎么讲？
5. Spring Boot 最核心解决了什么问题？
6. 什么叫约定优于配置？

## 3. 第二组：自动装配主线

7. 什么是自动装配？
8. 自动装配为什么能减少配置？
9. 自动装配的核心思想是什么？
10. 为什么说自动装配不是“全都自动配”？
11. 什么叫按条件装配？
12. 条件装配通常会看哪些条件？

## 4. 第三组：启动入口与注解

13. `@SpringBootApplication` 是什么？
14. 为什么它是 Spring Boot 的启动入口？
15. `@SpringBootApplication` 里面大概包含什么能力？
16. `@EnableAutoConfiguration` 是干什么的？
17. `@ComponentScan` 在 Spring Boot 里为什么也经常会被提到？

## 5. 第四组：条件注解与自动配置

18. 常见条件注解知道哪些？
19. `@ConditionalOnClass` 大概是什么意思？
20. `@ConditionalOnMissingBean` 大概是什么意思？
21. `@ConditionalOnProperty` 大概是什么意思？
22. 为什么自动装配一定要配合条件注解？

## 6. 第五组：starter

23. `starter` 是什么？
24. 为什么 `starter` 有价值？
25. `starter` 和自动装配是什么关系？
26. 为什么说一个偏依赖整合，一个偏配置生效？
27. 如果让你一句话解释 `starter`，你会怎么说？

## 7. 第六组：项目使用题

28. 你项目里 Spring Boot 最直接带来的好处是什么？
29. 你项目里最常见的 Spring Boot 能力有哪些？
30. 如果没有 Spring Boot，你觉得项目开发哪里会更麻烦？
31. 结合项目说一下 Spring Boot 和配置管理的关系。
32. 如果让你对非技术同学解释 Spring Boot 的价值，你会怎么说？

## 8. 第七组：追问串联题

33. 如果面试官先问“什么是 Spring Boot”，你下一句最自然会接什么？
34. 如果面试官继续问“为什么它能减少配置”，你怎么接？
35. 如果再追到“自动装配原理”，你怎么避免答散？
36. 如果再追到 `@SpringBootApplication`，你怎么顺着讲？
37. 如果再追到 `starter`，你怎么把它和自动装配区分开？

## 9. 最后 5 题，作为收尾自测

38. 不看笔记，30 秒说清什么是 Spring Boot。
39. 不看笔记，1 分钟说清为什么有了 Spring 还要 Spring Boot。
40. 不看笔记，1 分钟说清自动装配。
41. 不看笔记，1 分钟说清 `@SpringBootApplication`。
42. 不看笔记，2 分钟把 Spring Boot 这条线完整串起来。

## 10. 一句话收尾

> Spring Boot 这份模拟卷的关键不是答很深，而是看你能不能把“定位 -> 自动装配 -> 启动入口 -> starter”这条线讲得清楚、稳定、不绕。
