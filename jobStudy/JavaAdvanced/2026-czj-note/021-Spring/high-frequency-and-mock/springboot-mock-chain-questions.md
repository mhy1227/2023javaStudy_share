# Spring Boot 模拟追问链

## 1. 这份文档怎么用

这一篇专门练 Spring Boot 在真实面试里的连续追问。

Spring Boot 的追问链通常不像 Spring 本体那么散，它更集中在三条线上：

- 它为什么出现
- 自动装配怎么理解
- 启动注解和 `starter` 怎么讲

## 2. 追问链一：从 Spring Boot 是什么开始

### 第一问

什么是 Spring Boot？

### 第一层稳答

> Spring Boot 是基于 Spring 生态的一套快速开发框架，核心目标是减少繁琐配置，提升项目搭建、整合和启动效率。

### 第二问通常怎么追

- 为什么有了 Spring 还要 Spring Boot
- Spring Boot 和 Spring 是什么关系

### 第三层稳答

> 我一般不会把 Spring Boot 讲成另一套框架，它还是站在 Spring 这套体系上。Spring 解决的是 IOC、AOP、事务这些基础能力，而 Spring Boot 更像是把项目工程化整合和快速启动这件事做得更顺。

## 3. 追问链二：从“为什么需要它”追到“它解决了什么问题”

### 第一问

为什么有了 Spring 还要 Spring Boot？

### 第一层稳答

> 因为传统 Spring 项目虽然能力很强，但配置和整合相对繁琐。Spring Boot 通过默认约定和工程化整合，把很多样板配置收掉了，让项目更快搭起来。

### 第二问通常怎么追

- 它最核心解决了什么问题
- 什么叫约定优于配置

### 第三层稳答

> 我理解它最核心解决的是开发效率和工程整合问题。约定优于配置的意思就是，常见场景先给你一套默认可用方案，只有特殊情况再额外配置，而不是一切从零手写。

## 4. 追问链三：从自动装配追到条件装配

### 第一问

什么是自动装配？

### 第一层稳答

> 自动装配就是 Spring Boot 根据当前项目的依赖、配置和运行条件，自动决定加载哪些配置类和 Bean，从而减少手动配置工作。

### 第二问通常怎么追

- 自动装配为什么能减少配置
- 自动装配是不是启动时全都配一遍

### 第三层稳答

> 自动装配的核心不是“全都自动配”，而是“按条件装配”。也就是只有满足条件，对应的自动配置才真正生效。

### 再下一层常见追问

- 什么叫条件装配
- 常见条件注解有哪些

## 5. 追问链四：从 `@SpringBootApplication` 追到启动入口

### 第一问

`@SpringBootApplication` 是什么？

### 第一层稳答

> `@SpringBootApplication` 是 Spring Boot 启动类上的核心组合注解，可以理解成 Spring Boot 启动的统一入口。

### 第二问通常怎么追

- 它里面大概包含什么
- `@EnableAutoConfiguration` 是干什么的

### 第三层稳答

> 面试里我一般会讲三层能力：配置类声明、开启自动装配、组件扫描。对应可以记 `@SpringBootConfiguration`、`@EnableAutoConfiguration`、`@ComponentScan`。

## 6. 追问链五：从 `@EnableAutoConfiguration` 追到条件注解

### 第一问

`@EnableAutoConfiguration` 是干什么的？

### 第一层稳答

> 它的核心作用就是开启 Spring Boot 的自动装配能力，让 Spring Boot 根据条件去加载合适的自动配置类。

### 第二问通常怎么追

- 条件怎么判断
- 常见条件注解知道哪些

### 第三层稳答

> 条件判断通常会看类路径里有没有某个类、容器里有没有某个 Bean、配置文件里有没有某个属性。常见条件注解像 `@ConditionalOnClass`、`@ConditionalOnMissingBean`、`@ConditionalOnProperty`。

## 7. 追问链六：从 `starter` 追到自动装配关系

### 第一问

`starter` 是什么？

### 第一层稳答

> `starter` 是 Spring Boot 提供的一组场景化依赖集合，它把某类功能常用的依赖和整合方式先准备好，开发者引入后就能更快接上这类能力。

### 第二问通常怎么追

- 它和自动装配是什么关系
- 为什么说一个偏依赖整合，一个偏配置生效

### 第三层稳答

> 我一般会这样区分：`starter` 负责把材料带进来，解决的是“要引哪些依赖”；自动装配负责把这些材料按条件拼起来，解决的是“这些依赖进来以后配置怎么生效”。

## 8. Spring Boot 这组题最容易怎么答散

- 把 Spring Boot 讲成“另一个框架”
- 只会说“少写配置”，但讲不清少了什么
- 知道自动装配，却说不清“按条件装配”
- 背了 `@SpringBootApplication` 里面的注解名，但不知道它们各自干嘛
- 把 `starter` 和自动装配混成一个概念

## 9. 这篇怎么练最有效

建议练法：

1. 先回答第一问
2. 强迫自己只用 30 秒讲清
3. 再继续接第二层追问
4. 发现自己一乱，就回去补基础定位题

Spring Boot 这组题的关键不是深，而是“清楚、顺、别绕”。

## 10. 一句话收尾

> Spring Boot 模拟追问链的核心，是把“为什么需要它 -> 自动装配怎么理解 -> 启动入口和 starter 怎么配合”这条线讲顺。
