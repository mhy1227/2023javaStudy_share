# Spring Boot 易错陷阱题

## 1. 这份文档怎么用

这一篇专门整理 Spring Boot 里那些“看起来会，实际上很容易答错”的题。

这类题特别像：

- 牛客面经里的追问
- 一面后半段的加压题
- 二面里用来区分“背过”和“理解过”的题

## 2. 陷阱一：把 Spring Boot 讲成“替代 Spring”

### 常见错误答法

> Spring Boot 是新的框架，替代了 Spring。

### 更稳的答法

> Spring Boot 不是替代 Spring，而是站在 Spring 生态上的快速开发和工程化整合方案。Spring 负责基础能力，Spring Boot 负责让项目更快启动和整合。

## 3. 陷阱二：把自动装配讲成“启动时全都配一遍”

### 常见错误答法

> 只要引了依赖，Spring Boot 启动时就把所有配置全装进去。

### 更稳的答法

> 自动装配的核心不是“全都装”，而是“按条件装配”。只有满足类、Bean、配置属性这些条件，对应配置才会真正生效。

## 4. 陷阱三：只会背 `@SpringBootApplication`，不会解释

### 常见错误答法

> 里面有三个注解，背出来就结束了。

### 更稳的答法

> 不只要知道名字，还要知道它们分别承担配置类声明、自动装配和组件扫描这三层能力。

## 5. 陷阱四：`starter` 和自动装配分不清

### 常见错误答法

> starter 就是自动装配。

### 更稳的答法

> `starter` 更偏依赖整合，负责把某类场景常用依赖带进来；自动装配更偏配置生效，负责让这些依赖按条件接起来。

## 6. 陷阱五：Spring Boot 启动流程完全讲不出来

### 常见错误表现

- 只会说 main 方法启动
- 说不出 `SpringApplication.run`
- 说不清自动装配和容器初始化怎么串起来

### 更稳的答法

> 面试里不一定要源码级展开，但至少要知道启动入口是 `SpringApplication.run`，它会准备环境、创建并刷新容器、触发自动装配，把整条启动链串起来。

## 7. 陷阱六：Spring Boot 3 还在只说 `spring.factories`

### 为什么这题容易错

牛客里已经能看到有人问到 Boot 3 相关变化。

### 更稳的答法

> 如果只讲传统版本，说自动配置类通过 `spring.factories` 被发现没有问题；但如果提到 Spring Boot 3，最好顺手补一句，自动配置发现机制已经切到 `META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports`，这是加分项。

## 8. 陷阱七：`@Transactional + @Async` 组合题

### 常见错误答法

> 这两个一起用肯定没问题，都是 Spring 注解。

### 更稳的答法

> 这类题要先想到两者都和代理、线程切换有关。尤其是 `@Async` 会切线程，而事务很多时候和当前线程上下文绑定，所以组合使用时要特别关注事务上下文是否还能按预期传递。

## 9. 陷阱八：只会说“少写配置”

### 常见错误答法

> Spring Boot 最大好处就是少写配置。

### 更稳的答法

> 这个说法不算错，但太浅。更准确地说，它是把项目启动、配置管理和常见中间件整合工程化了，所以真正的价值更偏工程效率和团队标准化。

## 10. 陷阱九：项目题里说不出实际价值

### 常见错误表现

- 一直重复“很方便”
- 一直说“用得很多”
- 没有项目化表达

### 更稳的答法

> 可以从项目启动统一、配置管理集中、中间件接入自然、团队协作标准化这几个角度讲。

## 11. 这类陷阱题怎么练

建议这样练：

1. 先看错误答法
2. 再看稳答法
3. 强迫自己每题只用两三句话讲清

这类题不适合讲太长，越长越容易露怯。

## 12. 一句话收尾

> Spring Boot 陷阱题最怕的不是不会，而是“知道几个关键词，但回答一展开就露出理解层次不够”。
