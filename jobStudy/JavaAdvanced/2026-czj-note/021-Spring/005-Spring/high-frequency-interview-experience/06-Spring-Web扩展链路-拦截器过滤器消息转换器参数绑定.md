# Spring Web 扩展链路：拦截器、过滤器、消息转换器、参数绑定

## 1. 这份文档是干什么的
这份文档补的是 Spring Web 层最容易被追问的链路问题：

1. 拦截器和过滤器区别
2. 参数绑定怎么做
3. 为什么返回对象会自动变成 JSON
4. 消息转换器是什么
5. 这些能力在项目里怎么落地

---

## 2. 先记结论
先记住这几句：

1. 过滤器是 Servlet 规范里的，更底层。
2. 拦截器是 Spring MVC 里的，更贴近业务 Handler。
3. 参数绑定本质上是 Spring MVC 把请求数据转换成方法参数。
4. 返回对象自动转 JSON，靠的是消息转换器。
5. 前后端分离项目里，这几块通常是一起出现的。

---

## 3. 过滤器和拦截器有什么区别
标准回答：

> 过滤器是 Servlet 规范里的组件，作用在更底层，请求进入 Spring MVC 之前就可以先做处理；拦截器是 Spring MVC 体系里的，更靠近 Controller，能够拿到 Handler、方法信息这些上下文。简单说，过滤器更底层，拦截器更贴近 Spring Web。

---

## 4. 项目里一般怎么选
示例回答：

> 如果是一些和 Spring 无关、偏底层的通用处理，比如字符编码、跨域、请求包装，这类更适合放过滤器；如果是登录校验、权限判断、接口日志、用户上下文注入这类和业务接口更贴近的处理，我会更倾向用拦截器。

---

## 5. 参数绑定怎么理解
标准回答：

> 参数绑定本质上是 Spring MVC 根据请求里的不同数据来源，比如路径参数、查询参数、表单参数、请求体 JSON，把它们转换成 Controller 方法的入参对象。

更口语一点：

> 前端把数据传进来，Spring MVC 负责把这些数据“接住并转成你方法想要的参数类型”。

---

## 6. `@RequestParam`、`@PathVariable`、`@RequestBody` 怎么放到这条链路里
示例回答：

> 这几个注解就是参数绑定过程里的常见入口。`@RequestParam` 接查询参数或表单参数，`@PathVariable` 接路径占位参数，`@RequestBody` 接请求体里的 JSON。面试里如果讲参数绑定，顺手把这三个带进去就很自然。

---

## 7. 为什么返回对象会自动变成 JSON
示例回答：

> 因为 Spring MVC 在处理返回值时，会根据返回类型和当前配置选择合适的消息转换器。如果是前后端分离场景，常见就是通过 JSON 消息转换器把 Java 对象序列化成 JSON，然后写回响应体。

---

## 8. 什么是消息转换器
标准回答：

> 消息转换器可以理解成 Spring MVC 里负责“对象和 HTTP 消息之间互相转换”的组件。请求进来时，它可以把请求体转换成 Java 对象；返回响应时，它也可以把 Java 对象转换成 JSON、字符串等格式。

---

## 9. 面试官为什么爱把这几题放一起问
因为这几题本质上都属于 Spring MVC 的请求处理链路：

1. 请求先经过过滤器
2. 再进入 Spring MVC
3. 拦截器可以做进一步拦截
4. 参数绑定把请求数据转成方法参数
5. Controller 返回对象后，消息转换器再把它转成响应内容

如果你把这条链讲顺，面试官会觉得你不是在背碎片题。

---

## 10. 一段完整回答示例
如果面试官问：

> 你讲一下拦截器、过滤器、参数绑定和消息转换器。

你可以这样答：

> 我会把它们放到同一条 Web 请求链里理解。过滤器属于 Servlet 规范，更底层，请求进入 Spring MVC 前就能处理；拦截器属于 Spring MVC，更靠近 Controller，能拿到 Handler 和方法信息，所以更适合做登录校验、权限控制这类和业务接口相关的事情。参数绑定则是 Spring MVC 把路径参数、查询参数、请求体 JSON 转成 Controller 方法入参的过程，比如 `@RequestParam`、`@PathVariable`、`@RequestBody` 都属于这条链。方法执行完后，如果返回的是对象，Spring MVC 会通过消息转换器把它序列化成 JSON 再写回响应体。

---

## 11. 这题怎么和项目结合
你可以这样接：

> 结合我的项目，如果是前后端分离接口，请求进来之后一般会先经过一些通用过滤，然后在拦截器里处理登录态或上下文信息。Controller 这边通过 `@RequestBody`、`@RequestParam` 接参数，最后返回统一结果对象，再由消息转换器转成 JSON 响应给前端。

---

## 13. 速记版
至少记住下面 6 句：

1. 过滤器更底层，属于 Servlet。
2. 拦截器更贴近 Spring MVC 和 Controller。
3. 参数绑定是把请求数据转成方法参数。
4. `@RequestParam`、`@PathVariable`、`@RequestBody` 都属于参数绑定链路。
5. 返回对象转 JSON，靠消息转换器。
6. 这几题本质上都是一条 Spring MVC 请求处理链。
