## 2. IO 这块高频问题主要集中在哪

IO 这章的高频点通常比较集中，主要就是这几类：

1. `File` 和流的区别
2. 字节流和字符流的区别
3. 输入流和输出流的区别
4. 缓冲流为什么存在
5. `BufferedReader` 和 `readLine()`
6. 转换流是做什么的
7. 编码和乱码为什么会出现
8. 对象流和序列化
9. `try-with-resources` 和 `finally` 的区别
10. `System.out`、`PrintStream`

## 3. 第一梯队：几乎必问

### 3.1 字节流和字符流的区别

回答主线：

- 字节流按字节处理数据，更适合二进制文件
- 字符流按字符处理数据，更适合文本

### 3.2 InputStream / OutputStream、Reader / Writer 的区别

回答主线：

- `InputStream` / `OutputStream` 是字节流父类
- `Reader` / `Writer` 是字符流父类
- 输入是读进来，输出是写出去

### 3.3 File 是做什么的

回答主线：

- `File` 主要表示文件和目录路径
- 它不等于流
- 真正读写内容还是靠各种流

### 3.4 try-with-resources 为什么更推荐

回答主线：

- 代码更简洁
- 不容易漏关资源
- 很适合 IO 场景

## 4. 第二梯队：非常爱追问

### 4.1 为什么还要有缓冲流

回答主线：

- 提升读写效率
- 提升使用便利性
- 它通常包在基础流外面

### 4.2 BufferedReader 为什么常用

回答主线：

- 它是字符缓冲流
- 很适合处理文本
- `readLine()` 是高频方法

### 4.3 转换流是做什么的

回答主线：

- 负责在字节流和字符流之间搭桥
- `InputStreamReader` 和 `OutputStreamWriter` 最核心

### 4.4 为什么会出现乱码

回答主线：

- 字节和字符之间的编码 / 解码规则不一致
- 本质上和编码处理有关系

## 5. 第三梯队：常作为延伸问题

### 5.1 对象流是做什么的

回答主线：

- 读写对象
- 和序列化 / 反序列化强相关

### 5.2 Serializable 是做什么的

回答主线：

- 它是标记接口
- 告诉 Java 这个对象支持序列化

### 5.3 PrintStream 和 PrintWriter 的作用

回答主线：

- 都属于打印流
- 核心特点是输出方便
- `System.out` 和 `PrintStream` 强相关

## 6. 最容易出的组合题

IO 这章经常不是单点问，而是一串问下来。

例如：

1. 字节流和字符流的区别
2. 为什么字符流更适合文本
3. 为什么还要有转换流
4. 乱码怎么来的
5. 最后再问 `try-with-resources`

所以复习时不要把这些点完全割裂开。

## 7. 如果只剩一轮时间，优先背什么

建议优先级：

1. 字节流和字符流区别
2. `File` 和流的区别
3. 缓冲流的作用
4. 转换流和编码
5. `try-with-resources`
6. 对象流和序列化

## 8. 一段比较稳的总回答

可以这样答：

Java IO 我一般先按体系理解，先分 `File` 和流，`File` 主要表示文件和目录路径，本身不负责真正读写内容；流再分成字节流和字符流，字节流更适合二进制，字符流更适合文本；在基础流上还会有缓冲流提升效率，转换流负责在字节和字符之间搭桥，并且和编码问题强相关。实际开发里 IO 还经常和资源关闭联系在一起，所以 `try-with-resources` 也是一个高频点。

## 9. 当前阶段最该记住的结论

1. IO 高频点并不散，主要集中在少数几条主线
2. 字节流 / 字符流 / 缓冲流 / 转换流 是最核心的一组
3. 编码和乱码问题通常和转换流一起出现
4. IO 和异常、资源关闭天然绑在一起

## 10. 一句话总结

IO 这一章的高频面试题，核心不是背很多类名，而是把“文件、流、文本、编码、资源关闭”这几条主线讲顺。
