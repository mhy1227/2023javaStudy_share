# IO 场景选型问答

## 1. 这一份怎么用

这一份不是再讲 IO 族谱，而是把面试里最常见的：

- 看到场景怎么选流

这种题单独整理出来。

适合用法：

- 看完主干文档后复盘
- 面试前做快问快答
- 练“为什么这里选这个类，不选那个”

## 2. 为什么复制图片通常会想到字节流

答题主线：

- 图片属于二进制内容
- 字节流更适合处理二进制数据

可以直接这样答：

复制图片时通常更自然想到字节流，比如 `FileInputStream + FileOutputStream`，因为图片不是按字符来处理的文本内容，而是原始二进制数据。这个场景下如果去讲字符流，方向一般就偏了。

## 3. 为什么读文本通常会想到字符流

答题主线：

- 文本更适合按字符来处理
- 字符流更贴近文本语义

可以这样答：

如果是读普通文本文件，通常会优先想到字符流，比如 `FileReader`，因为文本的核心不是按原始字节解释，而是按字符内容处理。尤其当题目强调“读文本”而不是“复制文件”时，字符流会更自然。

## 4. 为什么很多时候又会进一步想到 `BufferedReader`

答题主线：

- 不只是能读
- 还要更顺手、更高效

可以这样答：

如果是按行读文本，很多时候会直接想到 `BufferedReader`，因为它除了有缓冲提升效率以外，还有 `readLine()` 这种很高频、很顺手的方法。所以实际文本处理里，`BufferedReader` 往往比单独 `FileReader` 更常被提到。

## 5. `FileReader` 和 `InputStreamReader` 怎么选

这也是 IO 场景题里很高频的一问。

一个比较稳的回答是：

- 如果只是最基础地读本地文本文件，可以先想到 `FileReader`
- 如果题目涉及编码、字节转字符、或者希望显式控制字符集，那更自然的是 `InputStreamReader`

所以你可以直接记：

- `FileReader` 更像简单文本读取
- `InputStreamReader` 更像字节到字符的桥接，并且常和编码控制一起讲

## 6. 为什么有时 `InputStreamReader` 比 `FileReader` 更自然

答题主线：

- 它不只是读字符
- 它还能把字节流转成字符流
- 并且更容易显式处理编码问题

可以这样答：

如果题目里出现了编码转换、网络字节流、或者明确要求指定 `UTF-8`，那 `InputStreamReader` 往往会比 `FileReader` 更自然，因为它更像是在字节流和字符流之间搭桥，而不是只做一个简单本地文本读取。

## 7. 为什么复制文本文件有时会提字节流，有时会提字符流

这是个很容易混的点。

可以先记一个口径：

- 如果题目强调“文件复制”，字节流通常是通用答案
- 如果题目强调“文本处理”，字符流通常更自然

也就是说，关键不是死记：

- 文本一定只能字符流

而是先看题目到底在强调：

- 文件复制
- 还是文本内容处理

## 8. 为什么 IO 代码总强调关闭资源

答题主线：

- IO 会占用系统资源
- 不关可能导致资源泄露

可以这样答：

IO 代码总强调关闭资源，是因为文件句柄、流对象这类底层资源不是无限的，不及时关闭可能导致资源泄露、文件占用、后续操作异常。所以在 IO 场景里，`try-with-resources` 也会特别常见。

## 9. 为什么 `try-with-resources` 在 IO 里很常见

答题主线：

- 简洁
- 不容易漏关

可以这样答：

`try-with-resources` 在 IO 里很常见，因为 IO 代码几乎总要处理资源关闭问题。相比手写 `finally`，它更简洁，也更不容易漏掉关闭逻辑，所以现代 Java 代码里通常更推荐这样写。

## 10. 看到对象保存到文件，应该想到什么

答题主线：

- 不是普通文本写出
- 而是对象序列化

可以这样答：

如果题目说要把对象直接写到文件里，再读回来，最自然应该想到对象流，也就是 `ObjectOutputStream` 和 `ObjectInputStream`，并顺手想到序列化、反序列化以及 `Serializable`。

## 11. 当前阶段最值得先记的几组场景

1. 复制图片 -> 字节流
2. 读文本 -> 字符流
3. 按行读文本 -> `BufferedReader`
4. 涉及编码 -> `InputStreamReader` / `OutputStreamWriter`
5. 写对象 -> 对象流
6. 强调资源关闭 -> `try-with-resources`

## 12. 一句话总结

IO 场景题的核心，不是把类名背得越多越好，而是看到场景以后，能立刻判断：这是二进制处理、文本处理、编码转换、对象序列化，还是资源管理问题。
