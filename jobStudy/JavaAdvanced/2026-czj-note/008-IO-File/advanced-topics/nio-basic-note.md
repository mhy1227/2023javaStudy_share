# NIO 入门笔记

## 1. 为什么这一份要单独补

前面的 IO 主干更多还是围绕：

- `InputStream`
- `OutputStream`
- `Reader`
- `Writer`

这些传统 IO 类来展开。

但面试里很容易继续追：

- 什么是 `NIO`
- `BIO` 和 `NIO` 有什么区别
- `Buffer`、`Channel`、`Path`、`Files` 是什么

所以这一份主要解决的是：

- 当前阶段对 `NIO` 到底该掌握到什么程度

## 2. 先一句话理解什么是 `NIO`

可以先记一个比较稳的口径：

- `NIO` 是 Java 提供的一套“更偏现代化”的 IO 处理方式，核心常会和缓冲区、通道、非阻塞这些词连在一起

更直白一点说：

- 前面传统 IO 更像“用流读写”
- `NIO` 更像“围绕缓冲区和通道组织数据”

## 3. `BIO` 和 `NIO` 怎么粗分

如果只是基础面试，先记住下面这条主线就够了：

- `BIO` 更常和传统阻塞式流模型放在一起讲
- `NIO` 更常和 `Buffer`、`Channel`、非阻塞思路放在一起讲

也就是说，面试里如果问：

- `BIO` 和 `NIO` 的区别

你至少要先能说出：

1. 组织方式不一样
2. 常见类不一样
3. `NIO` 更容易和非阻塞模型连起来

## 4. 什么是 `Buffer`

`Buffer` 可以先理解成：

- 一块用来暂存数据的缓冲区

在 `NIO` 里，它很重要，因为很多读写过程不是直接“从 A 到 B”，而是：

- 先把数据放进缓冲区
- 再从缓冲区取出来

你当前阶段不一定要把所有子类都背全，但最好知道常见的：

- `ByteBuffer`

## 5. 什么是 `Channel`

`Channel` 可以先理解成：

- 数据传输通道

你可以先粗记：

- 流更像单向读或写
- 通道更像数据传输的通路

常见会被提到的有：

- `FileChannel`

它常和文件读写放在一起讲。

## 6. `Buffer` 和 `Channel` 的关系怎么理解

这是 `NIO` 入门里最重要的主线之一。

可以先记一个很实用的说法：

- `Channel` 负责传输
- `Buffer` 负责暂存

也就是说：

- 数据通常通过 `Channel` 进入 `Buffer`
- 再从 `Buffer` 中取出来处理

面试里如果你能把这句话说清楚，已经够用了。

## 7. 什么是 `Path`

如果前面 `File` 这一章已经学过了，你可以把 `Path` 理解成：

- 更现代一点的路径表示方式

也就是说：

- 以前很多文件路径相关操作更容易想到 `File`
- `NIO` 里则常会想到 `Path`

## 8. 什么是 `Files`

`Files` 可以先理解成：

- 围绕 `Path` 提供的一组文件操作工具方法

你可以粗记它更像：

- 一个工具类入口

常见场景比如：

- 判断文件是否存在
- 读文件内容
- 写文件内容
- 复制文件

## 9. 面试里怎么回答 `BIO` 和 `NIO` 的区别

可以直接背一版：

传统 `BIO` 更常围绕流模型来讲，比如 `InputStream`、`OutputStream`、`Reader`、`Writer` 这些类，很多场景下是阻塞式处理；`NIO` 则更常围绕 `Buffer`、`Channel`、`Path`、`Files` 这些类来组织 IO，尤其 `Buffer` 和 `Channel` 是它很核心的主线。另外 `NIO` 也更容易和非阻塞、多路复用这些更进阶的话题联系起来。

## 10. 当前阶段最值得先记哪些类

如果你现在不想一下子学太散，可以优先先记：

1. `ByteBuffer`
2. `FileChannel`
3. `Path`
4. `Files`

这几个已经够搭起一个基础 `NIO` 认知框架。

## 11. 当前阶段不要答太满的点

`NIO` 很容易一追就追到：

- Selector
- 多路复用
- Netty

但如果你当前只是 Java 基础复习，不建议一上来把自己带得太深。

先答稳这些更重要：

1. `BIO` 和 `NIO` 的粗区别
2. `Buffer` 和 `Channel` 的关系
3. `Path` 和 `Files` 是做什么的

## 12. 当前阶段记忆重点

这一份最少要记住：

1. `NIO` 和传统 IO 的组织方式不一样
2. `Buffer` 是缓冲区
3. `Channel` 是传输通道
4. `Buffer + Channel` 是 `NIO` 的主线
5. `Path` 是路径表示
6. `Files` 是围绕 `Path` 的工具方法入口
7. 面试里能先把 `BIO / NIO`、`Buffer / Channel` 讲清楚就够用了
