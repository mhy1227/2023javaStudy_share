# MySQL 索引下推 ICP 笔记

## 1. 这题为什么值得单独记

索引下推不是最基础的第一层八股，但在牛客和 LeetCode 这类面经里出现频率并不低。

它通常和这些题一起出现：

- 回表
- 覆盖索引
- 联合索引
- 最左前缀
- EXPLAIN 优化

所以它很适合当“索引进阶追问题”。

## 2. 什么是索引下推

可以先这样理解：

> 索引下推就是把一部分原本要到 Server 层再判断的过滤条件，尽量下推到存储引擎层，在索引扫描阶段就先做过滤。

一句话记忆：

> **能在更早阶段过滤掉的数据，就尽量不要先拿上来再过滤。**

## 3. 它想解决什么问题

它主要想减少两类成本：

- 不必要的回表
- 不必要的记录上送

更口语一点说：

> 原本要先查出一堆候选记录，再一条条拿上来判断；现在是尽量在扫描索引时就先筛掉一部分，这样后面要处理的数据就更少。

## 4. 它经常和谁一起问

最常见的连环问法是：

- 什么是回表
- 什么是覆盖索引
- 什么是索引下推
- 覆盖索引和索引下推有什么区别

这里最容易答混。

## 5. 覆盖索引和索引下推的区别

更稳的回答：

> 覆盖索引强调的是“查询需要的字段都能直接从索引里拿到，所以不需要回表”；索引下推强调的是“在索引扫描阶段先把部分条件用掉，尽量更早过滤数据”。前者更关注要不要回表，后者更关注能不能更早过滤。

一句话记忆：

- 覆盖索引：少一次回表
- 索引下推：少一些无效扫描和无效上送

## 6. 什么时候更容易聊到它

通常是这种场景：

- 联合索引里，前面列能定位一部分范围
- 后面列虽然不能继续完整决定索引路径
- 但还能拿来先做过滤

面试里你不用把执行细节讲得特别底层，但要表达出：

> 它不是改变索引结构，而是优化索引扫描过程中的过滤时机。

## 7. 这题最适合举什么例子

比如联合索引是 `(name, age)`，查询像这样：

```sql
select * from user
where name like '张%'
  and age = 18;
```

更容易讲的点是：

- `name` 这列先决定扫描范围
- `age` 这列虽然不一定继续完整决定索引路径
- 但它仍然可以在索引扫描阶段帮助提前过滤

也就是说：

> 不一定要等把候选记录全拿上来之后再判断 `age = 18`，而是可以尽量在更早阶段就先筛掉一部分。

## 8. EXPLAIN 里通常怎么看到它

面试里如果继续追到执行计划，可以先记这个方向：

> 索引下推常常会和 `Extra` 里的 `Using index condition` 这类信息联系起来。

你不一定非要把所有执行计划字段背全，但最好知道：

- 它通常是执行计划层面的优化线索
- 它不等于“形成覆盖索引”
- 它出现时，往往说明过滤动作提前了

## 9. 它和覆盖索引谁更好

不能简单说谁“更高级”。

更稳的理解是：

- 覆盖索引更理想，因为它直接避免回表
- 索引下推是在“还做不到完全覆盖”时，尽量把过滤做早

所以这类题更像：

> 覆盖索引是更理想路径，索引下推是进一步减少无效处理的优化手段。

## 10. 面试里最容易被继续追什么

常见追问包括：

- 索引下推和最左前缀有什么关系
- 索引下推和回表有什么关系
- `Using index` 和 `Using index condition` 有什么区别
- 覆盖索引和索引下推能不能同时出现

这里最稳的回答方式不是拼细枝末节，而是始终抓住两点：

1. 它优化的是过滤时机
2. 它和覆盖索引不是一回事

## 11. 面试里可以怎么答

### 短答版

> 索引下推可以理解成把部分过滤条件下推到存储引擎层，在扫描索引时就先过滤掉不满足条件的记录，减少回表和无效数据上送。

### 稍完整版

> 我理解索引下推的核心价值，不是“换了一种索引”，而是把过滤动作做得更早。原本有些条件可能要到 Server 层再判断，现在如果这些条件能在存储引擎扫描索引时就先判断，就能提前过滤掉一批不需要的数据。它经常和联合索引、回表、覆盖索引放在一起问，因为本质上都在优化查询路径和减少额外成本。

## 12. 一版适合项目表达的回答

> 如果从项目排查角度讲，我会把索引下推理解成“在更早阶段做过滤”的优化。很多 SQL 不是有没有索引这么简单，而是即使走了索引，仍然可能拿出很多无效候选数据。如果能在存储引擎扫描索引时就先用掉部分条件，就能减少后续回表和上送成本。所以它经常和联合索引、回表、覆盖索引、EXPLAIN 一起出现。

## 13. 最容易答错的点

- 把索引下推说成“建立一种新的索引”
- 把索引下推和覆盖索引混成一回事
- 只会背名词，说不清它优化的是“过滤时机”

## 14. 一句话收尾

> 覆盖索引解决的是“要不要回表”，索引下推优化的是“能不能更早过滤”。它们都属于索引优化，但关注点不一样。
