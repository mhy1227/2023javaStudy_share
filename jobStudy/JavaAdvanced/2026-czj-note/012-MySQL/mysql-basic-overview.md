# MySQL 基础总览

## 1. 这份文档是干什么的

这份文档是 `012-MySQL` 章节的正式基础入口。

目标不是一上来把所有细节都讲完，而是先把 MySQL 最核心的复习主线搭起来，让你知道：

1. MySQL 是什么。
2. 为什么 Java 后端面试总会问它。
3. MySQL 这一章后续主要学什么。
4. 每条主线大概解决什么问题。

如果这份文档先读顺，后面的索引、事务、锁、日志、慢 SQL 排查都会更好接。

## 2. MySQL 是什么

可以先记住这个版本：

> MySQL 是一个关系型数据库，主要用来存储业务核心数据，支持表结构管理、SQL 查询、事务、一致性控制和复杂条件检索。

更口语一点可以说：

> 在 Java 后端项目里，MySQL 通常是核心持久化存储，像用户、订单、课程、试卷、答题记录、支付记录这类数据，最后一般都要落到 MySQL。

## 3. 为什么面试里 MySQL 高频

原因很现实：

- 几乎所有后端项目都会接数据库。
- 面试官很容易通过 MySQL 问出你对系统运行机制的理解深度。
- MySQL 既有基础题，也有性能题、场景题、排查题。

常见追问路径通常是这样的：

1. 你项目里用什么数据库？
2. 为什么用 MySQL？
3. 索引怎么理解？
4. 事务和隔离级别怎么理解？
5. 慢 SQL 怎么排查？
6. 锁和 MVCC 怎么理解？
7. redo log、undo log、binlog 分别做什么？

所以 MySQL 不是一个“只会增删改查就够了”的模块，它其实是后端面试里特别典型的纵深题。

## 4. MySQL 和 Redis 的分工是什么

这个问题也很常见。

可以先记这个回答：

> MySQL 更适合做持久化存储、事务控制和复杂查询，Redis 更适合做缓存、热点数据访问和高并发场景优化。实际项目里一般不是二选一，而是 MySQL 负责落库，Redis 负责加速。

所以不要把 MySQL 和 Redis 对立起来，它们更多是配合关系。

## 5. 学 MySQL 主要学哪几条主线

### 5.1 索引

索引这条线解决的是“为什么查询快”这个问题。

这里后续会重点学：

- 什么是索引。
- 为什么 B+ 树适合做索引。
- 主键索引、普通索引、唯一索引、联合索引。
- 最左前缀原则。
- 回表、覆盖索引、索引失效。

### 5.2 事务

事务这条线解决的是“多条操作怎么保证整体一致”这个问题。

这里后续会重点学：

- 什么是事务。
- ACID 四大特性。
- 四种隔离级别。
- 脏读、不可重复读、幻读。
- MySQL 默认隔离级别。

### 5.3 锁与 MVCC

这条线解决的是“并发访问下怎么保证正确性与性能”的问题。

这里后续会重点学：

- 行锁、表锁、间隙锁、临键锁。
- MVCC 是什么。
- 锁和 MVCC 的关系。
- 当前读和快照读。

### 5.4 日志

这条线解决的是“怎么保证崩溃恢复、回滚和复制”的问题。

这里后续会重点学：

- redo log
- undo log
- binlog
- 两阶段提交

### 5.5 SQL 执行与调优

这条线解决的是“SQL 到底怎么执行，以及慢在哪里”的问题。

这里后续会重点学：

- 一条 SQL 的执行流程。
- `explain` 怎么看。
- 慢 SQL 怎么排查。
- 哪些 SQL 写法容易出性能问题。

## 6. 初学阶段最值得先记的结论

你可以先背这五句：

1. **MySQL 是 Java 后端项目里最常见的核心关系型数据库。**
2. **索引解决的是查询效率问题。**
3. **事务解决的是一组操作的一致性问题。**
4. **锁和 MVCC 解决的是并发访问下的数据正确性问题。**
5. **日志和执行流程决定了 MySQL 的恢复能力、复制能力和排障能力。**
