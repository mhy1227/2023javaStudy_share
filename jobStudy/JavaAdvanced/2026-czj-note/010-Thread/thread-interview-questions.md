# 线程面试题清单

## 1. 这一份是做什么的

这一份单独只放问题，不展开答案。

适合用法：

- 快速过题
- 自测能不能口述出来
- 面试前最后一轮扫题

如果你想同时看标答，可以配合本目录下另一份：

- `thread-interview-answers.md`

## 2. 第一梯队：最常被直接问

1. 进程和线程有什么区别？
2. 什么是多线程？为什么要有多线程？
3. 并发和并行有什么区别？
4. 创建线程有哪几种方式？
5. `run()` 和 `start()` 有什么区别？
6. `Runnable` 和 `Callable` 有什么区别？
7. 线程有哪些状态？
8. 什么是线程安全？
9. 为什么 `i++` 在线程下不安全？
10. 什么是原子性、可见性、有序性？

## 3. 第二梯队：很容易连着追问

1. `sleep()` 和 `wait()` 有什么区别？
2. `wait()` 为什么必须在同步代码块中使用？
3. `notify()` 和 `notifyAll()` 有什么区别？
4. `sleep()` 会不会释放锁？
5. `wait()` 会不会释放锁？
6. `join()` 是做什么的？
7. `RUNNABLE` 状态是不是就表示线程一定正在运行？
8. 什么是上下文切换？

## 4. 第三梯队：同步手段高频题

1. 什么是 `synchronized`？
2. `synchronized` 有哪几种常见写法？
3. `synchronized` 锁的是什么？
4. `synchronized(this)` 和同步实例方法有什么关系？
5. 同步静态方法锁的是什么？
6. 为什么 `synchronized(new Object())` 可能起不到你想要的同步效果？
7. `synchronized` 主要解决什么问题？
8. 什么是 `volatile`？
9. `volatile` 能解决什么问题？
10. 为什么 `volatile` 不能保证 `i++` 线程安全？

## 5. 第四梯队：Lock 高频题

1. 为什么 Java 还有 `Lock`？
2. 什么是 `ReentrantLock`？
3. 什么是可重入？
4. `Lock` 和 `synchronized` 有什么区别？
5. 为什么 `unlock()` 一般要放到 `finally` 中？
6. `tryLock()` 和 `lock()` 有什么区别？
7. 什么是公平锁和非公平锁？

## 6. 第五梯队：线程池高频题

1. 为什么不推荐频繁 `new Thread()`？
2. 什么是线程池？
3. 为什么一般更推荐使用线程池？
4. `execute()` 和 `submit()` 有什么区别？
5. `ThreadPoolExecutor` 核心参数有哪些？
6. 线程池处理任务的大致流程是什么？
7. 常见拒绝策略有哪些？
8. 为什么不建议无脑使用 `Executors` 默认工厂？

## 7. 第六梯队：并发容器高频题

1. 为什么普通集合在并发环境下容易出问题？
2. 什么是并发容器？
3. `ConcurrentHashMap` 为什么比 `Hashtable` 更常用？
4. `CopyOnWriteArrayList` 适合什么场景？
5. `BlockingQueue` 有什么用？
6. 并发容器是不是就能解决所有并发问题？

## 8. 第七梯队：面试延伸题

1. 什么是死锁？怎么避免？
2. 线程池参数怎么结合业务去理解？
3. 线程安全是不是就等于加锁？
4. 项目里你一般在哪些地方会用线程池？
5. 项目里你一般在哪些地方会用并发容器？
