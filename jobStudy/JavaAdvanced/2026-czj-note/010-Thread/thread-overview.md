# 线程总览

## 1. 为什么线程是面试高频

因为线程和并发是 Java 面试里非常容易拉开差距的一章。

前面的字符串、集合、异常、IO、泛型，更多是基础能力；
线程往后会继续连到：

- 锁
- 线程池
- 并发容器
- JVM
- 项目里的高并发场景

所以很多面试官会先问基础，再慢慢往线程和并发上追。

## 2. 这一章到底要解决什么问题

学习线程，本质上是在回答这几个问题：

1. 为什么要有线程？
2. 多个线程一起跑时，会出什么问题？
3. Java 提供了哪些机制来控制这些问题？
4. 实际开发里为什么不建议乱开线程？

## 3. 先把几个最基础的概念分开

### 3.1 进程

进程可以理解成一个正在运行的程序实例。

例如：

- 打开一个 IDEA
- 打开一个微信
- 启动一个 Java 程序

这些都可以看成进程。

### 3.2 线程

线程是进程中的执行单元。

一个进程里可以有多个线程，它们共享进程里的大部分资源，比如：

- 堆内存
- 方法区中的类信息
- 打开的文件句柄等部分资源

但每个线程又有自己相对独立的部分，比如：

- 程序计数器
- 虚拟机栈
- 本地方法栈

### 3.3 并发

并发强调的是“同一段时间内，有多个任务都在推进”。

### 3.4 并行

并行强调的是“同一时刻，多个任务真的同时执行”。

面试里常见说法：

- 单核 CPU 更偏并发
- 多核 CPU 才更容易出现真正的并行

## 4. 为什么多线程会带来问题

单线程程序相对简单，执行顺序比较稳定。

多线程一进来，问题就出现了：

- 抢占执行顺序不可控
- 多个线程可能同时修改共享变量
- 一个线程改了值，另一个线程不一定立刻看得到

所以并发问题通常可以先粗分为三类：

1. 原子性问题
2. 可见性问题
3. 有序性问题

后面的 `synchronized`、`volatile`、`Lock`，本质上都是在解决这些问题。

## 5. 线程这一章建议怎么学

建议顺序不要一上来就扎进 JUC，而是先走主线：

1. 先分清进程、线程、并发、并行
2. 再看线程创建方式和生命周期
3. 再看线程安全问题从哪里来
4. 再学 `synchronized`、`volatile`、`Lock`
5. 最后补线程池和并发容器

## 6. 面试里最容易问到的第一批问题

1. 进程和线程有什么区别？
2. 创建线程有哪几种方式？
3. `run()` 和 `start()` 有什么区别？
4. 线程有哪些状态？
5. `sleep()` 和 `wait()` 有什么区别？
6. 什么是线程安全？
7. `synchronized` 和 `volatile` 有什么区别？
8. 为什么一般更推荐线程池？

## 7. 当前阶段先记住什么

如果你现在只是刚开启这一章，先记这几条：

1. 线程是进程中的执行单元
2. 多线程的核心难点是共享资源带来的线程安全问题
3. 并发问题往往离不开原子性、可见性、有序性
4. Java 并发这章不能只背 API，要始终围绕“问题和解决手段”去理解

## 8. 一句话总结

线程这一章真正要学的，不是“怎么开一个线程”，而是“多个线程一起工作时为什么会乱，以及 Java 怎么把这种乱控制住”。
