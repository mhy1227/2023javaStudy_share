# AQS 入门说明

## 1. 为什么这一份要单独补

如果线程这章只停留在：

- `synchronized`
- `volatile`
- `ReentrantLock`
- 并发工具类

那基础是够的。

但如果面试往下一层追，经常会出现：

- 你知道 AQS 吗？
- `ReentrantLock` 底层大概基于什么思想？
- 为什么 `CountDownLatch`、`Semaphore` 这些并发工具也能统一实现？

所以这一份不是源码细抠版，而是：

- 当前阶段够用的 AQS 入门版

## 2. AQS 是什么

AQS 的全称是：

- AbstractQueuedSynchronizer

可以先粗理解成：

- Java 并发包里一个很重要的同步器框架

更直白一点说：

- 它像一个“搭并发锁和同步工具的底座”

## 3. 为什么需要 AQS

你可以想一个问题：

- 锁要管获取和释放
- 等待线程要排队
- 没拿到资源的线程要等待
- 拿到资源后又要唤醒后续线程

这些逻辑很多同步工具都会反复遇到。

所以就会有一个很自然的想法：

- 能不能把这些通用能力抽出来，做成一个基础框架

AQS 基本就是干这个的。

## 4. AQS 解决的核心问题

从功能上粗看，它主要在做这些事：

1. 维护一个同步状态
2. 管理等待线程队列
3. 控制线程获取和释放资源
4. 在合适的时候挂起和唤醒线程

你当前阶段不需要一下子钻到源码里，但至少要知道：

- AQS 不只是“一个类名”，它是很多并发工具背后的通用框架

## 5. 怎么理解“状态 + 队列”

这是 AQS 最值得先记住的主线。

可以粗记成：

- 一个 `state`
- 一条等待队列

其中：

- `state` 可以理解成同步状态
- 队列里放的是没抢到资源、需要等待的线程

例如：

- 对锁来说，`state` 可以理解成当前锁的占用情况
- 对倒计时器来说，`state` 也可以理解成剩余计数

## 6. 为什么 `ReentrantLock` 常和 AQS 绑定讲

因为 `ReentrantLock` 并不是凭空自己把所有同步逻辑从零写一遍。

它很多底层同步控制能力，都是建立在 AQS 这一套框架思路之上的。

面试里你不一定要展开源码，但最好知道：

- `ReentrantLock` 和 AQS 关系非常紧

## 7. 为什么 `CountDownLatch`、`Semaphore` 也会提到 AQS

因为它们虽然表面上功能不同，但底层都存在一些共通问题：

- 怎么表示当前状态
- 怎么让线程等待
- 怎么在条件满足后唤醒线程

所以它们也常会借助 AQS 这一套同步框架思路来实现。

这也是为什么很多人会说：

- AQS 是 JUC 里很多同步器的基础骨架

## 8. 独占模式和共享模式

这是 AQS 里很高频的两个词。

可以先粗理解成：

- 独占：同一时刻更像只有一个线程能拿到资源
- 共享：同一时刻可以允许多个线程按规则共享资源

举例时可以简单说：

- `ReentrantLock` 更偏独占
- `Semaphore`、`CountDownLatch` 常会让你联想到共享式控制

当前阶段知道这个层面就够了。

## 9. 公平和非公平为什么也会扯到这里

因为线程既然要排队，就会涉及一个问题：

- 新来的线程能不能插队

所以你在讲 `ReentrantLock` 公平锁和非公平锁时，也会间接碰到 AQS 里的队列调度思想。

粗理解就行：

- 公平：更强调按排队顺序来
- 非公平：更强调吞吐，不一定严格按先来后到

## 10. AQS 和 `synchronized` 的关系怎么讲

这个问题也容易被问。

比较稳的回答方式是：

- `synchronized` 是 JVM 层面的内置同步机制
- AQS 更像 JUC 包里很多显式同步器的基础框架

所以它们都和并发同步有关，但路线不一样。

也就是说：

- 不是所有锁都基于 AQS

## 11. 面试里怎么回答“什么是 AQS”

可以直接背一版：

AQS 可以理解成 Java 并发包里一个很重要的同步器框架，很多锁和并发工具类都会基于它来实现。它的核心思路可以粗理解成“状态 + 等待队列”，也就是用一个同步状态来表示资源情况，再把没拿到资源的线程放进队列里管理，后续再做唤醒和调度。当前阶段我会把它理解成并发组件的基础骨架，而不是只记一个类名。

## 12. 面试里怎么回答“为什么需要 AQS”

可以背一版短的：

因为很多同步工具都会遇到资源获取、释放、线程排队、阻塞和唤醒这些共性问题，如果每个都从零实现一遍，重复很多。AQS 相当于把这些共通能力抽成一个基础框架，让上层同步器在这个框架上实现自己的规则。

## 13. 当前阶段不用死抠的部分

你现在不一定要马上深入：

- CLH 队列细节

- `acquire`、`release` 全部源码流程

因为当前阶段更重要的是先建立：

- AQS 是什么

- 哪些常见工具和它有关

## 14. 当前阶段记忆重点

这一份最少要记住：

1. AQS 是同步器框架
2. 很多 JUC 工具和它有关
3. 核心主线是“状态 + 等待队列”
4. `ReentrantLock` 常和 AQS 一起讲
5. `synchronized` 不是走 AQS 这条路
6. 当前阶段先建立整体理解，不用一下子钻源码
