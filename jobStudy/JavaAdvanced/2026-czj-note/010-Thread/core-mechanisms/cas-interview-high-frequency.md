# CAS 高频面试题

## 1. 这一份怎么用

这一份不是完整知识说明，而是把 CAS 这一条追问链里最值得反复背的问题单独拎出来。

适合用法：

- 面试前最后快速过题
- 模拟面试时抽问自己
- 配合 `cas-note.md` 和原子类问答一起复习

建议答题顺序：

- 先说结论
- 再补关键词
- 最后补场景或边界

## 2. 第一梯队：几乎必问

- 什么是 CAS？
- 为什么 CAS 常被归到乐观锁思路？
- `AtomicInteger` 为什么线程安全？
- CAS 有什么问题？
- 什么是 ABA？
- 怎么解决 ABA？

## 3. 第二梯队：很容易继续追问

- CAS 的核心执行流程是什么？
- CAS 为什么不一定要加锁？
- CAS 为什么会有自旋开销？
- `AtomicStampedReference` 是干什么的？
- 为什么单变量原子更新不等于整个业务线程安全？
- CAS 适合什么场景？不适合什么场景？

## 4. 第三梯队：经常串到原子类和计数器

- 原子类和 CAS 是什么关系？
- `AtomicInteger` / `AtomicLong` 适合什么场景？
- 为什么高并发计数常提 `LongAdder`？
- `AtomicLong` 和 `LongAdder` 怎么选？
- 为什么 `LongAdder` 不等于完全替代原子类？

## 5. ABA 高频题

- 什么是 ABA 问题？
- ABA 为什么会出现？
- 为什么值看起来没变，逻辑上却可能已经出问题？
- `AtomicStampedReference` 怎么处理 ABA？
- 版本号思路为什么能解决 ABA？

## 6. CAS 和锁的关系题

- CAS 和锁是什么关系？
- CAS 为什么不能完全替代锁？
- 乐观锁和悲观锁应该怎么选？
- 什么场景会优先考虑 CAS？
- 什么场景还是应该老老实实用锁？

## 7. 容易答偏的点

- 不要把 CAS 说成“完全没有成本”
- 不要把 CAS 说成“绝对比锁高级”
- 不要把 CAS 说成能解决所有线程安全问题
- 不要把单次原子更新误答成整个业务流程都安全
- 不要把 `LongAdder` 说成全面优于 `AtomicLong`

## 8. 一组常见追问链

面试里很常见的一条链路是：

1. 什么是乐观锁和悲观锁？
2. CAS 为什么更偏乐观锁？
3. `AtomicInteger` 为什么线程安全？
4. CAS 有什么问题？
5. 什么是 ABA？
6. 怎么解决 ABA？
7. 为什么高并发计数又会提到 `LongAdder`？

如果这条链你能答顺，CAS 这一块基本就比较稳。

- 为什么 CAS 更偏乐观锁？

- 什么是 ABA？为什么它是个问题？
- `AtomicStampedReference` 怎么解决 ABA？
- 为什么 CAS 不能解决所有线程安全问题？
