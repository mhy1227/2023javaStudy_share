# 顺序打印练习题

## 1. 这一份练什么

这一题主要练：

- 线程协作
- 等待与唤醒
- 锁对象一致性
- 状态控制

它属于锁练习里最常见、最基础的一类题。

## 2. 常见题型

顺序打印题最常见的通常有三种：

1. 两线程交替打印奇偶数
2. 三线程循环打印 `ABCABCABC`
3. 三个方法 `first`、`second`、`third` 固定顺序执行

虽然题面不一样，但核心都在练：

- 谁该执行
- 谁该等待
- 谁来唤醒下一个

## 3. 题 1：两线程交替打印

目标：

- 线程 A 打印奇数
- 线程 B 打印偶数
- 输出顺序正确

考察点：

- 共享状态控制
- `wait/notify` 或 `Condition` 配合
- 为什么判断条件通常放在 `while` 中

建议练法：

- 第一版：`synchronized + wait/notifyAll`
- 第二版：`ReentrantLock + Condition`

## 4. 题 2：三线程循环打印

目标：

- 线程 A 打印 `A`
- 线程 B 打印 `B`
- 线程 C 打印 `C`
- 最终循环输出 `ABCABCABC`

考察点：

- 多线程轮转控制
- 多条件唤醒
- 状态变量的设计

建议练法：

- 第一版：一个状态变量控制顺序
- 第二版：多个 `Condition` 分组等待

## 5. 题 3：固定顺序执行

目标：

- 三个方法 `first`、`second`、`third`
- 线程调用顺序可能是乱的
- 但执行顺序必须固定

考察点：

- 执行先后依赖
- 线程协作，而不是共享数据更新

建议练法：

- 第一版：`synchronized` + 状态控制
- 第二版：`CountDownLatch`

## 6. 做这类题时最容易犯的错

- `if` 写成条件判断，而不是 `while`
- 唤醒后没有重新判断条件
- 锁对象不一致
- 改状态和唤醒顺序写乱
- 只会写两线程版，三线程一变形就乱

## 7. 这类题真正考什么

表面看是在考打印，实际上更想看你能不能讲清楚：

- 线程为什么要等
- 等待条件是什么
- 什么时候才能继续
- 谁来负责唤醒下一个线程

也就是说，核心不是输出字符，而是：

- 线程协作模型

## 8. 复盘时要问自己的问题

- 我有没有清楚定义共享状态？
- 我有没有保证所有线程看的是同一把锁？
- 我有没有把条件判断写在循环里？
- 我有没有想清楚“先改状态还是先唤醒”？
- 如果线程数从 2 个变成 3 个，我的思路还能不能扩展？

## 9. 面试里怎么描述这类题

如果面试官问你做过什么锁题，可以这样说：

我练过比较典型的顺序打印题，比如两线程交替打印、三线程循环打印、固定顺序执行。这类题的核心不是打印本身，而是线程之间的等待和唤醒机制。我一般会先用 `synchronized + wait/notifyAll` 写一版，再用 `ReentrantLock + Condition` 写一版，重点看状态控制和条件判断是不是清楚。
