# 锁练习题清单

## 1. 这一份怎么用

这一份先不直接给完整答案，而是先把锁相关最值得练的题目集中起来。

适合用法：

- 先自己写
- 写完再自己复盘
- 最后再考虑补答案版或题解版

## 2. 第一梯队：最值得先练

### 题 1：两个线程交替打印

目标：

- 线程 A 打印奇数
- 线程 B 打印偶数
- 最终输出有序结果

建议练法：

- 先用 `synchronized + wait/notify`
- 再用 `ReentrantLock + Condition`

### 题 2：三个线程顺序打印

目标：

- 线程 A 打印 `A`
- 线程 B 打印 `B`
- 线程 C 打印 `C`
- 循环输出 `ABCABCABC`

建议练法：

- 先做状态控制版
- 再做多 `Condition` 版

### 题 3：固定顺序执行

目标：

- 有三个方法 `first`、`second`、`third`
- 三个线程调用顺序不固定
- 但最终执行顺序必须固定

建议练法：

- 先用 `synchronized`
- 再尝试 `CountDownLatch`

## 3. 第二梯队：经典协作题

### 题 4：生产者消费者

目标：

- 一个或多个线程生产数据
- 一个或多个线程消费数据
- 缓冲区满时不能继续生产
- 缓冲区空时不能继续消费

重点观察：

- 条件等待
- 条件唤醒
- 为什么判断条件要放在 `while` 中

### 题 5：有界阻塞队列简化版

目标：

- 自己实现一个简化版阻塞队列
- 支持放入和取出
- 队列满时等待
- 队列空时等待

建议练法：

- 先用 `synchronized + wait/notifyAll`
- 再用 `ReentrantLock + Condition`

## 4. 第三梯队：锁选型练习题

### 题 6：库存扣减怎么保护

目标：

- 多线程同时扣库存
- 保证库存不出现负数

思考重点：

- 为什么这类题通常不能只靠一个简单原子变量就完事
- 什么时候适合锁，什么时候要讲业务层约束

### 题 7：读多写少的缓存访问

目标：

- 大量线程读取缓存
- 少量线程更新缓存

思考重点：

- 为什么普通独占锁不一定最优
- 为什么这里会想到读写锁

### 题 8：计数器怎么选

目标：

- 同样是多线程计数
- 分别思考 `synchronized`
- `AtomicInteger`
- `LongAdder`

思考重点：

- 什么时候该用锁
- 什么时候更适合原子类
- 为什么高并发下又会追到 `LongAdder`

## 5. 做题时高频错误点

- 只用 `if` 判断，不用 `while`
- 锁对象不一致
- 先 `notify` 后逻辑状态没改好
- 唤醒了错误线程
- 把慢操作放到锁里
- 多把锁交叉使用导致死锁
