# 计数器选型练习题

## 1. 这一份练什么

这一题主要练：

- 并发计数场景分析
- 锁和原子类怎么选
- `AtomicInteger` / `AtomicLong` / `LongAdder` 的差异
- 为什么同样是计数，不同场景答案不一样

它是非常典型的面试选型题。

## 2. 题目核心目标

通常题目会给出类似目标：

- 多线程环境下做计数
- 要求结果正确
- 再进一步追问性能和吞吐

然后让你思考：

- `synchronized`
- `AtomicInteger`
- `AtomicLong`
- `LongAdder`

应该怎么选。

## 3. 这类题真正考什么

这类题最核心的通常有四点：

1. 是否只是单变量更新
2. 并发强度有多高
3. 更看重简单直接，还是更看重高并发吞吐
4. 你会不会把不同工具放到合适场景里

它不是要听你背工具名，而是想看：

- 你会不会做取舍

## 4. 为什么计数题不是只有一个答案

因为“计数”虽然听起来简单，但场景差异很大。

有时只是：

- 低并发下简单加一

有时却是：

- 高并发下持续累加统计

这两个场景下，思路就不应该完全一样。

## 5. 当前阶段常见选型主线

可以先记一个很实用的结论：

- 简单同步计数，可以想到 `synchronized`
- 一般原子计数，可以想到 `AtomicInteger` / `AtomicLong`
- 高并发累加统计，更容易追到 `LongAdder`

这个主线基本已经够应对大多数基础面试。

## 6. 什么时候会想到 `synchronized`

如果场景不仅仅是：

- 单纯计数

而是：

- 计数只是临界区里的一部分
- 还有其他逻辑必须一起保护

那这时候就不能只盯着原子类。

你更应该考虑的是：

- 整个临界区保护

## 7. 什么时候会想到 `AtomicInteger` / `AtomicLong`

如果场景就是很纯粹的：

- 一个共享数字做原子更新

这时候原子类很自然。

因为它比直接上锁更轻量，也更贴近题目本身。

所以这类题很适合顺带补一句：

- 它底层很多更新操作和 CAS 思路关系很紧

## 8. 为什么高并发下又会追到 `LongAdder`

因为如果很多线程都去更新同一个原子变量，就会出现：

- 单点竞争明显
- CAS 失败重试变多

这时面试官就很喜欢继续追：

- 那为什么高并发计数更常提 `LongAdder`

你要知道它的核心优势在于：

- 分散竞争

## 9. 这类题最容易犯的错

- 把所有计数题都答成一个方案
- 一看到并发就无脑答 `synchronized`
- 一看到原子类就完全忘了临界区保护
- 把 `LongAdder` 说成全面优于原子类
- 不去区分“简单计数”和“高并发统计”

## 10. 做题时建议先想什么

建议先想这几个问题：

1. 这是单变量更新还是复合业务逻辑
2. 并发强度大不大
3. 更看重正确性表达还是吞吐
4. 这是简单题，还是想把你追到 CAS / `LongAdder`

## 11. 面试里怎么描述这类题

如果面试官问你做过什么锁和原子类结合的选型题，可以这样说：

我练过计数器选型题，这个题很适合把锁、原子类和 `LongAdder` 串起来。如果只是一般并发下的单变量计数，我会先考虑 `AtomicInteger` 或 `AtomicLong`；如果并发特别高、是持续累加统计，我会进一步想到 `LongAdder`；如果计数只是临界区里的一部分，还有其他逻辑必须一起保护，那就不能只盯着原子类，而要回到锁。

## 12. 复盘时要问自己的问题

- 我有没有区分简单计数和复合临界区逻辑？
- 我有没有说清 `AtomicInteger` / `AtomicLong` 适合什么场景？
- 我能不能解释为什么高并发统计会想到 `LongAdder`？
- 我有没有把 `LongAdder` 说得太绝对？
- 如果面试官继续追，我能不能顺势讲到 CAS？
