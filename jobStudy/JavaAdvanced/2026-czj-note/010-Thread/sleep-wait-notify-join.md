# sleep、wait、notify、join 说明

## 1. 为什么这四个方法要单独拎出来

因为这四个方法几乎是线程基础里最容易混的一组。

常见混乱点：

- `sleep()` 和 `wait()` 都像“暂停”，但本质不同
- `notify()` 和 `notifyAll()` 容易只会背名字
- `join()` 看起来简单，但经常答不出它到底在等谁

所以这一份重点是：

1. 分清定义
2. 分清作用
3. 分清是否释放锁
4. 分清使用场景

## 2. `sleep()` 是什么

`sleep()` 是 `Thread` 类的静态方法。

它的作用是：

- 让当前线程休眠一段时间

最常见写法：

```java
Thread.sleep(1000);
```

意思是当前线程暂停大约 1 秒。

## 3. `sleep()` 的特点

最重要的几个点：

1. `sleep()` 属于 `Thread`
2. 调用后线程通常进入 `TIMED_WAITING`
3. `sleep()` 不会释放锁
4. 时间到了之后，线程会回到可运行状态，等待 CPU 再次调度

## 4. `wait()` 是什么

`wait()` 是 `Object` 类的方法。

它的作用是：

- 让当前线程进入等待状态
- 同时释放当前持有的对象监视器锁

最重要的一点是：

- `wait()` 通常必须放在同步代码块或同步方法中使用

## 5. 为什么 `wait()` 要在同步代码块里用

因为 `wait()` 本质上是和对象监视器配套使用的。

你只有先持有这个对象的监视器锁，才有资格调用这个对象的 `wait()`。

如果不在同步块里直接调用，通常会抛出：

- `IllegalMonitorStateException`

## 6. `wait()` 的特点

最重要的几个点：

1. `wait()` 属于 `Object`
2. 调用后线程通常进入 `WAITING` 或 `TIMED_WAITING`
3. `wait()` 会释放锁
4. 通常需要别的线程 `notify()` 或 `notifyAll()` 来唤醒

## 7. `notify()` 是什么

`notify()` 也是 `Object` 类的方法。

它的作用是：

- 唤醒一个在该对象监视器上等待的线程

但要注意：

- 只是唤醒一个
- 不是立刻就执行
- 被唤醒的线程还需要重新竞争锁

## 8. `notifyAll()` 是什么

`notifyAll()` 的作用是：

- 唤醒所有在该对象监视器上等待的线程

之后这些线程会一起去竞争锁，最终谁先拿到锁，谁先继续执行。

## 9. `join()` 是什么

`join()` 是 `Thread` 类的方法。

它的作用是：

- 让当前线程等待目标线程执行结束

最常见场景是主线程等子线程：

```java
Thread t = new Thread(() -> {
    System.out.println("child thread running");
});

t.start();
t.join();
System.out.println("main thread continue");
```

这里的意思不是让 `t` 等主线程，而是：

- 当前调用 `join()` 的线程去等 `t`

## 10. `sleep()` 和 `wait()` 的区别

这是高频必问题。

建议按这条线回答：

1. `sleep()` 是 `Thread` 的静态方法，`wait()` 是 `Object` 的方法
2. `sleep()` 不要求在同步代码块里，`wait()` 通常必须在同步代码块里
3. `sleep()` 不会释放锁，`wait()` 会释放锁
4. `sleep()` 时间到了会自动恢复，`wait()` 通常需要被其他线程唤醒

## 11. `notify()` 和 `notifyAll()` 的区别

回答主线：

1. `notify()` 只随机唤醒一个等待线程
2. `notifyAll()` 会唤醒所有等待线程
3. 被唤醒线程都不是立刻执行，而是先去重新竞争锁

## 12. `join()` 和 `wait()` 的区别

虽然都带“等待”效果，但它们关注点不同。

### 12.1 `join()`

更像是：

- 线程 A 等线程 B 执行完

### 12.2 `wait()`

更像是：

- 线程在某个对象监视器上等待通知

所以：

- `join()` 更偏线程之间的执行顺序控制
- `wait()` 更偏线程协作和通信

## 13. 这几个方法和锁释放的关系

这一点非常关键，最好单独记：

1. `sleep()` 不释放锁
2. `wait()` 释放锁
3. `notify()` / `notifyAll()` 本身不释放当前锁，通常是同步块执行完才释放
4. `join()` 本质上是等待目标线程结束，面试里一般重点不是它的锁释放细节，而是它的等待语义

## 14. 常见错误理解

### 14.1 认为 `sleep()` 也会释放锁

这是错的。

线程睡了，但如果它在同步块里，锁还是它的。

### 14.2 认为 `notify()` 后对方马上执行

这也不准确。

被唤醒线程还要重新竞争锁，不一定立刻就能继续。

### 14.3 认为 `join()` 是“让子线程等主线程”

这也是常见误解。

真正的理解应该是：

- 谁调用 `join()`，谁去等待

## 15. 一个简单对比表

| 方法 | 所属类 | 是否要求同步块 | 是否释放锁 | 主要作用 |
| --- | --- | --- | --- | --- |
| `sleep()` | `Thread` | 否 | 否 | 让当前线程休眠一段时间 |
| `wait()` | `Object` | 是 | 是 | 让线程等待并释放锁 |
| `notify()` | `Object` | 是 | 否 | 唤醒一个等待线程 |
| `notifyAll()` | `Object` | 是 | 否 | 唤醒所有等待线程 |
| `join()` | `Thread` | 否 | 一般面试不作为重点 | 让当前线程等待目标线程结束 |

## 16. 面试里怎么回答更稳

如果面试官问：

- `sleep()` 和 `wait()` 区别是什么？

可以这样答：

`sleep()` 是 `Thread` 类的静态方法，用来让当前线程休眠一段时间，不会释放锁；`wait()` 是 `Object` 类的方法，通常必须在同步代码块中调用，它会让当前线程进入等待状态并释放当前对象锁，通常需要 `notify()` 或 `notifyAll()` 来唤醒。

如果面试官继续问：

- `notify()` 和 `notifyAll()` 区别是什么？

可以补：

`notify()` 只会唤醒一个等待线程，`notifyAll()` 会唤醒所有等待线程，但它们被唤醒后都还要重新竞争锁，不是立刻执行。

## 17. 当前阶段最该记住的结论

1. `sleep()` 属于 `Thread`，`wait()` 属于 `Object`
2. `sleep()` 不释放锁，`wait()` 释放锁
3. `wait()` / `notify()` / `notifyAll()` 通常都和同步代码块一起出现
4. `notify()` 不是立刻让对方执行，只是发出唤醒信号
5. `join()` 的核心是“当前线程等待目标线程结束”

## 18. 一句话总结

这四个方法的核心不是死记名字，而是分清三件事：它属于谁、它会不会释放锁、它到底是在“暂停自己”还是“协调别的线程”。
