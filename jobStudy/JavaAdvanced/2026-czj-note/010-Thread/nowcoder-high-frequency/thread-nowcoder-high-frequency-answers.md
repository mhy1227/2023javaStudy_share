# 线程牛客高频面试题示例回复

## 1. 这一份怎么用

这一份不是标准唯一答案，而是偏“面试能说顺”的示例回复。

使用建议：

- 先自己答
- 再对照这一份查漏补缺
- 最后把表述改成你自己的话

## 2. 进程和线程有什么区别？

示例回复：

进程更偏资源分配单位，可以理解成一个正在运行中的程序实例；线程是进程中的执行单元，更偏 CPU 调度单位。进程之间资源相对独立，而同一进程中的多个线程会共享大部分资源，所以线程切换成本通常更低，但也更容易带来线程安全问题。

## 3. 什么是并发？什么是并行？

示例回复：

并发强调的是同一段时间内多个任务都在推进，不一定同一时刻真的同时执行；并行强调的是同一时刻多个任务真正同时执行。面试里一般还会补一句，单核 CPU 更偏并发，多核 CPU 更容易体现并行。

## 4. 创建线程有哪几种方式？

示例回复：

Java 里常见的线程创建方式有：继承 `Thread`、实现 `Runnable`、实现 `Callable` 并配合 `FutureTask`，以及通过线程池提交任务。实际开发里更常用的是线程池，因为它更方便统一管理线程。

## 5. `run()` 和 `start()` 有什么区别？

示例回复：

`run()` 只是线程任务逻辑的方法体，直接调用它本质上还是普通方法调用；`start()` 才是真正启动线程的方法，调用后会创建新的执行路径，再由新线程去执行 `run()` 里的逻辑。

## 6. 线程有哪些状态？

示例回复：

Java 线程常见有 6 种状态：`NEW`、`RUNNABLE`、`BLOCKED`、`WAITING`、`TIMED_WAITING`、`TERMINATED`。其中 `RUNNABLE` 不只是正在运行，也可能是在等待 CPU 时间片；`BLOCKED` 常见于等锁，`WAITING` 和 `TIMED_WAITING` 的区别主要在于是否带超时。

## 7. `sleep()` 和 `wait()` 有什么区别？

示例回复：

`sleep()` 是 `Thread` 类的静态方法，用来让当前线程休眠一段时间，不会释放锁；`wait()` 是 `Object` 类的方法，通常必须在同步代码块中使用，它会让线程进入等待状态并释放当前对象锁，通常需要 `notify()` 或 `notifyAll()` 来唤醒。

## 8. 什么是线程安全？

示例回复：

线程安全指的是在多线程并发访问共享资源时，程序仍然能够得到正确、可预期的结果。线程安全问题通常出现在多个线程同时访问或修改同一份共享数据时，核心往往和原子性、可见性、有序性有关。

## 9. 什么是原子性、可见性、有序性？

示例回复：

- 原子性：一个操作要么全部完成，要么全部不做，中间不能被打断。
- 可见性：一个线程修改了共享变量，其他线程能够及时看到这个变化。
- 有序性：程序执行顺序在并发环境下是否仍然符合预期，是否会受到重排序影响。

## 10. 什么是 `synchronized`？

示例回复：

`synchronized` 是 Java 的内置同步关键字，核心作用是让同一时刻只有一个线程进入被保护的临界区代码，从而帮助解决共享资源并发访问时的线程安全问题。它最直观地解决的是互斥和原子性问题，同时也能保证一定的可见性和有序性。

## 11. `synchronized` 锁的是什么？

示例回复：

要看具体写法。如果是同步实例方法，锁的是当前对象实例 `this`；如果是同步静态方法，锁的是当前类对应的 `Class` 对象；如果是同步代码块，锁的是括号里指定的那个对象。能不能起到同步效果，关键在于多个线程是不是在竞争同一把锁。

## 12. 什么是 `volatile`？

示例回复：

`volatile` 是 Java 提供的关键字，用来修饰共享变量，主要保证这个变量在多线程环境下的可见性，并在一定程度上保证有序性。它最经典的应用场景就是停止标记或状态开关。

## 13. 为什么 `volatile` 不能保证 `i++` 线程安全？

示例回复：

因为 `i++` 不是一个原子操作，它通常包含读取、计算、写回三个步骤。`volatile` 只能让线程更容易看到最新值，但不能把这三个步骤合成一个不可分割的原子动作，所以它不能单独保证 `i++` 的线程安全。

## 14. `Lock` 和 `synchronized` 有什么区别？

示例回复：

它们的目标类似，都是为了解决共享资源的并发访问问题。区别在于，`synchronized` 更偏语法层面的内置锁，写法简单直观；`Lock` 更偏 API 层面的显式锁，控制更灵活，比如支持 `tryLock()`、可中断获取锁、公平锁和非公平锁等。

## 15. 什么是可重入？

示例回复：

可重入指的是同一个线程在已经持有某把锁的前提下，还可以再次获得这把锁，而不会把自己阻塞住。`ReentrantLock` 是可重入锁，`synchronized` 也是可重入的。

## 16. 为什么 `unlock()` 一般要放在 `finally` 中？

示例回复：

因为如果业务代码执行过程中抛了异常，而你没有在 `finally` 中释放锁，就可能导致锁一直不被释放，其他线程一直拿不到锁。所以 `Lock` 一般都和 `try-finally` 搭配使用。

## 17. 为什么一般更推荐使用线程池？

示例回复：

因为频繁创建和销毁线程有成本，线程太多还会增加调度和上下文切换开销。线程池的核心价值是复用线程、控制并发线程数量、统一管理任务执行过程，所以它更适合生产环境。

## 18. `execute()` 和 `submit()` 有什么区别？

示例回复：

`execute()` 更偏执行一个不需要返回值的任务；`submit()` 更灵活，它可以提交任务并返回 `Future`，便于后续获取执行结果或处理异常。

## 19. 线程池核心参数有哪些？

示例回复：

`ThreadPoolExecutor` 常见核心参数包括核心线程数 `corePoolSize`、最大线程数 `maximumPoolSize`、空闲线程存活时间 `keepAliveTime`、时间单位 `unit`、任务队列 `workQueue`、线程工厂 `threadFactory` 和拒绝策略 `handler`。面试里最好还能继续讲任务流转过程。

## 20. 线程池处理任务的大致流程是什么？

示例回复：

可以简单理解成四步：先看当前线程数是否小于核心线程数，如果小于就优先创建核心线程执行任务；如果核心线程满了，任务先进入队列；如果队列也满了而线程数还没到最大线程数，就继续创建非核心线程；如果队列满了、线程数也到上限了，就触发拒绝策略。

## 21. 为什么不建议无脑使用 `Executors` 默认工厂？

示例回复：

因为 `Executors` 的默认工厂会屏蔽很多关键参数，不同工厂可能带来队列过长或线程数失控等风险。生产环境里更常见的建议是直接显式使用 `ThreadPoolExecutor`，方便控制核心线程数、最大线程数、队列和拒绝策略。

## 22. `ConcurrentHashMap` 为什么比 `Hashtable` 更常用？

示例回复：

它们都可以用于线程安全的 `Map` 场景，但 `Hashtable` 的设计更老、控制更粗，而 `ConcurrentHashMap` 更适合现代并发场景，通常性能和伸缩性更好，所以实际开发和牛客面经里更常提到 `ConcurrentHashMap`。

## 23. `CopyOnWriteArrayList` 适合什么场景？

示例回复：

`CopyOnWriteArrayList` 更适合读多写少的并发场景，因为它采用写时复制思路，读操作体验较好，但写操作成本更高，所以不适合频繁修改的数据结构。

## 24. `BlockingQueue` 有什么用？

示例回复：

`BlockingQueue` 是带阻塞能力的队列接口，常用于生产者消费者模型和任务缓冲。队列空时消费者可以等待，队列满时生产者也可以等待，所以它在并发任务协作里非常常见，线程池内部也会涉及类似的任务排队思路。

## 25. 什么是死锁？怎么避免？

示例回复：

死锁可以理解成多个线程互相等待对方释放资源，最终谁都走不下去。避免思路通常包括：尽量固定加锁顺序、减少同时持有多把锁、缩小锁粒度，以及在复杂场景下考虑 `tryLock()` 之类更灵活的方式。

## 26. 什么是 `ThreadLocal`？

示例回复：

`ThreadLocal` 可以理解成线程本地变量，它为每个线程都维护一份独立副本。这样不同线程访问同一个 `ThreadLocal` 时，拿到的是各自自己的值，常见于用户上下文、TraceId 之类场景。

## 27. `CountDownLatch` 和 `CyclicBarrier` 有什么区别？

示例回复：

`CountDownLatch` 更像倒计时器，常见场景是一个线程等待其他多个线程完成某件事；`CyclicBarrier` 更像栅栏，强调一组线程都到齐之后再一起往下走。一个更偏等待完成，一个更偏统一出发。

## 28. 一段比较稳的总回答

如果面试官让我整体讲线程和并发，我一般会先从进程、线程、多线程、并发并行这些基础概念说起，再讲线程创建方式、线程状态，以及 `sleep()`、`wait()`、`notify()`、`join()` 这些常见方法。真正进入并发核心后，重点是线程安全，也就是原子性、可见性和有序性。对应的解决手段里，`synchronized` 更偏互斥和原子性，`volatile` 更偏可见性，`Lock` 则提供了更灵活的显式锁控制。再往实际开发走，就会讲到线程池和并发容器，比如 `ThreadPoolExecutor`、`ConcurrentHashMap`、`CopyOnWriteArrayList` 和 `BlockingQueue`。

## 29. 面试前最后背的结论

1. 线程这章的主线不是工具名，而是“问题和解决手段”
2. `run()` 和 `start()`、`sleep()` 和 `wait()`、`synchronized` 和 `volatile` 都是高频区别题
3. 线程安全一定要能讲到原子性、可见性、有序性
4. `volatile` 不能保证 `i++` 是非常高频的考点
5. 线程池要会讲参数和任务流转过程
6. 并发容器要会讲 `ConcurrentHashMap`、`CopyOnWriteArrayList`、`BlockingQueue`
