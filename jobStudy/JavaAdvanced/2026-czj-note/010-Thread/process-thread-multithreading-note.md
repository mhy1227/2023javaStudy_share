# 进程、线程、多线程基础说明

## 1. 为什么这一份要单独写

线程这一章如果一上来就讲：

- 继承 `Thread`
- 实现 `Runnable`
- `Callable`
- 线程池

很容易出现一个问题：

- 写法知道一点
- 但定义说不清

面试里经常不是先问你“代码怎么写”，而是先问：

- 什么是进程？
- 什么是线程？
- 什么是多线程？
- 为什么要用多线程？

所以这一份先把最基础的定义讲清楚。

## 2. 什么是进程

进程可以理解成：

- 一个正在运行中的程序实例

例如：

- 启动一个 Java 程序
- 打开一个浏览器
- 打开一个微信

这些都可以看成一个进程。

进程的重点不在“它只是一个程序”，而在于：

- 它是资源分配的基本单位

也就是说，一个进程通常会持有自己的一部分资源，比如：

- 内存空间
- 文件资源
- 网络连接等系统资源

## 3. 什么是线程

线程可以理解成：

- 进程中的一个执行路径
- 进程中的执行单元

一个进程里可以只有一个线程，也可以有多个线程。

这些线程通常共享同一个进程里的大部分资源，所以线程之间协作更方便，但也更容易产生竞争问题。

面试里常见表述：

- 线程是 CPU 调度的基本单位

## 4. 什么是多线程

多线程就是：

- 一个进程里同时存在多个线程
- 这些线程共同完成不同任务，或者共同推进同一个任务

例如一个程序里：

- 一个线程负责接收请求
- 一个线程负责写日志
- 一个线程负责处理业务

这就可以看成多线程。

## 5. 为什么要有多线程

多线程不是为了“显得高级”，而是因为很多程序天然就需要同时处理多件事。

最常见的几个原因：

### 5.1 提高响应能力

例如：

- 主线程负责界面
- 后台线程负责耗时任务

这样界面不容易卡死。

### 5.2 提高资源利用率

当某个线程在等待 IO 时，CPU 可以去执行别的线程。

### 5.3 更适合拆任务

例如：

- 并发处理多个文件
- 同时请求多个接口
- 异步记录日志

## 6. 多线程的好处和代价

### 6.1 好处

- 提高程序响应速度
- 更好利用 CPU 和等待时间
- 适合处理多个独立任务

### 6.2 代价

- 代码更复杂
- 线程之间会抢资源
- 容易出现线程安全问题
- 线程太多会带来上下文切换开销

所以面试里不要只说优点，也要顺手补一句：

- 多线程能提升效率，但也会带来复杂度和同步成本

## 7. 进程和线程的区别怎么答

这是非常高频的一题，建议按这条线去答：

1. 进程是资源分配的基本单位，线程是 CPU 调度的基本单位
2. 进程之间资源相对独立，线程共享同一进程内的大部分资源
3. 进程切换成本通常更高，线程切换成本通常相对更低
4. 线程通信更方便，但线程安全问题也更多

## 8. 什么是单线程，什么是多线程

### 8.1 单线程

单线程就是一个进程里主要只有一条执行路径。

优点：

- 逻辑简单
- 执行顺序清晰

缺点：

- 一旦有耗时操作，整体容易阻塞

### 8.2 多线程

多线程就是一个进程里有多条执行路径并发推进。

优点：

- 更灵活
- 更适合复杂任务

缺点：

- 需要处理共享资源竞争

## 9. 面试里“什么是多线程”怎么回答更稳

可以这样答：

多线程就是在一个进程中存在多个线程，这些线程共享进程内的大部分资源，并由 CPU 调度执行。它的意义主要在于提高程序响应能力、提升资源利用率，以及更适合拆分和并发推进任务。但多线程也会带来线程安全、上下文切换和同步控制等问题。

## 10. 当前阶段容易漏掉什么

这一段最容易漏的不是定义本身，而是下面这些点：

1. 只会背“线程是执行单元”，但不会说为什么要有线程
2. 只会说多线程提高效率，但不会补充代价和问题
3. 只会说进程和线程区别，但说不清资源共享和切换成本
4. 只会说“能并发”，但不会继续讲线程安全问题

## 11. 一句话总结

这部分最核心的不是死背定义，而是把这条线讲顺：

- 进程是运行中的程序实例
- 线程是进程中的执行单元
- 多线程是一个进程中多个线程共同推进任务
- 多线程带来效率，也带来复杂度和线程安全问题
