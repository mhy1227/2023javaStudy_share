# 线程高频面试题

## 1. 这一份是干什么的

这一份先不展开所有细节，而是先把线程这一章最常见的高频问题拎出来。

后面等各子文档补全后，这一份可以继续扩写成：

- 只放问题版
- 问题加示例回复版

## 2. 第一梯队：几乎必问

1. 进程和线程有什么区别？
2. 创建线程有哪几种方式？
3. `run()` 和 `start()` 有什么区别？
4. `Runnable` 和 `Callable` 有什么区别？
5. 线程有哪些状态？
6. `sleep()` 和 `wait()` 有什么区别？
7. 什么是线程安全？
8. `synchronized` 和 `volatile` 有什么区别？
9. 为什么一般更推荐使用线程池？

## 3. 第二梯队：很容易继续追问

1. `wait()` 为什么必须在同步代码块中使用？
2. `notify()` 和 `notifyAll()` 有什么区别？
3. `sleep()` 会不会释放锁？
4. `wait()` 会不会释放锁？
5. `synchronized` 锁的是什么？
6. `volatile` 为什么不能保证 `i++` 线程安全？
7. `Lock` 和 `synchronized` 怎么选？
8. `submit()` 和 `execute()` 有什么区别？

## 4. 第三梯队：面试常见延伸题

1. 什么是死锁？怎么避免？
2. 什么是原子性、可见性、有序性？
3. 什么是上下文切换？
4. 线程池核心参数有哪些？
5. 为什么不建议无脑使用 `Executors.newFixedThreadPool()` 这类默认工厂？
6. `ConcurrentHashMap` 和 `Hashtable` 有什么区别？
7. `CountDownLatch` 和 `CyclicBarrier` 有什么区别？

## 5. 当前阶段怎么复习更有效

线程这一章不建议一上来就背很多工具名。

更建议按这条线复习：

1. 先把线程和进程的概念讲清楚
2. 再把线程创建和生命周期讲清楚
3. 再把 `sleep`、`wait`、`notify`、`join` 理清
4. 再去讲线程安全和锁
5. 最后再补线程池和并发工具

## 6. 一句话总结

线程面试高频的核心，不是工具多，而是主线很稳定：线程基础、线程协作、线程安全、锁、线程池。
