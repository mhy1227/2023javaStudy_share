# 线程实战专题高频面试题

## 1. 这一份是干什么的

这一份不是把线程基础题再抄一遍，而是把 `practical-topics` 这一层最容易被继续追问的问题拎出来。

当前主要围绕三块：

1. 线程池实战
2. `CompletableFuture`
3. 线程排障

## 2. 第一梯队：线程池实战几乎必问

1. 为什么不推荐在业务代码里频繁 `new Thread()`？
2. 为什么实际开发更推荐使用线程池？
3. 线程池核心参数有哪些？
4. 核心线程数和最大线程数怎么理解？
5. CPU 密集型任务和 IO 密集型任务怎么配置线程池？
6. 线程池的阻塞队列怎么选？
7. 拒绝策略有哪些？项目里怎么选？
8. `submit()` 和 `execute()` 有什么区别？
9. 为什么不建议无脑使用 `Executors` 默认线程池工厂？
10. 线程池打满时你一般先看什么？

## 3. 第二梯队：`CompletableFuture` 高频追问

1. `CompletableFuture` 是干什么的？
2. `Future` 和 `CompletableFuture` 有什么区别？
3. `runAsync()` 和 `supplyAsync()` 有什么区别？
4. `thenApply()`、`thenAccept()`、`thenRun()` 有什么区别？
5. `thenCompose()` 和 `thenCombine()` 怎么区分？
6. `allOf()` 和 `anyOf()` 分别适合什么场景？
7. `CompletableFuture` 里异常一般怎么处理？
8. 为什么 `CompletableFuture` 最好配合自定义线程池使用？
9. 如果多个异步任务要并行查数据再汇总，你会怎么做？
10. `join()` 和 `get()` 在使用感受上有什么区别？

## 4. 第三梯队：线程排障高频追问

1. CPU 飙高时你一般怎么查？
2. 线程池打满了你怎么排查？
3. 怀疑死锁时你会怎么定位？
4. 大量线程都在等待时，你怎么区分是锁问题还是下游阻塞？
5. 线程 dump 主要看什么？
6. 线程状态里的 `BLOCKED`、`WAITING`、`TIMED_WAITING` 怎么理解？
7. 线上偶发卡顿，你怎么判断是不是线程问题？
8. 为什么排障不能只会背 `jstack`？
9. 如果发现某个线程一直占 CPU，很可能有哪些原因？
10. 如果你怀疑是异步任务设计不合理，你会怎么优化？

## 5. 第四梯队：项目场景综合题

1. 接口里有多个远程调用，你会直接串行调还是考虑异步编排？
2. 如果一个接口既要快响应，又要做非核心日志或通知，你怎么设计？
3. 线程池为什么不能整个系统只配一个？
4. 核心业务线程池和非核心业务线程池为什么通常要隔离？
5. 线程池参数为什么要结合监控一起调，而不是一次配死？
6. 异步任务为什么也可能把系统拖慢？
7. 什么时候你会放弃继续加线程，而去做限流、降级或拆分任务？
8. 如果面试官问“你在线上怎么用过线程”，你最少要讲哪几层？

## 6. 当前阶段怎么复习更有效

建议复习顺序：

1. 先把线程池“为什么要用、怎么配、怎么排查”讲顺
2. 再把 `CompletableFuture` 的常见方法和异步编排口径讲顺
3. 最后把 CPU 飙高、死锁、线程池打满这几类排障问题串起来

## 7. 一句话总结

线程实战专题面试的核心不是再背线程定义，而是回答三个现实问题：

- 线程怎么统一管理
- 异步任务怎么组织
- 线上出问题了怎么查
