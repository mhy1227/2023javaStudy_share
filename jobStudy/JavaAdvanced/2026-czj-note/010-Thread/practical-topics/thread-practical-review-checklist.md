# 线程实战专题考前复盘 Checklist

## 1. 这一份怎么用

这一份不是长篇笔记，而是 `010-Thread/practical-topics` 这一层的考前压缩复盘清单。

适合用法：

- 面试前最后 10 到 15 分钟快速过一遍
- 检查自己对线程实战追问是否还能答顺
- 把“线程池、异步编排、排障”三块串成一条线

复习时建议每块只问自己四个问题：

- 这一块最核心在讲什么？
- 我最容易答乱的点是什么？
- 面试官最爱怎么追问？
- 我能不能用两三句话讲清楚？

## 2. 线程池实战快速复盘

主线：

- 线程池实战不是再背参数名，而是根据业务场景做配置和排查

必须记住：

- 不要在业务里随手 `new Thread()`
- CPU 密集型任务，线程数通常更接近 CPU 核数
- IO 密集型任务，线程数通常可以适当放大
- 线程池参数要结合任务耗时、峰值流量、下游响应时间一起看
- 线程池不是只看线程数，还要看队列、拒绝策略、监控

最容易错：

- 一上来死背“线程池大小 = 核数 * 2”
- 忽略任务执行时间和队列堆积
- 只会说四种拒绝策略，不会讲怎么选
- 把 `submit` 和 `execute` 说混

面试怎么说：

- 线程池实战更重要的是根据任务类型做参数选择，并且要能从活跃线程数、队列积压、任务耗时这些指标判断线程池是不是配置合理。

## 3. `CompletableFuture` 快速复盘

主线：

- `CompletableFuture` 核心不是“开异步”，而是“编排异步任务”

必须记住：

- `runAsync` 适合无返回值任务
- `supplyAsync` 适合有返回值任务
- `thenApply` 是接结果再加工并返回
- `thenAccept` 是消费结果但不返回
- `thenRun` 是不关心上一步结果，继续执行后续动作
- `allOf` 常用于等待多个任务都完成
- `exceptionally` 常用于兜底异常处理

最容易错：

- 把 `Future` 和 `CompletableFuture` 的能力说成一样
- 不知道链式调用到底是在串行编排还是组合结果
- 只会说方法名，不会说典型场景
- 忘了默认线程池和自定义线程池的区别

面试怎么说：

- `CompletableFuture` 比传统 `Future` 更适合做异步编排，比如多个远程调用并行执行、最后统一汇总结果，并且可以把后续加工和异常处理一起串起来。

## 4. 线程排障快速复盘

主线：

- 线程排障的核心是先判断问题属于哪一类，再选工具定位

必须记住：

- 死锁先看线程 dump
- CPU 飙高要落到具体线程和调用栈
- 线程池打满要看活跃线程数、队列长度、任务耗时、下游依赖
- 线程长时间等待，要区分是锁竞争、IO 阻塞还是外部服务慢
- 排障不能只报工具名，必须能讲排查顺序

最容易错：

- 上来就背 `jstack`、`jconsole`，但没有排障主线
- 只会说“看日志”，不会定位线程状态
- 把死锁、阻塞、线程池打满混成一类问题
- 不会把线程问题再落回代码和业务场景

面试怎么说：

- 我会先判断是死锁、CPU 飙高、线程池堆积还是线程阻塞，再结合监控和线程栈逐步缩小范围，最后落到具体线程在执行什么代码、等待什么资源。

## 5. 考前最后五个总复盘点

### 5.1 为什么不推荐频繁 `new Thread()`

- 成本高、难管理、线程数量容易失控，也不方便统一做监控和限流

### 5.2 线程池为什么不能只背参数

- 因为线程池本质上是在平衡吞吐、延迟、资源占用和风险，不结合场景就没有意义

### 5.3 为什么 `CompletableFuture` 常被问

- 因为现代项目里异步调用、聚合接口、并行查多个下游都很常见，它比传统 `Future` 更贴近真实开发

### 5.4 为什么排障题重要

- 因为这类题能直接区分“只背过概念”和“真正接触过线上问题”的候选人

### 5.5 这一层和前面线程主干的区别是什么

- 前面更多讲线程基础、锁和底层机制；这一层更强调真实项目里怎么配、怎么用、怎么排查

## 6. 最后一轮快问快答

### 6.1 面试官问：线程池你们怎么配？

可以先答：

- 我会先区分任务是 CPU 密集还是 IO 密集，再结合任务耗时、峰值流量、队列堆积和下游响应时间来定核心线程数、最大线程数和队列长度，而不是死背一个固定公式。

### 6.2 面试官问：`submit` 和 `execute` 有什么区别？

可以先答：

- `execute` 只负责执行任务，没有返回值；`submit` 会返回 `Future`，可以获取执行结果，也更方便感知任务异常。

### 6.3 面试官问：`CompletableFuture` 比 `Future` 强在哪？

可以先答：

- `Future` 更像异步结果占位，而 `CompletableFuture` 更适合异步编排，能把结果加工、多个任务汇总和异常处理串到一起。

### 6.4 面试官问：CPU 飙高你怎么查？

可以先答：

- 先确认是线程层问题，再定位高 CPU 线程，结合线程栈看它在执行什么逻辑，判断是死循环、频繁竞争锁还是某段热点代码异常消耗 CPU。

### 6.5 面试官问：线程池满了怎么办？

可以先答：

- 先判断是任务执行慢、下游阻塞还是峰值流量过高，再看活跃线程数、队列积压和拒绝策略，必要时做限流、降级、拆分任务或者重新调整线程池参数。

## 7. 一句话收尾

这一层最关键的不是把线程概念背得更抽象，而是要能回答三个现实问题：

- 线程怎么管理
- 异步怎么编排
- 出问题了怎么查
