# 锁分类说明

## 1. 为什么这一份要单独补

面试里问锁，很多时候不会一上来就问你源码细节。

更常见的问法其实是：

- 你知道锁有哪些分类？
- 你怎么理解乐观锁和悲观锁？
- 公平锁和非公平锁有什么区别？
- 可重入锁是什么？
- 共享锁和独占锁怎么区分？

也就是说，很多面试官会先看你有没有“锁分类的整体视角”。

所以这一份主要是把：

- 锁常见怎么分
- 每一类大概在说什么
- 当前阶段最值得先掌握哪些

统一收起来。

## 2. 先一句话理解“锁分类”在干什么

可以先记一个口径：

- 锁分类本质上是在从不同角度给锁打标签

也就是说：

- 不是说一把锁只属于一种类型
- 而是同一把锁，可能同时带有多个分类属性

例如：

- `ReentrantLock` 可以同时被看作可重入锁、独占锁、悲观锁

## 3. 为什么锁要分类

因为面试和实际开发里，大家关心的问题并不一样。

有时关心的是：

- 线程拿锁时是不是排队公平

有时关心的是：

- 同一个线程能不能重复进入

有时关心的是：

- 是不是允许多个线程共享访问

有时关心的是：

- 是先假设冲突很多，还是先假设冲突不多

所以锁的分类，本质上是为了：

- 更方便地理解锁的能力边界和使用场景

## 4. 最常见的一组分类图

当前阶段最常见、最值得先掌握的通常有：

1. 乐观锁 / 悲观锁
2. 公平锁 / 非公平锁
3. 可重入锁 / 不可重入锁
4. 共享锁 / 独占锁
5. 读锁 / 写锁
6. 内置锁 / 显式锁

如果是基础面试，这几组通常已经够用了。

## 5. 乐观锁和悲观锁

这一组通常是锁分类里最爱被问的。

可以先粗理解成：

- 乐观锁：先假设冲突不多，先执行，提交时再检查
- 悲观锁：先假设冲突容易发生，所以先锁住再操作

如果你要快速举例：

- CAS 更偏乐观锁思路
- `synchronized`、`ReentrantLock` 更偏悲观锁思路

这一组我建议你单独配合另一份文档一起复习。

## 6. 公平锁和非公平锁

这一组主要关注的是：

- 线程拿锁时，是否更强调排队顺序

可以先记成：

- 公平锁：更倾向按先来后到排队
- 非公平锁：不一定严格按排队顺序，更强调吞吐

一般来说：

- 公平锁更强调顺序感
- 非公平锁更强调效率

面试里如果被问：

- `ReentrantLock` 可以配公平策略还是非公平策略

你至少要有这个认知。

## 7. 可重入锁和不可重入锁

这一组主要关注的是：

- 同一个线程拿到锁后，能不能再次进入这把锁保护的代码

当前阶段最值得先记住的是：

- `synchronized` 是可重入的
- `ReentrantLock` 也是可重入的

所谓可重入，直白一点说就是：

- 同一个线程不会因为再次进入而把自己锁死

## 8. 共享锁和独占锁

这一组主要关注的是：

- 同一时刻是允许多个线程共享访问，还是只允许一个线程独占访问

可以先粗理解成：

- 独占锁：更像一次只让一个线程进
- 共享锁：更像按规则允许多个线程一起进

例如：

- `ReentrantLock` 更偏独占
- 读写锁里的读锁，更容易让你联想到共享访问

## 9. 读锁和写锁

这一组本质上是共享锁 / 独占锁在读写场景下的具体体现。

可以先记成：

- 读锁：通常允许多个读同时进行
- 写锁：写操作通常要求更严格的互斥

所以它更适合：

- 读多写少

这就是为什么面试里常会追：

- `ReentrantReadWriteLock` 适合什么场景？

## 10. 内置锁和显式锁

这一组也很常见。

可以先记成：

- 内置锁：例如 `synchronized`
- 显式锁：例如 `ReentrantLock`

它们的差异通常不在“能不能同步”，而在于：

- 使用方式
- 灵活程度
- 可中断、超时、公平策略等能力

## 11. 面试里怎么回答“锁有哪些分类”

可以直接背一版：

锁常见可以从不同角度分类，比如乐观锁和悲观锁、公平锁和非公平锁、可重入锁和不可重入锁、共享锁和独占锁、读锁和写锁、内置锁和显式锁。它们不是互斥关系，而是从不同维度给锁打标签。面试里我一般会先讲乐观和悲观，再讲可重入、公平、共享和独占这些更常见的分类。

## 12. 当前阶段最值得先掌握哪几类

如果你现在还不想把锁分类记得太散，可以优先先记下面这几组：

1. 乐观锁 / 悲观锁
2. 可重入锁 / 不可重入锁
3. 公平锁 / 非公平锁
4. 共享锁 / 独占锁
5. 内置锁 / 显式锁

这几组已经能撑起大部分基础面试。

## 13. 当前阶段记忆重点

这一份最少要记住：

1. 锁分类是在不同角度给锁打标签
2. 同一把锁可能同时属于多个分类
3. 乐观锁和悲观锁是最高频的一组
4. `synchronized` 和 `ReentrantLock` 都是可重入的
5. 公平锁更强调顺序，非公平锁更强调吞吐
6. 读写锁适合读多写少
7. `synchronized` 属于内置锁，`ReentrantLock` 属于显式锁
