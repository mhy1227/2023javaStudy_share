# 线程一页总览

## 1. 这一份是干什么的

这一份不是完整知识点，而是线程这一章的面试前速记版。

适合用法：

- 面试前最后 10 分钟
- 一轮快速复盘
- 找感觉，不细抠

## 2. 主线一定要记住

线程这一章不要背成一堆零散工具名，最核心主线只有这一条：

1. 为什么要有线程
2. 多线程为什么会出问题
3. Java 提供了哪些手段解决这些问题
4. 实际开发里怎么管理线程和共享数据

## 3. 最基础的几组概念

- 进程：更偏资源分配单位
- 线程：更偏 CPU 调度单位
- 并发：一段时间内多个任务都在推进
- 并行：同一时刻多个任务同时执行
- 多线程的收益：响应更快、资源利用率更高
- 多线程的代价：线程安全、上下文切换、复杂度上升

## 4. 创建线程这块最常问的

- 继承 `Thread`
- 实现 `Runnable`
- 实现 `Callable`
- 线程池提交任务

最常见口径：

- 入门演示可以用 `Thread`
- 实际开发更常见的是线程池

## 5. 必背区别题

- `run()` vs `start()`
- `sleep()` vs `wait()`
- `Runnable` vs `Callable`
- `synchronized` vs `volatile`
- `synchronized` vs `Lock`
- `execute()` vs `submit()`

## 6. 线程安全一定要会讲

线程安全通常围绕三件事：

1. 原子性
2. 可见性
3. 有序性

必会例子：

- `i++` 不是原子操作
- 停止标记适合用 `volatile`
- 卖票是典型共享资源竞争场景

## 7. 同步手段的最简主线

### 7.1 `synchronized`

- 最基础的同步手段
- 核心是互斥进入临界区
- 实例方法锁 `this`
- 静态方法锁 `Class` 对象

### 7.2 `volatile`

- 主要解决可见性
- 还能保证一定有序性
- 不能保证 `i++` 原子性

### 7.3 `Lock / ReentrantLock`

- 目标类似于 `synchronized`
- 但控制更灵活
- 记住：`unlock()` 一般放 `finally`

## 8. 线程池这块最简主线

为什么要用线程池：

- 复用线程
- 控制并发数量
- 统一管理任务

必会点：

- `execute()` 和 `submit()`
- `ThreadPoolExecutor` 核心参数
- 任务流转过程
- 为什么不建议无脑用 `Executors`

## 9. 并发容器最简主线

- `ConcurrentHashMap`：最常见线程安全 `Map`
- `CopyOnWriteArrayList`：读多写少
- `BlockingQueue`：生产者消费者、任务缓冲

## 10. 并发工具类最简主线

- `AtomicInteger`：原子计数
- `ThreadLocal`：线程隔离
- `CountDownLatch`：一个等多个完成
- `CyclicBarrier`：多个线程到齐再一起走
- `Semaphore`：限制并发数量

## 11. 死锁只记住这些

- 本质：循环等待资源
- 高危场景：多把锁、顺序不一致
- 避免：固定加锁顺序、减少嵌套锁、必要时用 `tryLock()`
- 排查：看线程栈、看锁等待关系
