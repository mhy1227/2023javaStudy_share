# 分布式基础设施高频问题清单

## 1. 这份文档解决什么问题

这一篇只做一件事：

- 把牛客、LeetCode、常见后端高并发面试里，围绕分布式基础设施常问的问题单独拎出来

注意：

- 这里只整理问题
- 不展开标准答案
- 不做太多解释

适合：

- 先看题，判断自己哪里不会
- 后续再配合回答包做口语训练

## 2. 高并发总线题

- 你怎么设计一个高并发系统
- 高并发系统为什么不能只靠数据库硬扛
- 高并发治理为什么不是一上来就分库分表
- 缓存、限流、异步、削峰、隔离、降级这几个词你怎么串起来
- 秒杀系统你会怎么设计

## 3. MQ / 削峰类问题

- 为什么 MQ 适合做异步削峰
- 削峰、异步、解耦分别是什么意思
- 如果所有流量都先进 MQ，会不会有新问题
- 消息积压怎么办
- 消费失败怎么办
- 消息重复消费怎么办
- 为什么说 MQ 不能自动解决一致性问题

## 4. 分布式锁类问题

- 为什么本地锁不能解决多实例并发问题
- Redis 分布式锁最核心要注意什么
- 为什么加锁要带过期时间
- 为什么解锁不能直接 `del`
- 为什么锁 value 要唯一
- 业务执行时间超过锁超时怎么办
- Redis 分布式锁一定绝对安全吗
- 分布式锁和分布式事务是什么关系

## 5. 分布式 ID 类问题

- 为什么订单号通常不用数据库自增
- UUID、Redis 发号、号段模式、雪花算法怎么比较
- 雪花算法解决的核心问题是什么
- 雪花算法为什么说是大致递增
- 雪花算法的核心风险点是什么
- 时钟回拨为什么危险
- 分布式 ID 和分库分表为什么经常一起问

## 6. 分库分表基础问题

- 什么情况下才值得考虑分库分表
- 分库分表本质上在解决什么问题
- 分库和分表有什么区别
- 横向拆分和纵向拆分怎么区分
- 为什么面试官讲分库分表时经常顺带追问横向和纵向
- 垂直拆分和水平拆分分别解决什么问题
- 分库分表是不是一定比单库单表高级

## 7. 分片键与路由问题

- 分片键怎么选
- 为什么分片键选错会很痛苦
- 常见分片算法有哪些
- 简单取模有什么问题
- 范围分片为什么容易热点
- 一致性哈希一般在什么场景下会被提到
- 非分片键查询怎么办

## 8. 分库分表后的查询问题

- 分库分表后跨库 `join` 怎么办
- 全局分页为什么会变麻烦
- 全局排序为什么会变麻烦
- 聚合查询为什么会更复杂
- range 查询和模糊查询为什么更难做
- 分库分表后怎么做历史数据查询

## 9. 热点与扩容问题

- 为什么分片之后仍然可能出现热点分片
- 热 key、热用户、热商户为什么会把某个分片打爆
- 热点分片怎么处理
- 后面从 4 个分片扩到 8 个怎么办
- 扩容和迁移为什么是高风险点
- 数据迁移期间怎么保证业务可用

## 10. 一致性与事务问题

- 分库之后为什么本地事务就不够了
- 强一致和最终一致怎么区分
- 为什么很多互联网系统更偏最终一致
- 分库分表后订单和库存不在一个库怎么办
- 为什么不能一听跨库事务就只会答 2PC
- MQ、本地消息表、补偿、幂等、对账怎么串起来
- 分布式锁为什么不能替代一致性方案

## 11. 高频场景题

- 秒杀系统里库存超卖怎么处理
- 下单成功但发券失败怎么办
- 订单创建成功但扣库存失败怎么办
- 为什么订单号通常要用分布式 ID
- 分库分表后的订单系统为什么还会继续问一致性
- 如果某个爆款商品访问量特别高怎么办
- 如果某个大商户数据都打到一个分片怎么办

## 12. 项目表达题

- 你项目里真的做过分库分表吗
- 你们为什么需要分布式 ID
- 你们为什么用了 MQ，不直接同步调不行吗
- 你们为什么用了 Redis 分布式锁
- 你们怎么处理订单、库存、优惠这些跨库动作
- 你们有没有遇到热点分片、扩容迁移、异常单对账这些问题

## 13. 一句话收尾

> 这份清单的价值，不是让你继续背更多名词，而是先把“面试官到底会追哪些问题”从杂乱知识点里单独抽出来。
