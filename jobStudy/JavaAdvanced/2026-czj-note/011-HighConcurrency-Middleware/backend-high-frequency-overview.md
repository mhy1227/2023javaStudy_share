# 后端高频方向总览：MySQL / Redis / Spring / MyBatis / MQ

## 1. 这份文档解决什么问题

线程、锁、线程池、高并发基础复习完之后，面试官通常不会只停留在 Java 语法层面，而是继续追问后端系统里更常见的几个方向：

- MySQL：数据怎么存、怎么查得快、事务怎么保证一致性。
- Redis：缓存怎么用、怎么避免缓存问题、为什么快、分布式锁怎么落地。
- Spring / Spring Boot：项目为什么能跑起来、Bean 怎么创建、事务为什么会失效。
- MyBatis：SQL 怎么执行、Mapper 为什么能代理、`#{}` 和 `${}` 有什么区别。
- MQ：为什么要异步、怎么削峰、消息丢了、重复了、积压了怎么办。

这些内容不是 Java 基础语法，但基本属于 Java 后端面试高频区。当前阶段可以先建立全局地图，后续再按模块拆目录详细展开。

## 2. 建议复习顺序

如果以面试收益排序，建议先按这个顺序走：

1. MySQL
2. Redis
3. Spring / Spring Boot
4. MQ
5. MyBatis

原因很简单：MySQL 和 Redis 最容易被问到场景题，Spring 是 Java 项目的基础框架，MQ 是高并发和系统设计的延伸，MyBatis 虽然常见，但整体深度通常低于前三个。

## 3. MySQL 高频地图

MySQL 面试的核心不是“会写 SQL”，而是能解释清楚：为什么这条 SQL 慢，索引为什么生效或失效，事务为什么能保证隔离，锁和日志分别解决什么问题。

常见问题：

- MySQL 索引是什么？为什么 B+ 树适合做索引？
- 聚簇索引和非聚簇索引有什么区别？
- 主键索引、唯一索引、普通索引、联合索引分别是什么？
- 联合索引的最左前缀原则是什么？
- 哪些情况会导致索引失效？
- `explain` 怎么看？重点看哪些字段？
- 什么是回表？什么是覆盖索引？
- MySQL 的事务 ACID 分别是什么？
- 事务隔离级别有哪些？默认隔离级别是什么？
- 脏读、不可重复读、幻读分别是什么？
- MVCC 是什么？解决了什么问题？
- undo log、redo log、binlog 分别干什么？
- MySQL 有哪些锁？行锁、表锁、间隙锁、临键锁分别是什么？
- 慢 SQL 怎么排查和优化？
- 主从复制大概怎么做？可能有什么延迟问题？

面试可先记一条主线：

> MySQL 高频主要围绕索引、事务、锁、日志、SQL 优化展开。索引用来提升查询效率，事务和 MVCC 解决并发一致性，锁解决并发写冲突，日志保证崩溃恢复和主从复制，`explain` 和慢日志用于定位 SQL 性能问题。

## 4. Redis 高频地图

Redis 面试通常从“为什么快”开始，然后进入缓存三大问题、持久化、过期淘汰、分布式锁和集群。

常见问题：

- Redis 为什么快？
- Redis 常见数据类型有哪些？分别适合什么场景？
- String、Hash、List、Set、ZSet 分别怎么用？
- 缓存穿透、缓存击穿、缓存雪崩分别是什么？怎么解决？
- 缓存和数据库一致性怎么保证？
- 先删缓存还是先更新数据库？
- Redis 的过期策略是什么？
- Redis 的内存淘汰策略有哪些？
- RDB 和 AOF 有什么区别？
- Redis 主从、哨兵、Cluster 分别解决什么问题？
- Redis 分布式锁怎么实现？
- `set nx ex` 为什么比 `setnx` 再 `expire` 更安全？
- 释放锁为什么要校验唯一标识？
- Redisson 看门狗机制解决什么问题？
- Redis 大 key、热 key 是什么？有什么风险？

面试可先记一条主线：

> Redis 高频主要围绕缓存、性能、持久化、高可用和分布式锁。回答时不要只背命令，要结合场景说明，比如热点数据用缓存提升查询性能，但会带来穿透、击穿、雪崩和一致性问题，需要用布隆过滤器、互斥锁、随机过期时间、延迟双删等方式处理。

## 5. Spring / Spring Boot 高频地图

Spring 面试最常问 IOC、AOP、Bean 生命周期、循环依赖、事务、Spring MVC 流程和 Spring Boot 自动配置。

常见问题：

- 什么是 IOC？什么是 DI？
- Spring Bean 的生命周期是什么？
- Bean 的作用域有哪些？
- 单例 Bean 是线程安全的吗？
- Spring 怎么解决循环依赖？
- 三级缓存分别是什么？
- 为什么构造器注入的循环依赖解决不了？
- AOP 是什么？底层是 JDK 动态代理还是 CGLIB？
- Spring 事务传播行为有哪些？
- `@Transactional` 为什么会失效？
- 同一个类里方法内部调用事务为什么可能不生效？
- Spring MVC 请求流程是什么？
- `DispatcherServlet` 做了什么？
- Spring Boot 自动配置原理是什么？
- `starter` 的作用是什么？

面试可先记一条主线：

> Spring 的核心是 IOC 和 AOP。IOC 负责对象创建和依赖管理，AOP 负责在不侵入业务代码的情况下增强逻辑，比如事务、日志、权限。Spring Boot 则是在 Spring 基础上做自动配置和快速启动，减少手动 XML 或配置类。

## 6. MyBatis 高频地图

MyBatis 面试通常不会特别深，但容易考基础细节，尤其是 SQL 注入、Mapper 代理、缓存和动态 SQL。

常见问题：

- MyBatis 是什么？和 JDBC 相比解决了什么问题？
- `#{}` 和 `${}` 有什么区别？
- 为什么 `${}` 有 SQL 注入风险？
- Mapper 接口没有实现类，为什么能调用？
- MyBatis 一级缓存和二级缓存是什么？
- MyBatis 动态 SQL 有哪些标签？
- `resultType` 和 `resultMap` 有什么区别？
- 一对一、一对多映射怎么处理？
- MyBatis 插件机制大概是什么？
- MyBatis 和 Hibernate 有什么区别？
- 批量插入怎么做？
- 分页怎么做？

面试可先记一条主线：

> MyBatis 是半自动 ORM 框架，核心是把 Java 方法调用映射到 SQL 执行。它保留了 SQL 的可控性，同时通过 Mapper 代理、参数映射、结果映射、动态 SQL 降低 JDBC 样板代码。

## 7. MQ 高频地图

MQ 经常和高并发、削峰、异步、解耦一起问。面试官更关心你是否知道 MQ 带来的新问题，而不是只说“用了 MQ 就好了”。

常见问题：

- 为什么要用消息队列？
- MQ 的异步、解耦、削峰分别是什么意思？
- MQ 会带来哪些问题？
- 消息丢失怎么处理？
- 重复消费怎么处理？
- 消息积压怎么处理？
- 如何保证消息顺序？
- 什么是死信队列？
- 什么是延迟消息？
- 什么是事务消息？
- Kafka、RabbitMQ、RocketMQ 大概有什么区别？
- 消费失败如何重试？
- 如何设计一个下单后异步发券、发短信、扣库存的流程？

面试可先记一条主线：

> MQ 的价值是把同步链路拆成异步链路，把瞬时高峰流量削成后端能慢慢处理的流量，同时降低系统耦合。但 MQ 会引入消息丢失、重复消费、顺序性、积压和一致性问题，所以回答时必须同时说收益和代价。

## 8. 它们之间怎么串起来

这些模块不是孤立的，面试场景题通常会把它们串起来问。

典型链路：

1. 用户请求进入 Spring MVC。
2. Controller 调用 Service。
3. Service 开启 Spring 事务。
4. MyBatis 执行 SQL 操作 MySQL。
5. 热点查询引入 Redis 缓存。
6. 高并发写请求通过 MQ 削峰。
7. 分布式场景下用 Redis / Redisson 做分布式锁。
8. 慢 SQL、缓存击穿、消息积压、线程池打满都会变成排查题。

所以后续复习时，不能只背单个知识点，最好按业务场景串联：

- 查询场景：Spring MVC -> Service -> Redis -> MySQL -> 索引优化。
- 下单场景：接口限流 -> 分布式锁/库存扣减 -> MySQL 事务 -> MQ 异步处理。
- 秒杀场景：前端限流 -> 网关限流 -> Redis 预扣库存 -> MQ 削峰 -> MySQL 落库。
- 故障场景：慢 SQL、缓存雪崩、MQ 积压、线程池耗尽、Full GC。

## 9. 当前阶段怎么学

当前不建议一口气把所有方向全展开，否则文件会很多，容易只是在“写笔记”，没有真正记住。

更合适的做法：

1. 先用这份总览建立地图。
2. 下一步优先展开 MySQL，因为面试收益最高。
3. MySQL 写完索引、事务、锁、日志、优化之后，再切 Redis。
4. Redis 写完缓存问题、持久化、集群、分布式锁之后，再切 Spring。
5. Spring 写完 IOC、AOP、事务、MVC、Boot 自动配置之后，再补 MyBatis 和 MQ。

当前阶段最值得先背的版本：

> Java 后端面试除了 Java 基础、集合、异常、IO、泛型、线程之外，还会高频考 MySQL、Redis、Spring、MyBatis 和 MQ。MySQL 重点是索引、事务、锁、日志和 SQL 优化；Redis 重点是缓存问题、持久化、高可用、过期淘汰和分布式锁；Spring 重点是 IOC、AOP、Bean 生命周期、循环依赖和事务；MyBatis 重点是 Mapper 代理、参数映射、缓存和动态 SQL；MQ 重点是异步、解耦、削峰，以及消息丢失、重复消费、顺序和积压问题。
