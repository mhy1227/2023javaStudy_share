# 分布式基础设施最终 Checklist

## 1. 这份清单怎么用

这不是让你从头学这一章的。

它是给你在面试前最后一轮快速过稿用的。

你只需要拿这份清单检查自己：

- 这些点我能不能自己讲出来
- 哪些点我只是看着眼熟，但一张嘴就空
- 哪些点我能讲定义，但一进场景就接不住

## 2. 整体定位

- 我能说清楚 `011-HighConcurrency-Middleware` 这一章为什么不再是线程基础，而是线程之后的下一层
- 我能说清楚这一章主要在解决什么问题：高并发治理、分布式协作、数据扩展和一致性治理
- 我不会把 MQ、分布式锁、分布式 ID、分库分表这些点讲成彼此孤立的名词
- 我能把这一章整体讲成一条业务链路，而不是一堆零散技术点

## 3. 高并发系统设计主线

- 我知道高并发设计不是一上来就上中间件
- 我知道先判断瓶颈在 CPU、数据库、缓存、网络还是下游服务
- 我知道缓存、限流、异步、削峰、隔离、降级是一条常见治理主线
- 我知道稳定性和正确性很多时候比单纯吞吐更重要

## 4. MQ 与削峰

- 我能说清楚为什么 MQ 适合做异步削峰
- 我知道削峰、解耦、异步化不是完全一回事
- 我知道 MQ 只是把瞬时流量摊平，不代表问题自动消失
- 我知道继续追问时要能讲消息积压、重复消费、失败重试、一致性问题
- 我不会只答“用了 MQ 就能抗高并发”

## 5. 分布式锁

- 我知道本地锁为什么不能解决多实例并发互斥
- 我知道 Redis 分布式锁常见口径：唯一 value、过期时间、安全解锁
- 我知道为什么不能直接 `del`
- 我知道业务执行时间超过锁超时时间是一个真实边界
- 我知道分布式锁解决的是互斥，不是完整一致性方案

## 6. 分布式 ID

- 我能说清楚为什么单库自增不适合长期支撑分布式高并发场景
- 我知道 UUID、Redis 发号、号段模式、雪花算法各自大概适合什么场景
- 我知道订单系统这类核心链路为什么会自然引出分布式 ID
- 我知道雪花算法的核心价值是全局唯一、本地高性能生成和大致递增
- 我知道雪花算法的核心边界包括时钟回拨和机器号配置

## 7. 分库分表

- 我知道分库分表解决的是容量和吞吐问题，不是所有慢 SQL 的万能答案
- 我能区分垂直拆分、水平拆分、分库、分表
- 我能说清楚纵向拆分偏业务边界，横向拆分偏数据规模和吞吐
- 我知道分片键怎么选，至少会提分布均匀、查询命中和未来扩容
- 我知道分片策略选错会带来热点、倾斜和跨库查询问题
- 我知道分库分表不是第一选择，通常是在索引优化、缓存、读写分离之后才考虑

## 8. 分库分表后的代价

- 我知道分库分表后跨库 `join` 会更麻烦
- 我知道全局分页、排序、聚合都会更复杂
- 我知道扩容和数据迁移是高风险点
- 我知道分库分表是在拿复杂度换容量和吞吐
- 我不会把分库分表讲成“拆完就好了”

## 9. 一致性治理

- 我知道分库之后，本地事务边界为什么会被打破
- 我知道一致性不能脱离业务要求讲
- 我能区分强一致思路和最终一致思路
- 我知道互联网业务里更常见的是最终一致，而不是所有链路都上重型强一致方案
- 我不会一听跨库事务就只会背 2PC / 3PC

## 10. 最终一致常见手段

- 我知道 MQ、本地消息表、重试、补偿、幂等、对账是一套组合拳
- 我知道本地消息表主要是在解决“业务成功但消息没发出去”
- 我知道补偿机制不是随便重试一下，而是有明确恢复动作
- 我知道幂等几乎是所有异步重试链路的必选项
- 我知道对账是在捞异常单和收口状态

## 11. 场景题输出能力

- 如果面试官让我讲秒杀系统，我能按限流、缓存、削峰、库存正确性、一致性治理这条线讲吗
- 如果面试官问库存超卖，我能讲原子扣减、幂等、互斥和补偿，而不是只会答“加锁”吗
- 如果面试官问订单号为什么不用数据库自增，我能自然讲到分布式 ID 和分库分表吗
- 如果面试官问订单和库存不在一个库怎么办，我能讲最终一致链路吗
- 如果面试官问热点分片、扩容和迁移，我能接住吗

## 12. mock / oral 能力

- 我能用 1 分钟讲清楚这一章整体在解决什么问题
- 我能用 3 分钟讲一个完整高并发下单链路
- 我能在被连续追问 3 到 5 轮后还能接住
- 我不会一紧张就只会报 Redis、MQ、分库分表这些名词
- 我能把“为什么这样做”和“这样做的代价是什么”一起讲出来

## 13. 易翻车点

- 我不会说“分布式锁就能解决一致性问题”
- 我不会说“上了 MQ 问题就没了”
- 我不会说“分库分表一定比单库单表更高级”
- 我不会说“最终一致就是不一致也没关系”
- 我不会说“用了雪花算法就绝对没有风险”

## 14. 最后自测三问

- 如果面试官问“你怎么设计一个高并发下单系统”，我能讲 3 分钟吗
- 如果面试官继续追问“分库分表和一致性怎么处理”，我能讲 5 分钟吗
- 如果面试官最后追“这些方案的边界和代价是什么”，我还能往下接吗

## 15. 一句话收尾

> 这份 Checklist 的核心，不是确认你记住了多少术语，而是确认你有没有把这一章从“会看”推进到“能讲链路、能讲取舍、能讲边界”。
