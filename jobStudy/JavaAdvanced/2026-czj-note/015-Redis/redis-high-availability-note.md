# Redis 高可用与集群

## 1. 这份文档怎么用

Redis 基础题讲到后面，通常会自然追到：

- Redis 挂了怎么办
- 主从怎么理解
- 哨兵是什么
- 集群怎么理解

这一篇先把这条主线拉顺。

## 2. 为什么 Redis 也要讲高可用

因为 Redis 经常承接的是：

- 热点数据
- 高并发流量
- 核心缓存

如果 Redis 整体不可用，数据库可能会瞬间承压。

所以 Redis 不是“能用就行”，还得考虑高可用。

## 3. 主从复制怎么理解

最基础的高可用思路就是：

- 一个主节点
- 一个或多个从节点

主节点负责写，从节点可以承担一部分读，或者作为备份节点。

### 面试里怎么说

> Redis 主从复制可以理解成主节点负责核心写操作，从节点同步主节点数据，用来做数据备份、读扩展和故障恢复基础。

## 4. 主从复制的意义

### 4.1 数据冗余

有多个副本，不至于一台机器挂了数据就完全没了。

### 4.2 读扩展

有些场景下可以把部分读请求分摊到从节点。

### 4.3 故障恢复基础

后续哨兵选主，也得建立在主从复制基础上。

## 5. 哨兵是什么

哨兵可以先粗暴理解成：

- 监控 Redis 主从状态
- 在主节点异常时做故障转移

它的价值不在于存数据，而在于：

- 监控
- 通知
- 自动切换主节点

### 面试里怎么说

> 哨兵模式本质上是给 Redis 主从架构补上故障检测和自动故障转移能力，让 Redis 在主节点出问题时能更快完成主从切换。

## 6. Redis Cluster 怎么理解

如果数据量和请求量继续变大，光靠单主多从可能还不够。

这时候就会引出集群。

集群最关键的理解是：

- 不只是做高可用
- 还做数据分片

也就是说，它解决的不只是“机器挂了怎么办”，还包括：

- 单机容量不够怎么办
- 单机压力太大怎么办

## 7. 主从、哨兵、集群怎么区分

这三个很容易混。

### 7.1 主从

重点是：

- 数据复制
- 读写分离基础

### 7.2 哨兵

重点是：

- 监控
- 自动故障转移

### 7.3 集群

重点是：

- 分片
- 高可用
- 横向扩展

## 8. 面试里一个很稳的总答法

> Redis 的高可用可以按三层理解。第一层是主从复制，解决副本和读扩展基础；第二层是哨兵，解决主节点故障检测和自动切换；第三层是集群，不只是高可用，还进一步解决数据分片和横向扩展问题。

## 9. 一句话收尾

> Redis 高可用题最怕把主从、哨兵、集群混成一团，记住它们分别解决“复制”“切换”“分片扩展”就够了。
