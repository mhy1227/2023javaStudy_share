# Redis 真实面试坑题

## 1. 这篇文档怎么用

这一篇不主打讲基础定义，而是专门整理：

- 面试官容易追着问的坑点
- 你一不小心就会答绝对、答满、答错边界的问题

建议把它当成“防翻车题单”。

## 2. 坑题 1：Redis 这么快，那为什么不直接替代 MySQL

### 错误答法

- Redis 更快，所以完全可以代替数据库

### 更稳的答法

> Redis 和 MySQL 解决的问题不一样。Redis 更适合承接热点读、临时状态、排行榜、分布式协调这类高性能场景；MySQL 更适合做持久化、事务、一致性和复杂查询。项目里通常是配合关系，不是完全替代关系。

## 3. 坑题 2：Redis 分布式锁是不是绝对安全

### 错误答法

- 是，Redis 锁已经能保证分布式场景完全没问题

### 更稳的答法

> Redis 锁能解决很多业务互斥问题，但不是绝对强一致锁。它需要关注原子加锁、过期时间、误删、业务超时、主从切换等风险，很多时候还要结合业务幂等和数据库约束一起兜底。

## 4. 坑题 3：缓存一致性能不能做到 100% 强一致

### 错误答法

- 能，只要先更新数据库再删缓存就行

### 更稳的答法

> 工程上通常不是追求绝对零偏差，而是尽量缩小不一致窗口。先更新数据库再删缓存是更常见的实践，但仍有删除失败、并发回填等风险，所以还会结合延时双删、重试、MQ 或补偿机制。

## 5. 坑题 4：有了哨兵是不是就不会丢数据

### 错误答法

- 对，有哨兵就自动切主，不会出问题

### 更稳的答法

> 哨兵主要解决故障检测和自动切主，不代表绝对不丢数据。主从复制本身有延迟，切主窗口内仍然可能存在短暂不一致或少量数据丢失风险。

## 6. 坑题 5：Redis Cluster 就是一致性哈希吗

### 错误答法

- 对，Cluster 就是一致性哈希

### 更稳的答法

> Redis Cluster 更直接的理解是槽位分片机制。它和一致性哈希都在解决分布式数据分布问题，但 Cluster 面试里更建议按 `16384` 槽位、节点负责槽位范围、槽位迁移这条线来讲。

## 7. 坑题 6：缓存穿透、击穿、雪崩是不是一个东西

### 错误答法

- 都是缓存没了打到数据库

### 更稳的答法

> 三者结果上都可能冲击数据库，但本质不一样。穿透是查不存在数据，击穿是热点 key 失效，雪崩是大面积失效或 Redis 整体不可用。面试里一定要把问题成因区分开。

## 8. 坑题 7：HyperLogLog 能不能拿来做精确去重

### 错误答法

- 能，反正就是统计人数

### 更稳的答法

> HyperLogLog 适合做近似基数统计，比如 UV，它的优势是空间小，代价是结果不是绝对精确。如果业务必须精确去重，就不能只靠它。

## 9. 坑题 8：Redis 能不能直接当消息队列

### 错误答法

- 能，Redis 就是 MQ

### 更稳的答法

> Redis 可以承接轻量消息流场景，比如 `List` 或 `Stream`，但如果业务对可靠性、重试、顺序、堆积处理要求高，通常还是专业 MQ 更合适。

## 10. 坑题 9：只要用了 Redis，数据库压力就一定小很多

### 错误答法

- 对，Redis 一上就能彻底解决数据库瓶颈

### 更稳的答法

> Redis 确实能显著缓解热点读压力，但如果缓存设计不好，像缓存击穿、雪崩、热 key、失效错峰没做好，数据库反而可能在异常流量下被打得更狠。

## 11. 坑题 10：为什么不把所有数据都放 Redis

### 错误答法

- 因为没必要

### 更稳的答法

> Redis 更适合高频访问、热点、临时状态和高性能读写场景，不适合把所有业务数据都当最终真相放进去。内存成本、持久化能力、一致性要求、复杂查询能力都会限制它的适用边界。

## 12. 坑题 11：分布式锁过期时间设大一点是不是就没问题了

### 错误答法

- 对，TTL 设长一点就安全了

### 更稳的答法

> TTL 设太短会导致业务没执行完锁先过期，TTL 设太长又会让故障恢复变慢，所以单纯把过期时间拉长不是根本解法。更合理的做法是结合业务耗时评估和自动续期机制。

## 13. 坑题 12：Redis 高可用是不是就等于主从

### 错误答法

- 对，主从就是高可用

### 更稳的答法

> 主从是高可用基础，但它只解决副本和读扩展问题；真正自动故障转移通常还要靠哨兵，而更大规模的容量和分片问题则要靠 Cluster。

## 14. 一个总的答题提醒

Redis 面试最容易翻车的地方，不是不会背，而是：

- 把相对方案说成绝对方案
- 把工程折中说成银弹
- 把基础能力说成无边界能力

更稳的口径通常是：

- 先说它解决什么
- 再说它不解决什么
- 最后说工程上怎么兜底

## 15. 一句话收尾

> Redis 面试坑题的核心，不是知识点有多难，而是你能不能把边界、代价和工程取舍讲出来。
