# Redis 最终 Checklist

## 1. 这份清单怎么用

这不是让你从头学 Redis 的。

它是给你在面试前最后一轮快速过稿用的。

你只需要拿这份清单自查：

- 这些点我能不能自己讲出来
- 哪些点我只是看着眼熟，但一张嘴就空

## 2. 基础定位

- 我能说清楚 Redis 是什么，不会只答“内存数据库”
- 我能说清楚 Redis 在系统里主要承担什么角色
- 我能区分 Redis 和 MySQL 是配合关系，不是简单替代关系
- 我能说清楚 Redis 为什么快，至少会提到内存、单线程模型、IO 多路复用、数据结构设计

## 3. 数据结构

- 我能区分 `String`、`Hash`、`List`、`Set`、`ZSet`
- 我能把常见场景和结构挂起来
- 我知道排行榜常想到 `ZSet`
- 我知道对象属性聚合常想到 `Hash`
- 我知道消息流不只会答 `List`，还知道 `Stream`

## 4. 持久化

- 我能说清楚 `RDB` 和 `AOF` 的区别
- 我知道两者分别更偏性能、恢复、数据完整性哪一侧
- 我知道混合持久化是个什么口径
- 我不会把 Redis 讲成“只要开了持久化就和数据库一样稳”

## 5. 缓存高频问题

- 我能区分缓存穿透、击穿、雪崩
- 我不会把三者混成一句“缓存没命中”
- 我知道穿透常见手段：空值缓存、布隆过滤器、参数校验
- 我知道击穿常见手段：热点保护、逻辑过期、互斥重建
- 我知道雪崩常见手段：错峰过期、多级缓存、限流降级、高可用

## 6. 缓存一致性

- 我知道为什么缓存和数据库天然容易不一致
- 我知道更常见的口径是“先更新数据库，再删缓存”
- 我知道这样做也不是绝对完美
- 我知道延时双删是干什么的
- 我知道删缓存失败需要补偿，不是删一次就算完
- 我知道可以继续往 MQ / 异步重试 / binlog 补偿方向讲

## 7. 过期与淘汰

- 我知道 Redis key 为什么会过期
- 我知道过期和淘汰不是一回事
- 我知道内存不够时会涉及淘汰策略
- 我知道大概能说出 `LRU / LFU` 这类口径

## 8. 热 key 与 big key

- 我知道热 key 风险不只是“访问量大”，而是容易把单点打爆
- 我知道 hot key 常配合本地缓存、多级缓存、限流、预热来讲
- 我知道 big key 风险不仅是内存大，还可能影响删除、迁移、阻塞

## 9. 高可用

- 我能区分主从、哨兵、集群
- 我知道主从解决副本和读扩展
- 我知道哨兵解决监控和自动切主
- 我知道集群解决分片、扩容和高可用
- 我知道主从复制不是强一致

## 10. 主从复制进阶

- 我知道全量复制和增量复制
- 我知道 `psync` 是干什么的
- 我知道 `offset` 和 `repl_backlog_buffer` 的口径
- 我知道主从复制有延迟
- 我知道读写分离可能读到旧数据
- 我知道脑裂是网络分区或故障判定窗口下的风险场景

## 11. Cluster 深一点

- 我知道 `16384` 槽位怎么理解
- 我知道 key 是先映射到槽，再由槽位归属节点
- 我知道 Cluster 不只是“多主多从”，还涉及分片与迁移
- 我知道 `MOVED / ASK` 是路由与迁移相关口径

## 12. 分布式锁

- 我知道为什么本地锁不够
- 我知道为什么加锁要带过期时间
- 我知道为什么 `setnx` 和 `expire` 不能拆开
- 我知道为什么锁 value 要唯一
- 我知道为什么释放锁常用 Lua
- 我知道锁过期但业务没跑完是一个真实风险
- 我知道可以讲自动续期 / 看门狗
- 我知道主从切换下 Redis 锁也有边界

## 13. 特殊结构与特殊场景

- 我知道 `Bitmap` 常用于签到、布尔状态压缩
- 我知道 `HyperLogLog` 常用于 UV 近似统计
- 我知道 `GEO` 常用于附近门店、附近骑手
- 我知道 `Stream` 更适合轻量消息流场景
- 我知道 `Lua` 常用于把多步操作做成 Redis 端原子执行

## 14. 项目题

- 我能说清楚项目里为什么要引入 Redis
- 我能说清楚 Redis 在项目里放在哪一层
- 我能说清楚 Redis 帮系统扛了什么压力
- 我能说清楚 Redis 带来了哪些新风险
- 我能说清楚如果 Redis 挂了，业务怎么兜底

## 15. 易翻车点

- 我不会说“Redis 可以完全替代 MySQL”
- 我不会说“Redis 分布式锁绝对安全”
- 我不会说“哨兵一上就不会丢数据”
- 我不会说“先更新数据库再删缓存就 100% 一致”
- 我不会把 Redis 能做 MQ 讲成 Redis 就等于 MQ

## 16. 最后自测三问

- 如果面试官问“Redis 为什么快”，我能讲 1 分钟吗
- 如果面试官问“你项目里 Redis 怎么用”，我能讲 3 分钟吗
- 如果面试官继续追问“缓存一致性、分布式锁、高可用边界”，我能往下接吗

## 17. 一句话收尾

> 这份 Checklist 的核心不是看你记了多少条，而是确认你有没有把 Redis 从“背概念”推进到“能讲系统、能讲边界、能讲项目”。
