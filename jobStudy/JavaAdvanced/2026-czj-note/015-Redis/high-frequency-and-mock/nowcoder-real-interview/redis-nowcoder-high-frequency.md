# Redis 牛客高频面经题

## 1. 这份文档怎么用

这一篇不是系统讲 Redis 原理，而是把牛客公开面经里高频反复出现的 Redis 问法压成题单。

适合：

- 面试前快速看 Redis 真实问法
- 判断自己 Redis 哪一块最薄弱
- 决定接下来该优先补哪类题

## 2. 第一梯队：几乎必看

- Redis 有哪些数据类型？分别适合什么场景？
- Redis 的持久化方案有哪些？RDB 和 AOF 有什么区别？
- 怎么保证缓存和数据库的一致性？
- 缓存穿透、缓存击穿、缓存雪崩分别是什么？怎么解决？
- Redis 为什么这么快？
- Redis 一般有哪些使用场景？

## 3. 第二梯队：非常容易接着追问

- Redis 的内存淘汰策略有哪些？
- Redis 的过期删除策略有哪些？
- Redis 分布式锁怎么实现？有哪些坑？
- Redis 主从、哨兵、集群分别是什么？区别是什么？
- MySQL 和 Redis 有什么区别？为什么项目里两个都要用？
- 项目里你是怎么用 Redis 的？

## 4. 第三梯队：细节追问题

- Redis 为什么早期选择单线程？
- Redis 6 之后是不是变成多线程了？
- IO 多路复用是什么？和 Redis 的高性能有什么关系？
- Redis 事务怎么理解？能不能保证强一致？
- pipeline、Lua 脚本为什么会被问到？
- RedLock 是什么？为什么它有争议？

## 5. 第四梯队：场景和故障类

- hotkey 是什么？怎么处理？
- bigkey 是什么？会带来什么问题？
- Redis 变慢的原因有哪些？
- 键过期、大量淘汰、内存打满时会发生什么？
- 分布式锁误删、锁过期、锁续期怎么办？

## 6. 牛客公开面经里最常出现的题型分类

### 基础认知题

- Redis 是什么？
- Redis 和 Memcached 的区别？
- Redis 和 MySQL 的区别？

### 原理题

- Redis 为什么快？
- Redis 单线程怎么理解？
- Redis 为什么用 IO 多路复用？

### 数据结构题

- 五种基本数据类型分别适合什么场景？
- List 和 Sorted Set 有什么区别？
- Hash 为什么适合存对象？

### 持久化和高可用题

- RDB 和 AOF 区别是什么？
- 主从、哨兵、集群怎么选？
- 宕机恢复时会发生什么？

### 缓存问题题

- 双写一致性怎么保证？
- 穿透、击穿、雪崩怎么区分？
- 热点 key 怎么治理？

### 分布式锁题

- 用 Redis 做分布式锁怎么实现？
- setnx 为什么不够？
- 为什么要带过期时间？
- 为什么释放锁常用 Lua 脚本？

### 项目表达题

- 你项目里用 Redis 做了什么？
- 你为什么选 Redis 而不是只用 MySQL？
- Redis 出过什么问题？你怎么排查的？

## 7. 复习优先级建议

### 如果时间很紧

先看这 6 个：

1. 数据类型
2. RDB vs AOF
3. 为什么快
4. 缓存穿透 / 击穿 / 雪崩
5. 缓存一致性
6. 分布式锁

### 如果还能再往下一层

再补这 5 个：

1. 淘汰策略
2. 过期删除策略
3. 主从 / 哨兵 / 集群
4. hotkey / bigkey
5. 项目里怎么用 Redis
