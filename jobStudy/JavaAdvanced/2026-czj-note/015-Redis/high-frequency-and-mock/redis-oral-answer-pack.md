# Redis 口语答法包

## 1. 这份文档怎么用

这一篇专门解决一个问题：

- 明明知道 Redis 这些点，但一开口就答散了

所以这里不追求特别全，而是尽量压成面试里能直接说出口的话。

## 2. 题 1：Redis 是什么

### 口语版

> Redis 是一个基于内存的高性能 Key-Value 存储系统，实际项目里最常用来做缓存，也会用于计数器、排行榜、分布式锁和一些临时状态数据存储。它最大的价值是承接高频访问和热点数据，减轻数据库压力。

## 3. 题 2：Redis 为什么快

### 口语版

> Redis 快，核心还是因为它主要基于内存读写，其次它对常见数据结构做了专门优化。另外命令执行这一层是单线程，避免了很多锁竞争和线程切换开销，再加上 IO 多路复用，所以整体吞吐能力会比较强。

## 4. 题 3：Redis 常见数据结构有哪些

### 口语版

> 常见的有 `String`、`Hash`、`List`、`Set`、`ZSet`。我一般会按场景记，简单值和计数器常用 `String`，对象字段常用 `Hash`，去重集合用 `Set`，排行榜更常联想到 `ZSet`。

## 5. 题 4：为什么排行榜一般用 ZSet

### 口语版

> 因为 `ZSet` 既能存成员，又能给成员绑定 score，还能按 score 排序，所以特别适合积分榜、热度榜这种既要存数据又要排序的场景。

## 6. 题 5：Redis 持久化怎么讲

### 口语版

> Redis 虽然以内存为主，但为了降低数据丢失风险，还是要讲持久化。面试里通常讲 `RDB` 和 `AOF`，前者偏快照，恢复通常更快；后者偏写命令日志，数据安全性通常更好一些。

## 7. 题 6：缓存穿透、击穿、雪崩怎么区分

### 口语版

> 我一般这么区分：穿透是查根本不存在的数据；击穿是热点 key 失效瞬间，大量请求一起回源；雪崩是大面积缓存失效或者 Redis 整体不可用。它们本质上都是在防止异常流量把数据库直接打穿。

## 8. 题 7：缓存和数据库怎么保持一致

### 口语版

> 我不会把它讲成绝对强一致，而是尽量缩小不一致窗口。比较常见的口径是先更新数据库，再删除缓存；如果业务更敏感，还可以结合延迟双删、MQ 或补偿机制进一步降低风险。

## 9. 题 8：Redis 分布式锁怎么讲

### 口语版

> Redis 分布式锁的核心是通过锁 key 做跨节点互斥控制，因为本地锁只能锁住当前 JVM。一般还会给锁设置过期时间防止死锁，同时加唯一标识，保证释放锁时不会误删别人的锁。

## 10. 题 9：热点 key 怎么处理

### 口语版

> 热点 key 我会优先考虑不让它轻易失效，比如逻辑过期或预热；如果流量特别高，再考虑本地缓存、多级缓存、限流和降级，尽量别让热点请求全集中到 Redis 上。

## 11. 题 10：Redis 高可用怎么讲

### 口语版

> 我会分三层讲：主从复制解决副本和读扩展基础；哨兵解决主节点故障检测和自动切换；集群不只是高可用，还进一步解决数据分片和横向扩展问题。

## 12. 一句话收尾

> Redis 口语题最关键的不是说得多，而是每一题都能把“场景、核心原理、风险点”三件事说清楚。
