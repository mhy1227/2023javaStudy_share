# Redis 复习清单

## 1. 这份清单怎么用

这一篇适合面试前 5 到 10 分钟快速过一遍。

用法很简单：

- 会讲，打勾
- 讲不顺，回对应文档补
- 能讲 30 秒，再尝试讲 1 分钟版本

## 2. 基础清单

- 我能解释 Redis 是什么
- 我能解释 Redis 为什么快
- 我知道 Redis 的“单线程”主要是指命令执行层
- 我能说出 Redis 和 MySQL 的分工

## 3. 数据结构清单

- 我能说出 `String`、`Hash`、`List`、`Set`、`ZSet`
- 我能讲 `Hash` 和 `String` 存对象时的区别
- 我能讲为什么排行榜常用 `ZSet`
- 我能根据场景做基本选型

## 4. 持久化清单

- 我能讲 Redis 为什么要持久化
- 我能讲 `RDB`
- 我能讲 `AOF`
- 我能讲 `RDB` 和 `AOF` 的核心差异
- 我不会把 Redis 讲成“绝对不会丢数据”

## 5. 缓存问题清单

- 我能区分穿透、击穿、雪崩
- 我能讲常见解决思路
- 我能讲缓存一致性的主流口径
- 我不会把一致性讲成绝对强一致

## 6. 锁与风险清单

- 我能讲为什么需要分布式锁
- 我能讲为什么过期时间重要
- 我能讲为什么释放锁不能直接删
- 我能讲热点 key
- 我能讲大 key

## 7. 高可用清单

- 我能讲主从复制
- 我能讲哨兵
- 我能讲集群
- 我不会把主从、哨兵、集群混成一团

## 8. 项目题清单

- 我能回答项目里 Redis 怎么用
- 我能回答为什么不直接查 MySQL
- 我能回答 Redis 挂了怎么办
- 我能回答热点 key 怎么处理
- 我能回答缓存和数据库怎么配合

## 9. 最后自测

如果下面 4 题你都能在 30 秒到 1 分钟内讲顺，这一章就已经够用了：

1. Redis 为什么快
2. 缓存穿透、击穿、雪崩怎么区分
3. Redis 分布式锁怎么理解
4. Redis 高可用怎么理解

## 10. 一句话收尾

> Redis 复习到最后，目标不是知道更多命令，而是能把高频题讲得稳定、像做过项目的人。
