# Redis 真实面经混合题

## 1. 这份文档怎么用

这一篇不是标准八股清单，而是更贴近真实面经里“Redis 被夹在项目和中间件问题里一起问”的那种感觉。

它适合：

- 模拟真实一面、二面
- 看看 Redis 会怎么和 MySQL、MQ、高并发、项目选型混着问
- 训练自己不要一听到 Redis 就只会背“为什么快”

## 2. 混合题 1

### 问题

你项目里 Redis 主要用来做什么？为什么不用 MySQL 直接扛？

### 回答提示

- 先说 Redis 在你系统里的位置
- 再说高频读、热点流量、数据库减压
- 最后补一句 Redis 和 MySQL 的分工

## 3. 混合题 2

### 问题

如果缓存和数据库不一致了，你们怎么处理？

### 回答提示

- 不要讲成绝对强一致
- 强调缩小不一致窗口
- 常见口径：先更数据库，再删缓存

## 4. 混合题 3

### 问题

如果删缓存失败怎么办？有没有后续补偿思路？

### 回答提示

- 可以顺手带延迟双删
- 可以带 MQ、重试、补偿任务
- 重点是表达“我知道失败路径”

## 5. 混合题 4

### 问题

如果某个商品详情页非常热，Redis 这里可能会出什么问题？

### 回答提示

- 热点 key
- 热点 key 失效导致击穿
- 本地缓存、多级缓存、预热、限流

## 6. 混合题 5

### 问题

Redis 挂了怎么办？数据库会不会被打穿？

### 回答提示

- Redis 高可用
- 降级限流
- 数据库保护

## 7. 混合题 6

### 问题

你们为什么这里选择 Redis 分布式锁，而不是数据库锁？

### 回答提示

- 多实例部署下跨节点互斥
- 说明 Redis 锁更偏高性能协调
- 也要承认 Redis 锁有风险边界

## 8. 混合题 7

### 问题

Redis 分布式锁最大的风险是什么？

### 回答提示

- 锁过期但业务未完成
- 误删别人的锁
- Redis 自身高可用风险

## 9. 混合题 8

### 问题

你们有没有遇到大 key 或热点 key？这两个有啥区别？

### 回答提示

- 热点 key 看访问集中
- 大 key 看数据规模过大
- 两者都可能影响性能，但不是一个维度

## 10. 混合题 9

### 问题

Redis 和 MQ 的边界怎么理解？什么时候该上 MQ，什么时候 Redis 就够了？

### 回答提示

- Redis 可以做轻量队列场景
- 正式消息可靠投递还是 MQ 更合适
- Redis 更偏缓存和热点流量承接

## 11. 混合题 10

### 问题

如果面试官问“Redis 为什么快”，你怎么避免答得太八股？

### 回答提示

- 先讲内存和数据结构
- 再讲命令执行层单线程和 IO 多路复用
- 最后带一句“快不代表没有边界”

## 12. 混合题 11

### 问题

为什么排行榜更常用 ZSet，而不是 MySQL 排序直接查？

### 回答提示

- 高频读
- 低延迟排序
- Redis 承接热点场景

## 13. 混合题 12

### 问题

你们有没有做缓存预热？为什么要做？

### 回答提示

- 活动前热点数据提前放入 Redis
- 避免首次流量全部打到数据库
- 顺手带击穿防护

## 14. 混合题 13

### 问题

Redis 的持久化和高可用，你在项目里更看重哪一块？

### 回答提示

- 看业务
- 有些场景更看数据恢复
- 有些场景更看缓存层别整体崩

## 15. 混合题 14

### 问题

为什么说 Redis 题经常不是单独问，而是和项目题混着问？

### 回答提示

- Redis 在项目里通常就是“系统中一层”
- 面试官更关心你怎么用、为什么用、出问题怎么办

## 16. 一句话收尾

> 真实面经里的 Redis，往往不是一道孤立八股题，而是项目选型、缓存设计、高并发风险和中间件边界一起追着问。
