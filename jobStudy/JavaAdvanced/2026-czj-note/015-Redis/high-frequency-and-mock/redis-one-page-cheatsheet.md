# Redis 一页速背

## 1. Redis 是什么

- 基于内存的高性能 Key-Value 存储系统
- 常见用途：缓存、计数器、排行榜、分布式锁、临时状态

一句话答法：

> Redis 主要用来承接高频访问、热点数据和低延迟读写场景。

## 2. Redis 为什么快

- 内存读写
- 数据结构高效
- 命令执行层面单线程
- IO 多路复用

一句话答法：

> Redis 快的核心是内存、高效数据结构、单线程避免锁竞争，以及 IO 模型的高效处理能力。

## 3. 常见数据结构

- `String`：简单值、计数器、序列化对象
- `Hash`：对象字段
- `List`：轻量队列、最新列表
- `Set`：去重集合
- `ZSet`：排行榜、热度排序

## 4. 持久化

- `RDB`：快照
- `AOF`：写命令日志

一句话区分：

> `RDB` 偏恢复快，`AOF` 偏数据更不容易丢。

## 5. 缓存三兄弟

- 穿透：查不存在的数据
- 击穿：热点 key 失效
- 雪崩：大面积失效或 Redis 整体不可用

一句话答法：

> 穿透看“不存在”，击穿看“热点失效”，雪崩看“大面积崩”。

## 6. 缓存一致性

常见口径：

- 先更新数据库，再删除缓存
- 结合延迟双删、MQ、补偿机制

重点：

- 不追求绝对强一致
- 重点是缩小不一致窗口

## 7. 分布式锁

核心点：

- 多实例部署下，本地锁不够
- Redis 锁 key 做互斥
- 带过期时间防死锁
- 带唯一标识防误删

## 8. 过期与淘汰

- 过期：key 到时间了怎么办
- 淘汰：内存不够时删谁

一句话区分：

> 过期看时间，淘汰看内存。

## 9. 高可用

- 主从：复制
- 哨兵：自动切换
- 集群：分片 + 高可用

一句话答法：

> 主从解决副本，哨兵解决切主，集群解决分片扩展。

## 10. 热点 key / 大 key

- 热点 key：访问过于集中
- 大 key：单个 key 数据量太大

一句话区分：

> 热点 key 看流量集中，大 key 看数据规模过大。

## 11. 项目题总答法

> Redis 在项目里主要是承接高频读和热点数据，降低数据库压力。面试里通常会继续追问缓存一致性、热点 key、分布式锁和高可用，所以答 Redis 不能只答“快”，还得把风险和兜底讲出来。
