# Redis 高频问答

## 1. 这份文档怎么用

这一篇把 Redis 里最常被问的题压成问答形式。

适合：

- 自己抽问自己
- 面试前快速过题
- 检查自己卡在定义、原理还是项目表达

## 2. 高频题 1

### 问题

Redis 是什么？

### 回答骨架

> Redis 是一个基于内存的高性能 Key-Value 存储系统，常见用途包括缓存、计数器、排行榜、分布式锁和临时状态存储。

## 3. 高频题 2

### 问题

Redis 为什么快？

### 回答骨架

> Redis 快，首先因为它主要基于内存读写；其次它对常见数据结构做了优化；再加上命令执行层面是单线程，避免了很多锁竞争和上下文切换开销，同时结合 IO 多路复用，整体吞吐能力会很强。

## 4. 高频题 3

### 问题

Redis 为什么用单线程？

### 回答骨架

> Redis 命令执行层面采用单线程，可以避免多线程带来的锁竞争和线程切换开销。因为 Redis 本身以内存操作为主，瓶颈很多时候不在 CPU 计算，而在网络和系统 IO，所以这种模型反而很高效。

## 5. 高频题 4

### 问题

Redis 常见数据结构有哪些？

### 回答骨架

> 面试里最常讲的是 `String`、`Hash`、`List`、`Set`、`ZSet`。不同结构更对应不同业务场景，比如对象字段适合 `Hash`，排行榜适合 `ZSet`。

## 6. 高频题 5

### 问题

为什么排行榜一般用 `ZSet`？

### 回答骨架

> 因为 `ZSet` 既能存成员，又能给成员绑定 score，还能按 score 排序，所以特别适合做积分榜、热度榜这种既要存数据又要排序的场景。

## 7. 高频题 6

### 问题

Redis 持久化怎么理解？

### 回答骨架

> Redis 以内存为主，所以需要持久化来降低数据丢失风险。面试里通常讲 `RDB` 和 `AOF`，前者偏快照，后者偏操作日志。

## 8. 高频题 7

### 问题

`RDB` 和 `AOF` 有什么区别？

### 回答骨架

> `RDB` 更像阶段性全量快照，恢复速度通常更快，但数据丢失窗口相对更大；`AOF` 更像写命令日志，数据安全性更好，但文件体积和恢复成本通常更高。

## 9. 高频题 8

### 问题

什么是缓存穿透？

### 回答骨架

> 缓存穿透指的是请求查询的数据本身不存在，缓存里没有，数据库里也没有，结果每次请求都直接打到数据库。常见处理方式有缓存空值、布隆过滤器和参数校验。

## 10. 高频题 9

### 问题

什么是缓存击穿？

### 回答骨架

> 缓存击穿一般是热点 key 失效瞬间，大量并发请求同时回源数据库，所以常见思路是给热点 key 做保护，比如互斥重建、逻辑过期或者预热。

## 11. 高频题 10

### 问题

什么是缓存雪崩？

### 回答骨架

> 缓存雪崩强调的是大面积缓存失效，或者 Redis 整体不可用，导致大量请求同时打到数据库。常见应对方式包括过期时间打散、限流降级和提升缓存高可用能力。

## 12. 高频题 11

### 问题

缓存和数据库怎么保持一致？

### 回答骨架

> 缓存一致性通常不是追求绝对强一致，而是尽量缩小不一致窗口。常见口径是先更新数据库，再删除缓存；如果业务复杂，可能再结合延迟双删、MQ 或补偿机制。

## 13. 高频题 12

### 问题

Redis 分布式锁怎么理解？

### 回答骨架

> Redis 分布式锁的核心是用锁 key 做互斥控制，并设置过期时间防止死锁。为了避免误删，通常还会给锁加唯一标识，释放时校验“是不是自己加的锁”。

## 14. 高频题 13

### 问题

为什么本地锁不能替代分布式锁？

### 回答骨架

> 因为本地锁只能锁住当前 JVM 内的线程，多实例部署下它控制不了其他机器上的并发访问，所以跨节点互斥时才需要分布式锁。

## 15. 高频题 14

### 问题

什么是热点 key，怎么处理？

### 回答骨架

> 热点 key 指访问量特别集中的 key，风险是局部压力过高甚至引发击穿。常见处理方式有热点预热、多级缓存、逻辑过期和限流降级。

## 16. 高频题 15

### 问题

什么是大 key，为什么危险？

### 回答骨架

> 大 key 指单个 key 对应的数据量特别大，问题主要在于网络传输开销大、操作耗时长，甚至可能拖慢 Redis 整体性能，所以通常要考虑拆分、分页或分片处理。

## 17. 高频题 16

### 问题

Redis 高可用怎么理解？

### 回答骨架

> 可以分三层理解：主从复制解决副本和读扩展基础；哨兵解决主节点故障检测和自动切换；集群进一步解决数据分片和横向扩展问题。

## 18. 一句话收尾

> Redis 高频题的核心，不是背名词，而是把“原理、场景、风险、解决思路”讲顺。
