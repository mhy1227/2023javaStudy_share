# Redis 缓存穿透、击穿、雪崩对比

## 1. 这份文档怎么用

这一篇专门解决 Redis 里最容易混的三个问题：

- 缓存穿透
- 缓存击穿
- 缓存雪崩

很多人不是不会，而是：

- 名字记住了
- 一到面试就讲混了
- 解决办法也容易串掉

所以这篇就是专门做“横向对比”。

## 2. 先记住最短区分

- 穿透：查的是不存在的数据
- 击穿：热点 key 失效了
- 雪崩：一大批 key 同时失效，或者 Redis 整体挂了

如果只背一句，就背这一句。

## 3. 先分别理解一下

### 3.1 缓存穿透

缓存穿透的核心是：

- 请求查询的数据本来就不存在
- 缓存没有
- 数据库也没有
- 结果每次请求都会直接打到数据库

这类问题最典型的特点是：

- 访问的是“无效数据”

### 3.2 缓存击穿

缓存击穿的核心是：

- 某个 key 本来是热点
- 结果这个热点 key 突然失效
- 大量并发请求同时回源数据库

这类问题最典型的特点是：

- 出问题的是“热点 key”

### 3.3 缓存雪崩

缓存雪崩的核心是：

- 不是一个 key 出问题
- 而是一大片 key 同时失效
- 或者 Redis 整体不可用
- 导致大量请求一下子全压到数据库

这类问题最典型的特点是：

- 出问题的是“整体缓存层”

## 4. 三者最核心的区别

### 4.1 从“访问对象”看

- 穿透：访问的是不存在的数据
- 击穿：访问的是存在的热点数据，但热点缓存刚失效
- 雪崩：不是一个热点，而是大量缓存一起出问题

### 4.2 从“影响范围”看

- 穿透：单次无效请求不断穿过缓存
- 击穿：集中在某一个热点 key
- 雪崩：影响范围最大，可能是很多 key，甚至整个 Redis 层

### 4.3 从“风险来源”看

- 穿透：问题来源是无效请求
- 击穿：问题来源是热点 key 过期
- 雪崩：问题来源是大面积过期或缓存整体不可用

## 5. 对应解决办法怎么记

### 5.1 缓存穿透的解决办法

常见方案：

- 缓存空值
- 布隆过滤器
- 参数校验

#### 怎么理解

核心思路就是：

- 尽量在进入数据库之前，把这些“本来就不存在”的请求挡住

#### 面试口语版

> 缓存穿透本质是查不存在的数据，所以常见思路是缓存空值、用布隆过滤器做前置拦截，再加上接口层的参数校验，尽量别让无效请求每次都打到数据库。

### 5.2 缓存击穿的解决办法

常见方案：

- 热点 key 不轻易过期
- 逻辑过期
- 互斥锁
- 热点预热

#### 怎么理解

核心思路就是：

- 重点保护那个“特别热”的 key
- 不要让它一失效就把数据库暴露出来

#### 面试口语版

> 缓存击穿一般发生在热点 key 失效瞬间，所以核心是保护热点数据，比如不让热点 key 轻易过期，或者用逻辑过期、互斥重建和预热等方式避免大量请求同时回源数据库。

### 5.3 缓存雪崩的解决办法

常见方案：

- 过期时间加随机值
- Redis 高可用
- 限流和降级
- 多级缓存

#### 怎么理解

核心思路就是：

- 不要让很多 key 在同一时间一起失效
- 也不要让 Redis 一挂就直接把数据库暴露出去

#### 面试口语版

> 缓存雪崩强调的是大面积缓存失效或者 Redis 整体不可用，所以要从两个方向兜底：一是通过随机过期避免集中失效，二是通过高可用、限流降级和多级缓存减少对数据库的冲击。

## 6. 一个表格式理解

### 6.1 问题本质

- 穿透：查无效数据
- 击穿：热点 key 失效
- 雪崩：大面积缓存失效或缓存层挂掉

### 6.2 影响范围

- 穿透：通常是单类无效请求持续穿透
- 击穿：集中在单个或少量热点 key
- 雪崩：影响范围最大，可能波及整个系统

### 6.3 常见手段

- 穿透：空值缓存、布隆过滤器、参数校验
- 击穿：热点保护、逻辑过期、互斥锁、预热
- 雪崩：随机过期、高可用、限流降级、多级缓存

## 7. 面试里一个很稳的总答法

> Redis 常见缓存问题主要有穿透、击穿、雪崩。穿透是查不存在的数据，所以重点是把无效请求挡住；击穿是热点 key 失效，所以重点是保护热点数据别同时回源；雪崩是大面积失效或者 Redis 整体不可用，所以重点是避免集中失效，并通过高可用、限流和降级做系统级兜底。

## 8. 最容易踩的坑

### 坑 1

把击穿和雪崩混成一类。

记住：

- 击穿更偏“单点热点”
- 雪崩更偏“整体崩”

### 坑 2

把穿透也理解成“缓存过期”。

其实穿透和过期没关系。

它是因为：

- 这个数据压根不存在

### 坑 3

解决方案串了。

比如：

- 拿布隆过滤器去解释击穿
- 拿互斥锁去解释雪崩

这都会显得很乱。

## 9. 一句话收尾

> 穿透、击穿、雪崩这三个问题，最关键的不是背术语，而是把“问题对象、影响范围、解决重点”区分清楚。
