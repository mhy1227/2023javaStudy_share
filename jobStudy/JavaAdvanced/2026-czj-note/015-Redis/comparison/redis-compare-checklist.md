# Redis 对比题复习清单

## 1. 这份清单怎么用

这一篇适合在你看完 Redis 对比专题之后，做最后一轮自检。

用法很简单：

- 会讲，打勾
- 讲不顺，回前面的对比文档补
- 尽量把每个点都压成 30 秒左右的口语回答

## 2. String vs Hash

- 我能讲清 `String` 更适合简单值、token、验证码、整体对象
- 我能讲清 `Hash` 更适合结构化对象和按字段更新
- 我不会把 `String` 和 `Hash` 说成只是“一个字符串、一个哈希表”
- 我能结合项目说出自己怎么选

## 3. RDB vs AOF

- 我能讲清 `RDB` 偏快照
- 我能讲清 `AOF` 偏写命令日志
- 我能讲清两者在数据丢失窗口和恢复速度上的差别
- 我不会把 Redis 说成“开了持久化就绝对不丢数据”

## 4. 主从 vs 哨兵 vs 集群

- 我能讲清主从解决复制和读扩展基础
- 我能讲清哨兵解决故障检测和自动切主
- 我能讲清集群除了高可用，还解决分片和横向扩展
- 我不会把三者混成“都是高可用方案”

## 5. 热点 key vs 大 key

- 我能讲清热点 key 的问题是访问量过于集中
- 我能讲清大 key 的问题是单个 key 数据规模过大
- 我能说出两者的风险点不同
- 我能说出两者常见处理思路

## 6. 本地锁 vs Redis 分布式锁

- 我能讲清本地锁只解决当前 JVM 内的互斥
- 我能讲清 Redis 分布式锁解决跨节点互斥
- 我能解释为什么多实例部署下本地锁不够
- 我不会把 Redis 分布式锁讲成“本地锁的高级版”

## 7. 缓存 vs MySQL

- 我能讲清 MySQL 更偏最终持久化和事务型数据
- 我能讲清 Redis 更偏热点数据承接和低延迟读取
- 我不会把 Redis 和 MySQL 说成互相替代
- 我能用“主存 + 加速层”的口径去讲

## 8. 穿透 vs 击穿 vs 雪崩

- 我能区分穿透是查不存在的数据
- 我能区分击穿是热点 key 失效
- 我能区分雪崩是大面积失效或 Redis 整体不可用
- 我不会把三者的解决办法串掉

## 9. 最后自测

如果下面这 5 题你都能讲顺，Redis 对比专题就基本够用了：

1. `String` 和 `Hash` 怎么选
2. `RDB` 和 `AOF` 怎么区分
3. 主从、哨兵、集群分别解决什么问题
4. 热点 key 和大 key 有什么区别
5. 穿透、击穿、雪崩怎么区分

## 10. 一句话收尾

> Redis 对比题复习到最后，目标不是知道更多名词，而是不会把相近概念讲混。
