# Redis 对比题口语版

## 1. 这份文档怎么用

这一篇是压缩版。

如果你已经看过主线和对比速记版，但想把它真正讲出口，就看这一篇。

## 2. String vs Hash

### 口语版

> `String` 更适合简单值、token、验证码或者整体序列化对象；`Hash` 更适合结构化对象，尤其是需要按字段更新的时候会更灵活。所以我一般按“有没有字段化修改需求”来选。

## 3. RDB vs AOF

### 口语版

> `RDB` 更像快照，恢复通常更快，但数据丢失窗口相对更大；`AOF` 更像写命令日志，数据安全性通常更好，但文件体积和恢复成本一般更高。

## 4. 主从 vs 哨兵 vs 集群

### 口语版

> 我一般分三层讲：主从解决复制，哨兵解决自动切主，集群解决分片和横向扩展。

## 5. 热点 key vs 大 key

### 口语版

> 热点 key 的问题是访问太集中，容易形成局部高并发压力；大 key 的问题是单个 key 太大，容易带来传输和操作开销，所以一个偏流量问题，一个偏数据规模问题。

## 6. 本地锁 vs Redis 分布式锁

### 口语版

> 本地锁解决的是单 JVM 内的并发互斥，Redis 分布式锁解决的是多实例、多节点下的跨机器互斥，所以本地锁不是不行，而是有边界。

## 7. Redis vs MySQL

### 口语版

> MySQL 更负责最终持久化数据，Redis 更负责高频访问和热点流量承接，它们在项目里更多是配合关系，而不是替代关系。

## 8. 穿透 vs 击穿 vs 雪崩

### 口语版

> 穿透是查不存在的数据，击穿是热点 key 失效，雪崩是大面积 key 同时失效或者 Redis 整体挂了。三者都可能把数据库打穿，但问题对象和处理重点不一样。

## 9. 一句话收尾

> Redis 对比口语题最稳的讲法，不是背定义，而是直接说“它们分别解决什么问题”。
