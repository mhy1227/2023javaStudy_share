# JVM 基础总览

## 1. 这份文档怎么用

这一篇是 `013-JVM` 的第一篇总览文档。

它的目标不是把所有 JVM 细节一次讲完，而是先把主线骨架立起来，让你知道：

1. JVM 是什么
2. JVM 为什么重要
3. JVM 面试最常从哪几条线展开
4. 后续应该按什么顺序继续复习

如果你后面再看 GC、类加载、JMM、OOM，这一篇就是总入口。

## 2. 先记住最重要的 5 句

先把下面这 5 句记住：

1. JVM 是 Java 程序运行的虚拟机环境。
2. JVM 最常被问的是内存结构、GC、类加载、JMM 和线上排查。
3. 堆主要存对象，栈主要和方法调用有关。
4. GC 的核心目标是回收不再使用的对象内存。
5. JVM 面试后半段常常会追到 OOM、Full GC 和调优排查。

这 5 句就是 JVM 主线的最小骨架。

## 3. 什么是 JVM

最稳的回答：

> JVM 就是 Java 虚拟机，它负责把 Java 程序加载进来并运行，提供字节码执行、内存管理、垃圾回收等能力。

更口语一点：

> 你可以把 JVM 理解成 Java 程序真正跑起来的运行环境。

## 4. 为什么 Java 总要讲 JVM

因为 Java 程序不是直接把源码交给操作系统执行，而是：

1. 先编译成字节码
2. 再交给 JVM 去加载和执行

所以 Java 的很多能力都和 JVM 直接相关，比如：

- 跨平台
- 内存管理
- 垃圾回收
- 类加载机制

一句话记忆：

> Java 的“能跑、怎么跑、跑出问题怎么查”，背后很多都绕不开 JVM。

## 5. JVM 面试最常见的 5 条主线

如果你问 JVM 面试到底在考什么，最常见就是这 5 条：

1. 内存结构
2. 垃圾回收
3. 类加载
4. JMM 与 `volatile`
5. OOM / 调优 / 线上排查

很多题看起来分散，其实都可以挂到这 5 条线上。

## 6. 内存结构为什么一定会问

因为这是 JVM 的入口题。

最常见问法：

- JVM 内存结构有哪些
- 堆和栈有什么区别
- 方法区怎么理解
- 程序计数器、本地方法栈知道吗

面试里你至少要先知道：

- 程序计数器
- 虚拟机栈
- 本地方法栈
- 堆
- 方法区

但真正高频深入追问的，主要还是：

- 堆
- 栈
- 方法区

## 7. GC 为什么是第二条主线

因为只要讲到堆，就很容易继续问：

- 垃圾回收怎么理解
- 对象什么时候可以回收
- 常见垃圾回收器知道哪些
- CMS、G1 有什么印象

所以 GC 其实是从“对象放哪里”自然延伸出来的。

一句话理解：

> 堆解决“对象放哪”，GC 解决“这些对象什么时候该回收”。 

## 8. 类加载为什么也很高频

因为面试官讲 JVM，不会只停在“对象和内存”。

继续往下很容易问：

- 类怎么加载进 JVM
- 类加载过程是什么
- 双亲委派是什么
- 为什么要有双亲委派

所以类加载是 JVM 里另一条非常典型的独立主线。

## 9. JMM 和 `volatile` 为什么常被并到 JVM

严格来说，JMM 更偏并发和内存可见性问题。

但 Java 面试里通常会把它和 JVM 一起问，因为它们都和“程序运行机制”有关。

常见问法：

- 什么是 JMM
- `volatile` 解决什么问题
- 可见性、有序性怎么理解
- happens-before 知道吗

所以你可以把它理解成：

> JVM 主线讲完之后，经常顺手就会切到 JMM 和 `volatile`。

## 10. 为什么后面还会追到 OOM 和调优

因为这部分最像真实工作场景。

常见问法：

- OOM 怎么排查
- Full GC 频繁怎么排查
- CPU 飙高怎么查
- 常见 JVM 工具知道哪些

这类题特别适合区分：

- 只会背基础定义的人
- 和知道线上问题怎么答的人

## 11. 一版适合面试开场的总回答

如果面试官问：

> 你先整体讲一下 JVM。

你可以这样答：

> JVM 我一般从几条主线来理解。第一条是运行时内存结构，比如堆、栈、方法区这些区域分别干什么；第二条是垃圾回收，也就是对象什么时候回收、常见回收器怎么理解；第三条是类加载和双亲委派，也就是类怎么从 `.class` 变成 JVM 可用的数据；第四条是 JMM 和 `volatile`，更偏并发下的可见性和有序性；最后如果往真实工作场景讲，我会再补 OOM、Full GC 和 JVM 排查思路。

## 12. 复习 JVM 最合适的顺序

建议按这个顺序走：

1. 先看 JVM 是什么
2. 再看内存结构
3. 再看堆、栈、方法区
4. 然后补 GC
5. 再补类加载
6. 再补 JMM / `volatile`
7. 最后补 OOM、调优和线上排查

这样不会一开始就陷进碎细节里。

## 13. 最容易答错的点

- 一上来就只会背堆和栈，讲不出 JVM 主线
- 只会背 GC 名词，说不清为什么要回收
- 把类加载、JMM、调优割裂得太开
- 一被问 JVM 就想直接讲参数和调优，结果主线没立住

## 14. 一句话收尾

> JVM 这章最稳的复习方式，不是先背碎知识点，而是先把“内存结构 -> GC -> 类加载 -> JMM -> 排查调优”这条主线立住。
