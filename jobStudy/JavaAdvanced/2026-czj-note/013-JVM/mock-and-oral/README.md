# mock-and-oral

## 1. 这个子目录是干什么的

这个子目录专门放 JVM 的“输出训练层”材料。

它和顶层那些主线笔记不太一样，重点不是继续补定义，而是训练你：

- 面试时怎么开口
- 不同时长怎么回答
- 被追问时怎么接
- 哪些题最容易答错

如果说顶层文档更偏“学会”，那这个子目录更偏“说出来”。

## 3. 建议使用顺序

建议按下面顺序使用：

1. 先看 `jvm-1min-3min-5min-answer-pack.md`
   先把 1 分钟、3 分钟、5 分钟的讲法练顺。
2. 再看 `jvm-mock-qa-script.md`
   练“面试官追问，你怎么继续接”。
3. 再看 `jvm-handwritten-qa.md`
   专门修那些最容易答错、最容易被追崩的点。
4. 面试前一天看 `jvm-last-day-revision.md`
   只抓最后一轮最值得扫的内容。
5. 自测时看 `jvm-self-check-scorecard.md`
   判断自己到底是不是已经过线。
6. 如果担心“不会往项目上讲”，看 `jvm-project-scene-answers.md`
   练怎么把 JVM 和真实项目现象连起来。
7. 最后冲刺时做 `jvm-final-mock-paper.md`
   用连续题目检查自己会不会中途断线。
8. 进面试前最后几分钟看 `jvm-cheatsheet-one-page.md`
   只看最关键的锚点和速背段落。

## 4. 这个目录最适合什么时候看

最适合下面几种场景：

- 面试前 1 到 3 天，准备开始口头演练
- 已经会知识点，但表达不稳定
- mock 里发现自己容易被打断后接不上

## 5. 一句话收尾

> 这个子目录的核心目标不是补知识点，而是把 JVM 从“我知道”练到“我能稳定讲出来”。
