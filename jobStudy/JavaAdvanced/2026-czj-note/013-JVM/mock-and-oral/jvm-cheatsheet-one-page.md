# JVM 一页速背版

## 1. 这一页怎么用

这一页就是极限压缩版。

适合场景：

- 面试前最后 5 到 10 分钟
- 进面试前在脑子里快速过一遍
- 不想翻太多文档，只想看最关键的锚点

## 2. JVM 主线

先记这 5 条：

1. 内存结构
2. GC
3. 类加载
4. JMM / `volatile`
5. OOM / 排查

一句话总述：

> JVM 面试最核心的不是死背名词，而是把程序怎么跑、对象放哪、内存怎么回收、类怎么加载、多线程下数据为什么会出问题、线上出了问题怎么查这几条线讲清楚。

## 3. 内存结构

最小过线版：

> 堆主要放对象，线程共享；栈主要和方法调用、局部变量、栈帧有关，线程私有；方法区更偏类信息和常量。

高频追问：

- 堆和栈区别
- 哪些区域线程共享 / 私有
- 对象一般分配在哪
- 栈溢出和堆溢出怎么区分

## 4. GC

最小过线版：

> GC 本质上是 JVM 自动回收不再使用对象内存的机制。主流判断对象能不能回收，会讲可达性分析和 GC Roots。

一定记住：

- 不主讲引用计数
- 要会说为什么有分代思想
- Full GC 比 Minor GC 更重

## 5. 回收器

最小过线版：

> 常见回收器像 CMS、G1。简单讲，CMS 更偏低停顿，G1 更偏停顿可控，在现代服务端场景里更常见。

别答错：

- 不要只背名字
- 不要一听 Full GC 就只会说“加内存”

## 6. 类加载

五步先背顺：

1. 加载
2. 验证
3. 准备
4. 解析
5. 初始化

双亲委派一句话：

> 类加载请求先交给父加载器尝试处理，父加载器处理不了，子加载器才自己处理。

核心意义：

- 避免重复加载
- 保护核心类

## 7. JMM 与 `volatile`

JMM 一句话：

> JMM 是多线程下的一套内存访问规范，主要涉及可见性、有序性、原子性。

`volatile` 两点：

1. 保证可见性
2. 限制指令重排

必须会答：

> `volatile` 不能保证 `i++`，因为 `i++` 不是原子操作，它至少包含读、加一、写回几个步骤。

## 8. 对象创建与引用

`new` 一个对象先背这 5 步：

1. 检查类是否加载
2. 分配内存
3. 设置默认零值
4. 设置对象头
5. 执行构造方法

引用类型一句话：

> 强引用正常用；软引用内存紧张时再回收；弱引用 GC 来了更容易被回收；虚引用主要跟踪对象回收。

## 9. OOM / Full GC / CPU 飙高

这三条一定分开：

- OOM：更偏内存分配失败
- Full GC 频繁：更偏对象回收压力大
- CPU 飙高：更偏线程执行问题

最稳表达：

> JVM 排查不能把所有问题都往“内存不够”上靠，要先区分是内存、GC 还是线程问题。

## 10. 常见 JVM 工具

至少记住：

- `jps`：看 Java 进程
- `jstat`：看 GC
- `jmap`：看堆 / 导出快照
- `jstack`：看线程栈

## 11. 最容易答错的点

千万别混：

- 堆和栈
- GC 和“定时删对象”
- `volatile` 和完整线程安全
- 双亲委派和继承关系
- OOM / Full GC / CPU 飙高

## 12. 最后一段总背诵

> JVM 我一般会从几条主线讲。第一条是内存结构，比如堆、栈、方法区这些区域分别干什么；第二条是垃圾回收，也就是对象怎么判断能不能回收，以及常见回收器怎么理解；第三条是类加载和双亲委派；第四条是 JMM 和 `volatile`，更偏多线程下的可见性、有序性和原子性问题；如果往真实工作场景去讲，我会再补 OOM、Full GC 和排查思路。
