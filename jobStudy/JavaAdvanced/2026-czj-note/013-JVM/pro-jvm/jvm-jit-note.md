# JVM JIT 与即时编译笔记

## 1. 这份文档怎么用

这一篇不是基础 JVM 的必背定义，而是更偏进阶理解。

它主要回答几个问题：

- Java 不是“解释执行”就结束了吗，为什么还会讲编译执行
- 什么是 JIT
- 为什么 JVM 程序跑久了可能会更快
- 热点代码、方法内联、逃逸分析这些词到底在说什么

如果你想把 JVM 从“会背主线”提升到“能解释为什么性能会变化”，这一篇就值得补。

## 2. 先记住最核心的 6 句

先把下面这 6 句记住：

1. Java 代码先编译成字节码，再由 JVM 执行。
2. JVM 执行字节码，不代表永远只靠解释执行。
3. JIT 就是 Just-In-Time Compiler，即时编译器。
4. JIT 会把热点代码编译成本地机器码，提高运行效率。
5. 程序跑一会儿之后变快，常常和热点探测、JIT 优化有关。
6. 方法内联、逃逸分析、标量替换、锁消除，都是 JIT 常见优化话题。

## 3. 为什么 JVM 还要讲 JIT

很多人学 JVM 时，会形成一个很粗糙的印象：

> Java 程序就是编译成字节码，然后 JVM 一条条解释执行。

这句话不算错，但不完整。

更完整的说法是：

> JVM 既可以解释执行，也可以把热点代码编译成本地机器码执行，而 JIT 就是负责这件事的重要机制。

也就是说，JVM 不是死板地“只解释”，而是会根据运行情况做动态优化。

## 4. 什么是 JIT

最稳的回答：

> JIT 就是即时编译器。它会在程序运行过程中，把那些执行频率高的热点代码编译成本地机器码，这样后续再执行这些代码时，就不需要一直按字节码解释执行了，效率会更高。

一句话记忆：

> JIT 的核心，就是把“经常跑的代码”变成“更快的本地代码”。

## 5. 为什么程序跑久了可能更快

这在面试里很适合被问。

更稳的答法：

> 因为 JVM 在运行过程中会做热点探测。某些方法或者代码路径执行得足够多之后，JIT 会把它们编译并优化，所以程序在运行一段时间后，可能比刚启动时更快。

这一题的重点，不是背阈值参数，而是知道：

> JVM 会“观察程序行为”再做优化，而不是一上来把所有代码都重编译一遍。

## 6. 什么是热点代码

可以这样理解：

> 热点代码就是那些在运行中被频繁调用、频繁执行的方法或代码片段。JIT 不会盲目优化所有代码，而是优先盯住这些热点区域。

为什么需要热点探测？

> 因为并不是所有代码都值得花编译成本去优化，只有经常跑的代码，优化收益才大。

## 7. 解释执行和编译执行怎么理解

更稳的回答：

> 解释执行更灵活，启动更快，但单次执行效率相对低；编译执行前期有额外成本，但一旦编译成机器码，后续执行更快。JVM 的价值就在于，它不是二选一，而是把两者结合了起来。

一句话记忆：

> 解释执行偏灵活，编译执行偏高效，JVM 会在运行时动态平衡。

## 8. 方法内联是什么

这是 JIT 面试里很爱提的一点。

最稳的说法：

> 方法内联可以理解成，JIT 在发现某个方法很短、很频繁、很适合展开时，会把方法调用直接展开到调用点附近，减少方法调用本身的开销。

为什么它重要？

> 因为方法调用本身有成本，而内联之后，后续更多优化也更容易发生。

## 9. 什么是逃逸分析

这也是进阶 JVM 里最常见的词之一。

更稳的回答：

> 逃逸分析本质上是在分析一个对象的作用域，看它会不会逃出当前方法、当前线程。如果 JVM 能判断某个对象不会逃逸，就有机会对它做更激进的优化。

一句话记忆：

> 逃逸分析是在判断“这个对象会不会跑出去”。

## 10. 标量替换怎么理解

可以这样答：

> 如果 JVM 通过逃逸分析判断一个对象不会逃逸，而且它的使用方式也比较简单，那么就可能不真的创建这个对象，而是把对象拆成几个独立字段来处理，这种思路就可以理解成标量替换。

重点不是把术语背死，而是理解：

> 有时候 JVM 甚至能把“对象”优化到“不像对象那样存在”。

## 11. 锁消除怎么理解

最稳的回答：

> 如果 JVM 通过逃逸分析判断一段加锁代码其实不会产生线程竞争，比如对象根本没有逃出当前线程，那么这个锁就可能没有实际必要，JIT 可能会把这类无意义的锁开销消掉。

一句话记忆：

> 锁消除不是乱删锁，而是删掉那些“根本不会竞争”的锁。

## 12. JIT 在面试里一般怎么问

常见问法：

- 什么是 JIT
- 为什么 Java 程序运行久了会更快
- 解释执行和编译执行怎么理解
- 什么是方法内联
- 什么是逃逸分析
- 你知道哪些 JVM 编译优化

面试里的稳法是：

> 不追求把编译器实现细节讲得多深，而是讲清 JIT 为什么存在、怎么帮助程序变快、常见优化大概在做什么。

## 13. 一版适合面试的完整回答

> JVM 并不是永远只靠解释执行字节码。它在运行过程中会做热点探测，发现某些方法或者代码路径执行得足够频繁后，会通过 JIT 把这些热点代码编译成本地机器码，提高后续执行效率。所以 Java 程序有时跑久了会更快。再往下延伸的话，JIT 常见优化里我知道方法内联、逃逸分析、标量替换、锁消除这些概念。我的理解是，它们的核心都是在运行时根据代码实际行为，尽量减少不必要的开销。

## 14. 最容易答错的点

- 把 Java 说成永远只解释执行
- 把 JIT 说成“程序启动前就全部编译完”
- 听过逃逸分析，但说不清它在判断什么
- 把锁消除理解成“JVM 会随便删锁”

## 15. 一句话收尾

> JIT 这条线的核心，不是背术语，而是理解 JVM 为什么能在运行过程中越来越懂你的程序，并把热点代码优化得更快。
