# JVM 对象头、Mark Word 与锁状态笔记

## 1. 这份文档怎么用

这一篇是 JVM 和并发交叉区域里非常典型的进阶题。

它主要解决：

- Java 对象除了字段值，还有什么
- 什么是对象头
- Mark Word 到底在说什么
- 为什么对象头会和锁状态扯上关系

这部分如果能讲顺，面试官通常会觉得你对 `synchronized` 和对象运行时结构的理解更扎实。

## 2. 先记住最核心的 5 句

1. Java 对象在内存里，不只有实例数据。
2. 对象通常可以粗略理解成对象头、实例数据、对齐填充。
3. 对象头里会放一些运行时信息。
4. Mark Word 是对象头里很核心的一部分。
5. 锁状态、哈希码、GC 年龄等信息，都可能和对象头有关。

## 3. 对象头到底是什么

最稳的回答：

> 对象头可以理解成 Java 对象的运行时说明区域，它不属于业务字段，但会存储对象在 JVM 运行过程中的一些关键信息，比如哈希码、GC 年龄、锁相关状态等。

为什么对象头重要？

> 因为很多 JVM 行为并不是靠业务字段完成的，而是依赖这些运行时元数据。

## 4. Java 对象大概怎么布局

面试里通常先讲三部分就够了：

1. 对象头
2. 实例数据
3. 对齐填充

更稳的说法：

> Java 对象在内存里，一般可以粗略理解成对象头、实例数据和对齐填充三部分。对象头负责一些运行时信息，实例数据才是类里定义的成员变量，对齐填充则更多是为了满足内存对齐要求。

## 5. 什么是 Mark Word

这部分不用背成位图细节，先把本质讲清。

更稳的回答：

> Mark Word 可以理解成对象头里一块会随着运行状态变化而变化的区域。它里面会和对象哈希码、GC 年龄、锁标记等信息相关，所以它不是一个死的结构，而是会随着对象状态变化而承载不同含义。

一句话记忆：

> Mark Word 是对象头里最“动态”的部分之一。

## 6. 为什么 Mark Word 会和锁扯上关系

可以这样答：

> 因为 JVM 在处理 `synchronized` 这类锁时，需要知道对象当前是不是处于某种锁状态，而这些状态信息就会和对象头里的 Mark Word 相关。所以对象头不仅是“身份说明”，也会参与并发控制相关的运行时表示。

这题的重点不是画位图，而是理解：

> 对象和锁并不是完全分离的两套东西，锁状态会落到对象运行时结构上。

## 7. 什么是锁状态

在 JVM 进阶语境里，说锁状态，通常是在说：

- 当前对象有没有和锁相关的标记
- 对象处于哪种锁语义下
- JVM 怎么在运行时区分这些状态

更稳的回答：

> 锁状态可以理解成对象当前在并发访问下所处的某种运行时标记状态。面试里不一定非要把所有底层位都背下来，但至少要知道对象头会参与锁状态表达。

## 8. 为什么这条线常和 `synchronized` 一起问

因为很多人只会从代码层讲：

> `synchronized` 是互斥锁。

但再往下一层，面试官想看的是：

> 你知不知道这个“锁”在 JVM 里并不是一个纯抽象词，而是会和对象头、Mark Word、监视器这些底层表示关联起来。

## 9. 什么是监视器（Monitor）

这部分可以先讲概念，不必展开到过细实现。

更稳的回答：

> 从更底层的角度说，`synchronized` 的实现会和监视器这种机制有关。你可以把 Monitor 理解成 JVM 层面对互斥和线程同步的一种支撑结构，而对象头和锁标记又会和这些运行时机制发生联系。

一句话记忆：

> 面试里提对象头，不是为了背结构，而是为了把对象、锁、Monitor 三者串起来。

## 10. 面试里一般怎么问

常见问法：

- Java 对象内存布局是什么
- 什么是对象头
- Mark Word 是什么
- 对象头和锁有什么关系
- `synchronized` 为什么会和对象头相关

这类题的稳法是：

> 先讲布局，再讲对象头，再讲 Mark Word，最后讲它为什么会和锁状态有关。

## 11. 一版适合面试的完整回答

> Java 对象在内存里，我一般先粗略讲成对象头、实例数据和对齐填充三部分。对象头里会放一些运行时元数据，其中比较核心的一部分常常会提到 Mark Word。我的理解是，Mark Word 不是一个死结构，它会和对象哈希码、GC 年龄、锁标记这些信息相关，所以对象头不只是“身份说明”，也会和并发访问下的锁状态联系起来。这也是为什么讲 `synchronized` 时，很多面试官会继续追到对象头和底层表示。

## 12. 最容易答错的点

- 把对象头说成“就是类字段”
- 只会背对象头三个字，不知道和运行时信息有关
- 知道 Mark Word，但说不出它为什么重要
- 把对象头和锁状态彻底割裂开

## 13. 一句话收尾

> 对象头这条线真正要讲清的，不是结构名词，而是 Java 对象在运行时会承载很多和 GC、哈希、锁相关的信息。
