# Pro JVM 模拟题

## 1. 这份文档怎么用

这一篇不是基础 JVM 题，而是专门针对 `pro-jvm` 这 8 份进阶文档出的模拟题。

它的目标是检查你：

- 是不是真的理解了进阶 JVM，不只是看过文档
- 能不能把 JIT、对象头、STW、GC 日志、堆外内存、Metaspace、调优参数这些点连起来
- 被问到“更深一层”时会不会直接卡住

建议用法：

1. 先不看笔记，自己口头答。
2. 每题尽量控制在 30 秒到 2 分钟。
3. 如果某一组明显答不顺，再回对应文档补。

## 2. 第一组：JIT 与运行时优化

1. JVM 为什么还要讲 JIT，不是已经有字节码了吗？
2. 为什么 Java 程序跑久了可能更快？
3. 什么是热点代码？
4. 方法内联怎么理解？
5. 逃逸分析到底在判断什么？
6. 标量替换和锁消除分别在优化什么？

这一组最关键的不是背术语，而是：

> 你能不能讲清 JVM 为什么会在运行时越来越“懂程序”。

## 3. 第二组：对象头、Mark Word、锁状态

7. Java 对象在内存里大概怎么布局？
8. 什么是对象头？
9. Mark Word 是什么？
10. 为什么对象头会和锁状态扯上关系？
11. 为什么讲 `synchronized` 时，面试官会继续追到对象头？

这一组的核心是：

> 把“对象结构”和“锁运行时表示”连起来。

## 4. 第三组：Safepoint 与 STW

12. 什么是 STW？
13. 为什么 JVM 有时必须停业务线程？
14. 什么是 Safepoint？
15. 为什么线程不能在任意位置被粗暴停住？
16. 为什么说 GC 不是唯一会触发 STW 的来源？

这一组看的是：

> 你会不会把 GC 停顿理解成 JVM 一致性机制，而不是只背“GC 会卡顿”。

## 5. 第四组：GC 日志阅读

17. 为什么 GC 日志是 JVM 排查里很重要的入口？
18. 如果给你一段 GC 日志，你先看什么？
19. 为什么要特别关注 Full GC 和老年代？
20. 如果回收后内存下降很有限，你会怎么理解？
21. GC 日志和堆快照分别更擅长解决什么问题？

这一组的目标是：

> 看你会不会从日志里快速提炼“GC 类型、停顿、回收效果、老年代压力”。

## 6. 第五组：堆外内存与 Direct Memory

22. 为什么堆没满，程序也可能出内存问题？
23. 什么是堆外内存？
24. 什么是 Direct Memory？
25. 为什么 NIO 常常和 Direct Memory 一起问？
26. 为什么 Netty 这类框架也会和这条线关联？

这一组考的重点是：

> 你是不是还停留在“JVM 内存只看堆”的阶段。

## 7. 第六组：ClassLoader、Metaspace、类卸载

27. Metaspace 主要和什么有关？
28. 为什么类加载器会和内存问题扯上关系？
29. 什么是类卸载？
30. 为什么 Tomcat、热部署、插件化常常会问到这条线？
31. 什么是类加载器隔离？

这一组真正想看的是：

> 你能不能把“类怎么加载”和“类相关内存怎么释放”连起来。

## 8. 第七组：调优参数与思路

32. 为什么 JVM 调优不是一上来就改参数？
33. `Xms` 和 `Xmx` 主要控制什么？
34. 为什么 GC 日志也属于调优前提？
35. 如果系统响应时间敏感，回收器选择思路会怎么变？
36. 为什么参数调整离不开监控和验证？

这一组不考你背参数表，而考：

> 你知不知道参数只是手段，不是调优起点。

## 9. 第八组：进阶案例题

37. 堆没满但服务进程内存持续上涨，你先想到什么？
38. Full GC 频繁但回收效果一般，你先往哪几个方向怀疑？
39. CPU 飙高但堆占用不高，你的第一判断方向是什么？
40. 服务运行一段时间后变快，可能和 JVM 哪条机制有关？
41. 热部署几次后系统越来越不稳定，你会想到哪条 JVM 线？
42. 接口偶发性卡顿，但 SQL 不慢，你会不会想到 STW？

这一组的核心是：

> 你能不能把“现象”映射到“正确的 JVM 判断方向”。

## 10. 压力追问

43. 如果我说你讲的 JIT 太像背书，你会怎么换一种更口语的说法？
44. 如果我问你 Mark Word 你只知道名字，不知道为什么重要，你会怎么补救？
45. 如果我要求你把 Metaspace、类加载器、热部署放在一句话里讲清，你怎么说？
46. 如果我让你把“GC 日志、堆快照、热点线程”三者的分工区分清楚，你能不能顺下来？

## 11. 最后的使用建议

做这套题时，不要追求每题特别长。

更好的标准是：

- 能不能讲顺
- 能不能接基础追问
- 能不能把进阶概念讲得不像死背

## 12. 一句话收尾

> Pro JVM 模拟题真正考的，不是你知不知道更多词，而是你能不能把这些更深的机制讲成一套有逻辑的解释。
