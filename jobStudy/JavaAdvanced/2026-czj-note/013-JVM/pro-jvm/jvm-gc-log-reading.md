# JVM GC 日志阅读入门

## 1. 这份文档怎么用

这一篇不是教你背所有 GC 日志格式，而是教你面试里怎么讲：

- GC 日志能看出什么
- 面试官给你一段日志时，你该先抓什么
- 为什么会有 Full GC、停顿时间、回收前后堆占用这些信息

如果你能把 GC 日志讲成“我会先看哪几项”，已经比只会说“看日志”强很多了。

## 2. 先记住最核心的 5 句

1. GC 日志不是为了背格式，而是为了看趋势和问题。
2. 面试里先看 GC 类型、停顿时间、回收前后内存变化。
3. 如果 Full GC 频繁，要重点关注老年代。
4. 如果回收后内存下降不明显，通常要怀疑对象存活过多或回收不掉。
5. GC 日志通常是排查的入口，不是终点。

## 3. 为什么要看 GC 日志

最稳的回答：

> 因为 GC 日志能让我们看到 JVM 在垃圾回收时到底发生了什么，比如发生的是 Minor GC 还是 Full GC、停顿时间多长、回收前后堆占用变化如何。很多时候，单看“服务变慢”是不够的，要靠 GC 日志把问题往内存和回收方向收窄。

一句话记忆：

> GC 日志是 JVM 排障里最典型的第一手材料。

## 4. 面试里看到 GC 日志先看什么

不需要先讲复杂格式，先讲顺序。

最稳的顺序是：

1. 这是 Minor GC 还是 Full GC
2. 停顿时间有多长
3. 回收前后堆占用变化大不大
4. 老年代压力高不高
5. 这种现象是偶发还是持续

这比一上来背字段名字更像真实排查。

## 5. GC 类型怎么理解

面试里的最小表达：

> 如果日志显示的是 Minor GC，通常我会优先理解成新生代回收；如果是 Full GC，我会更警惕老年代压力、停顿时长和对象长期存活问题。

重点不是绝对定义，而是知道：

> Full GC 一般比普通 Minor GC 更值得紧张。

## 6. 停顿时间怎么讲

可以这样答：

> 停顿时间反映的是这次 GC 对业务线程的影响有多大。如果停顿很短，可能只是正常回收；如果停顿明显偏长，尤其还频繁出现，就会直接影响接口响应和系统稳定性。

一句话记忆：

> 业务感受到的“卡”，很多时候不是回收次数本身，而是停顿时间。

## 7. 回收前后内存变化怎么理解

这是 GC 日志里非常实用的一点。

可以这样答：

> 如果一轮 GC 之后内存占用明显下降，说明这次回收确实释放了不少对象；如果回收后下降很有限，甚至几乎没降，就要怀疑对象存活时间太长、老年代压力过大，或者有对象回收不掉。

这一点特别适合用来回答：

> 为什么 Full GC 频繁还没什么效果？

## 8. 为什么看老年代

因为很多棘手 JVM 问题最后都会落到这里。

更稳的答法：

> 如果日志里 Full GC 很频繁，或者每次回收后老年代占用还是很高，我会优先怀疑老年代压力，因为这往往意味着有长生命周期对象、大对象，或者缓存、集合这类对象一直活着。

## 9. 面试里如果给你一段日志，你怎么说

即使不展开日志细节，也可以这样答：

> 我一般会先判断这是哪种 GC，再看停顿时间和回收前后内存变化。如果是 Full GC，而且回收后老年代占用还很高，那我会往对象存活过多、缓存堆积、大对象或者回收不掉这几个方向去想。GC 日志更多是帮我确认方向，后面通常还要结合堆快照、监控和业务对象去看。

## 10. GC 日志和堆快照的关系

这题很适合进阶一点回答。

可以这样答：

> GC 日志更像动态行为记录，告诉你 JVM 回收时发生了什么；堆快照更像某个时刻的静态截面，告诉你内存里是谁占得多。所以 GC 日志通常用来发现问题趋势，堆快照再用来定位具体对象。

一句话记忆：

> GC 日志看趋势，堆快照找元凶。

## 11. 面试里一般怎么问

常见问法：

- 你怎么看 GC 日志
- Full GC 频繁你会先看什么
- GC 日志里什么信息最关键
- 为什么看完 GC 日志还不够

这类题最稳的方式是：

> 不炫格式，不背参数，先讲“我会先看哪几项，再怎么判断”。

## 12. 一版适合面试的完整回答

> GC 日志我一般会先看几件事：第一是发生的是 Minor GC 还是 Full GC；第二是停顿时间有多长；第三是回收前后内存占用变化大不大，尤其老年代占用有没有明显下降。如果 Full GC 很频繁，而且回收后老年代还是很高，我会优先怀疑有长生命周期对象、大对象或者缓存堆积。我的理解是，GC 日志主要用来帮我确认问题方向，后面通常还要结合堆快照和业务对象继续定位。

## 13. 最容易答错的点

- 一上来就背一堆日志格式
- 只会说“看日志”，说不出先看什么
- 把 GC 日志当成终点，而不是排查入口
- 不知道为什么要特别关注 Full GC 和老年代

## 14. 一句话收尾

> GC 日志阅读这条线，核心不是背格式，而是学会从日志里快速看出“GC 类型、停顿、回收效果、老年代压力”这几个最关键的信号。
