# JVM 最终总复盘

## 1. 这份文档怎么用

这一份不是详细笔记，也不是单纯的问题清单，而是 JVM 这一章的考前总复盘版。

适合在下面几种场景使用：

- 你已经把 JVM 主线过了一遍，想做最后收口
- 面试前想快速把整章重新串起来
- 你担心知识点是散的，想把它们重新连成一条线

建议用法：

1. 先通读一遍，确认 JVM 主线有没有断点。
2. 再挑自己最薄弱的一条线回看对应专题文档。
3. 最后只背“1 分钟总述”和“最后速记版”。

## 2. JVM 这一章到底在讲什么

JVM 这一章如果只背名词，很容易越学越碎。

更好的理解方式是：

> JVM 这一章本质上在回答 5 个问题：程序怎么跑起来，对象放在哪，内存怎么回收，类怎么加载，多线程下数据为什么会出问题，以及线上出了问题怎么查。

如果你能把这 5 个问题讲顺，JVM 这一章就算真的进入面试状态了。

## 3. 第一条主线：程序怎么跑起来

这条线对应的是：

- JVM 是什么
- JDK、JRE、JVM 的关系
- Java 为什么能跨平台
- 类加载为什么重要

一句话理解：

> Java 程序先编译成字节码，再交给不同平台上的 JVM 去运行，所以 Java 能强调跨平台，而 JVM 则负责把这些字节码真正执行起来。

## 4. 第二条主线：对象放在哪，方法怎么执行

这条线对应的是 JVM 运行时内存结构。

最核心的区域通常是：

- 堆
- 虚拟机栈
- 方法区

一句话理解：

> 堆主要放对象，栈主要和方法调用、局部变量、栈帧有关，方法区更偏类元数据、常量这些信息。

面试高频重点：

- 堆和栈有什么区别
- 哪些区域线程私有，哪些区域线程共享
- 对象一般分配在哪
- 栈溢出和堆溢出怎么区分

## 5. 第三条主线：内存怎么回收

这条线对应 GC。

真正要讲清的不是“GC 是垃圾回收”这么简单，而是：

- JVM 怎么判断对象是不是垃圾
- 为什么主流答案是可达性分析
- 为什么要有分代思想
- 常见垃圾回收算法和回收器怎么理解

一句话理解：

> GC 的核心不是“删对象”，而是判断对象还能不能继续活着，然后用合适的回收策略把没用的对象空间收回来。

这一组最常见的考点：

- 可达性分析
- GC Roots
- 标记清除、复制、标记整理
- Minor GC、Full GC
- CMS、G1
- Stop-The-World

## 6. 第四条主线：类怎么从 `.class` 变成可运行数据

这条线对应类加载。

你至少要能讲顺：

- 加载
- 验证
- 准备
- 解析
- 初始化

以及：

- 类加载器是什么
- 双亲委派是什么
- 为什么双亲委派重要

一句话理解：

> 类加载本质上是 JVM 把 `.class` 文件变成自己能识别、能使用的类数据结构，而双亲委派则是在这个过程中保证加载稳定性和核心类安全性的一种机制。

## 7. 第五条主线：多线程下为什么会出问题

这条线对应 JMM 和 `volatile`。

最重要的是把这 3 个词分开：

- 可见性
- 有序性
- 原子性

一句话理解：

> JMM 不是一块真实内存，而是一套多线程下的访问规范。`volatile` 主要解决可见性和一定程度上的有序性，但不能保证像 `i++` 这种复合操作的原子性。

高频追问：

- 为什么 `volatile` 不能保证 `i++`
- happens-before 是什么
- `volatile` 和 `synchronized` 怎么区分

## 8. 第六条主线：对象从无到有是怎么创建出来的

这条线常常会和内存结构一起被追问。

核心步骤可以先记成：

1. 检查类是否已经加载
2. 分配对象内存
3. 设置默认零值
4. 设置对象头
5. 执行构造方法

另外这条线还经常接着问：

- 对象在内存里怎么布局
- 什么是对象头
- 强引用、软引用、弱引用、虚引用怎么区分

一句话理解：

> 对象创建考的是“对象从无到有的过程”，引用类型考的是“对象在 GC 下会被怎么对待”。

## 9. 第七条主线：线上出了问题怎么查

这条线对应：

- OOM
- 内存泄漏
- Full GC 频繁
- CPU 飙高
- JVM 工具
- JVM 调优

这里最重要的不是背工具名，而是分清问题类型。

一句话理解：

> OOM 更偏内存分配失败，Full GC 频繁更偏对象存活和回收压力，CPU 飙高更偏线程执行问题，这三条线不能混着答。

你至少要知道这些工具：

- `jps`
- `jstat`
- `jmap`
- `jstack`
- `jconsole`
- `jvisualvm`

## 10. JVM 最常见的错误答法

最容易翻车的地方通常是：

- 把 JVM 讲成一堆名词，不成体系
- 把堆和栈说混
- 只会说 GC 是回收垃圾，不知道怎么判断垃圾
- 只知道双亲委派，不知道类加载五个阶段
- 只会说 `volatile` 线程安全
- 把 OOM、Full GC、CPU 飙高混在一起答
- 一讲调优就只会说“调参数”

## 11. 面试前最该先背的一版

> 我一般把 JVM 分成几条主线来理解。第一条是运行时内存结构，比如堆、栈、方法区这些区域分别干什么；第二条是垃圾回收，也就是对象什么时候回收、怎么判断对象可不可以回收，以及常见回收器怎么理解；第三条是类加载和双亲委派，也就是类怎么从 `.class` 变成 JVM 里的可用数据；第四条是 JMM 和 `volatile`，更偏多线程下的可见性、有序性和原子性问题；最后如果往真实工作场景讲，我会再补 OOM、Full GC 频繁、CPU 飙高和 JVM 排查思路。

## 12. 如果只剩最后 3 分钟

只记下面这 6 句：

1. JVM 主线是内存结构、GC、类加载、JMM、排障。
2. 堆主要放对象，栈主要和方法调用有关。
3. GC 判断对象回收主流靠可达性分析，不是主讲引用计数。
4. 类加载要会五个阶段和双亲委派。
5. `volatile` 保证可见性和一定有序性，但不能保证 `i++` 的原子性。
6. OOM、Full GC、CPU 飙高是三条不同的排查思路。

## 13. 一句话收尾

> JVM 这一章真正的复习目标，不是把所有知识点背碎，而是把“程序运行、对象存放、垃圾回收、类加载、并发模型、线上排查”这几条线讲成一整套完整故事。
