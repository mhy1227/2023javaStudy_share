# JVM 面经高频题整理

## 1. 这份文档怎么来

这一篇不是凭印象罗列，而是基于公开中文面经讨论页做的整理，来源以：

- 牛客公开面经贴
- LeetCode 中文讨论区里的 Java / 后端面经贴

为主。

结论先说：

> 近两年公开面经里，JVM 被问得最多的仍然是内存结构、GC、类加载、`volatile` / JMM、OOM / Full GC / 排查这几条线，和你前面整理的主线基本一致。

## 2. 从面经里反复出现的 JVM 主线

从公开面经看，JVM 高频题大体还是集中在下面 6 条：

1. JVM 整体认知与 1 分钟总述
2. 运行时内存结构，尤其是堆、栈、方法区
3. GC，可达性分析、分代、Full GC、回收器
4. 类加载与双亲委派
5. JMM、`volatile`、`synchronized`
6. OOM、Full GC、CPU 飙高、线上排查

也就是说，面经并没有把 JVM 问偏，还是围绕最经典的骨架在追。

## 3. 第一类：开场先让你整体讲 JVM

公开面经里很常见的问法：

- 你先整体讲一下 JVM
- 你说一下你对 JVM 的理解
- JVM 这一章你一般怎么讲

这类题的本质不是让你背百科，而是看你：

- 有没有主线
- 会不会一上来就散
- 后面能不能顺着你的回答继续追

## 4. 第二类：堆、栈、方法区

这类题在面经里出现得非常稳：

- JVM 内存结构有哪些
- 堆和栈的区别
- 对象放在哪
- 方法区怎么理解
- 栈溢出和堆溢出怎么区分

从面经表达看，这类题经常是“一开场 JVM 总述之后的第一追问”。

也就是说：

> 面试官常常会先让你整体讲 JVM，然后第一刀直接切到堆、栈、方法区。

## 5. 第三类：GC 和 Full GC

公开面经里 GC 相关问题依旧是绝对高频：

- 什么是 GC
- JVM 怎么判断对象能不能回收
- 为什么不主讲引用计数
- 什么是 GC Roots
- 为什么要有分代思想
- CMS 和 G1 怎么区分
- Full GC 频繁怎么查

这里有个很明显的趋势：

> 面试官不太满足于“你会背 GC 定义”，而是更喜欢继续追 Full GC、老年代压力、对象回收不掉这类更接近场景的问题。

## 6. 第四类：类加载与双亲委派

这条线在面经里也很稳定：

- 类加载过程有哪些阶段
- 双亲委派是什么
- 为什么要有双亲委派
- 如果没有双亲委派会怎么样

这类题经常不是单独出现，而是和“你讲 JVM”连在一起。

典型面试节奏是：

1. 先让你整体讲 JVM
2. 再抽一条主线
3. 然后继续追到双亲委派

## 7. 第五类：`volatile`、JMM、线程安全

在公开面经里，这一条也非常高频，尤其 Java 后端岗：

- 什么是 JMM
- `volatile` 有什么作用
- `volatile` 为什么不能保证 `i++`
- `volatile` 和 `synchronized` 怎么区分
- happens-before 是什么

从面经看，这里最喜欢问的不是“大定义”，而是：

> `volatile` 为什么不能保证 `i++`

因为这题可以一下子看出候选人有没有把：

- 可见性
- 有序性
- 原子性

分清。

## 8. 第六类：OOM、CPU 飙高、线上排查

这一条在线上经验相关岗位里出现率很高：

- 什么是 OOM
- 什么是内存泄漏
- OOM 和内存泄漏有什么区别
- Full GC 频繁怎么查
- CPU 飙高怎么查
- 常见 JVM 工具有哪些

一个比较明显的面经信号是：

> 面试官很喜欢把 OOM、Full GC、CPU 飙高放在一起问，看你会不会混着答。

所以这三条线一定要强行拆开。

## 9. 第七类：对象创建与引用类型

这类题虽然没有前面几条那么绝对高频，但在公开面经里也反复出现：

- `new` 一个对象发生了什么
- 对象在内存里怎么布局
- 什么是对象头
- 强软弱虚引用怎么区分

这类题的特点是：

- 看起来偏细
- 但很适合顺着内存结构继续往下追

所以你不能完全不准备。

## 10. 面经里比较真实的追问方式

从这些公开面经看，JVM 很少只问一道孤立题。

更常见的是下面这种链路：

1. 你先讲 JVM
2. 你讲一下内存结构
3. 堆和栈区别
4. GC 怎么判断对象回收
5. 为什么 `volatile` 不能保证 `i++`
6. Full GC 频繁怎么查

也就是说，公开面经里真正高频的不是单题，而是：

> 面试官顺着 JVM 主线连续抽查。

## 11. 从面经里提炼出的高频程度

如果粗分优先级，可以这样看：

第一梯队，几乎必备：

- JVM 总述
- 堆、栈、方法区
- GC / 可达性分析 / 分代
- `volatile` / `i++`
- OOM / Full GC / 排查

第二梯队，高概率追问：

- CMS / G1
- 类加载五步
- 双亲委派
- `volatile` 和 `synchronized`

第三梯队，偶尔深入：

- 对象创建过程
- 对象头
- 强软弱虚引用

## 12. 结论

结合这些公开面经，可以做一个很实用的判断：

> 你前面给 JVM 这一章整理的主线方向是对的，没有偏。真正需要继续强化的，不是再补太多新知识，而是把这些高频线讲顺、讲稳、能接住追问。
