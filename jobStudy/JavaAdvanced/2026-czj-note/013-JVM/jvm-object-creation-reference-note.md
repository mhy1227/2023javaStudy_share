# JVM 对象创建过程与引用类型笔记

## 1. 这份文档怎么用

这一篇专门补 JVM 里一组很容易被单独追问的问题：

- `new` 一个对象到底发生了什么
- 对象通常分配在哪
- 对象在内存里大概怎么布局
- 什么是对象头
- 强引用、软引用、弱引用、虚引用怎么区分

这组题经常和内存结构、GC 一起混着问，所以单独拉出来整理更容易记。

## 2. 先记住最核心的 7 句

先把下面这 7 句记住：

1. `new` 一个对象时，JVM 一般会先检查类是否已经加载。
2. 如果类还没加载，就要先完成类加载。
3. 对象通常在堆上分配内存。
4. 分配完内存之后，JVM 会先给对象字段设默认零值。
5. 然后 JVM 会设置对象头这类运行时信息。
6. 最后才会执行构造方法，把对象真正初始化完成。
7. 引用类型不只有强引用，还有软引用、弱引用、虚引用。

## 3. `new` 一个对象大概发生了什么

最稳的回答：

> 我一般会把 `new` 一个对象理解成几个步骤。首先 JVM 会先检查这个类是不是已经被加载，如果没有，就先完成类加载。然后在堆里为对象分配内存，给字段先设默认零值，再设置对象头信息，最后再执行构造方法，把对象真正初始化完成。

这一题不是考你背底层实现细节，而是考你能不能把“对象从无到有”的流程讲顺。

## 4. 为什么对象创建前要先检查类加载

可以这样答：

> 因为 JVM 要先知道这个类的元数据、字段信息、方法信息这些内容，才能知道这个对象该分配多大、该怎么初始化。所以对象创建之前，类至少得先处于可用状态。

一句话记忆：

> 没有类信息，就没法正确创建对象。

## 5. 对象一般分配在哪

更稳的回答：

> 面试里通常先回答对象主要分配在堆里。因为堆是 JVM 里最主要的对象存储区域，也是垃圾回收重点关注的地方。

注意这里不要一上来就把问题讲复杂。先把“主要在堆里”答稳，再等面试官决定要不要继续追。

## 6. 为什么构造方法不是第一步

可以这样答：

> 因为构造方法本质上是在已经有对象内存空间的前提下，去完成对象的业务初始化。如果对象内存都还没分出来，构造方法也没有对象可以操作。

## 7. 对象在内存里大概怎么布局

面试里通常讲三部分就够了：

1. 对象头
2. 实例数据
3. 对齐填充

更稳的回答：

> Java 对象在内存里，面试里一般可以粗略理解成三部分。第一部分是对象头，里面会存一些运行时元数据；第二部分是实例数据，也就是对象真正的字段值；第三部分是对齐填充，主要是为了满足内存对齐要求。

## 8. 对象头怎么理解

可以这样答：

> 对象头可以理解成对象的运行时说明区域，里面会放一些和对象身份、锁状态、GC 年龄、哈希码等相关的信息。它不属于业务字段，但对 JVM 运行很重要。

这一题的重点不是把对象头每一位都背下来，而是知道：

> Java 对象不是只有字段值，对象头同样承载了很多运行时信息。

## 9. 实例数据是什么

可以这样答：

> 实例数据就是对象真正保存的字段值，也就是你在类里定义的成员变量对应的数据区域。

## 10. 对齐填充是什么

更口语一点的说法：

> 对齐填充可以理解成为了满足 JVM 内存对齐要求，可能会在对象末尾补一点空位。它不是业务数据，更像是为了让对象布局更规整。

## 11. 什么是引用类型

标准回答：

> Java 里引用类型常见会分成强引用、软引用、弱引用和虚引用。它们的区别，本质上在于对象在 GC 发生时被回收的条件和语义不同。

## 12. 强引用怎么讲

可以这样答：

> 强引用就是最普通的引用方式，比如 `Object obj = new Object()`。只要强引用还存在，垃圾回收器通常就不会主动回收这个对象。

一句话记忆：

> 强引用最常见，也是最不容易被回收的。

## 13. 软引用怎么讲

更稳的回答：

> 软引用的特点是内存充足时对象可以先保留，内存紧张时更容易被回收。所以它常常被拿来和缓存场景联系在一起。

一句话记忆：

> 软引用更像“内存不够了再让步”。

## 14. 弱引用怎么讲

可以这样答：

> 弱引用比软引用更弱，只要发生垃圾回收，如果对象只剩弱引用，通常就会被回收，不太强调内存是否紧张。所以它更适合那些“不再强依赖就尽快清掉”的场景。

一句话记忆：

> 弱引用更像“GC 一来就容易被清掉”。

## 15. 虚引用怎么讲

更稳的回答：

> 虚引用本身几乎拿不到对象实例，它的重点不在于继续使用对象，而在于配合引用队列去感知对象什么时候被回收。也就是说，它更偏回收跟踪，而不是对象访问。

## 16. 强软弱虚引用怎么一句话区分

可以直接背：

> 强引用是正常使用对象；软引用是内存不够时再回收；弱引用是 GC 来了就更容易被回收；虚引用主要是跟踪对象回收过程。

## 17. 面试里常见追问怎么接

如果面试官问：

> 你讲一下 `new` 一个对象发生了什么。

你可以这样答：

> 我一般会讲成几个步骤。先检查类有没有加载，没有的话先加载；然后在堆里分配对象内存，给字段设默认零值，再设置对象头，最后执行构造方法完成初始化。

如果面试官继续追：

> 为什么要先设默认值？

可以稳一点这样答：

> 因为对象创建出来之后，即使构造方法里没有主动给所有字段赋值，字段也必须先处于一个合法初始状态，所以 JVM 会先做默认零值初始化。

如果面试官继续问：

> 这几种引用有什么实际意义？

你可以这样答：

> 主要是为了给不同场景的内存回收语义。强引用适合正常持有对象，软引用适合缓存这种“可以保留、也可以在紧张时让出内存”的场景，弱引用适合不需要强持有的辅助性对象，虚引用则更偏底层的回收跟踪。

## 18. 一版适合面试的完整回答

> `new` 一个对象时，我一般会理解成先检查类有没有加载，没有的话先完成类加载；然后在堆里分配内存，先把字段设成默认零值，再设置对象头，最后执行构造方法，把对象真正初始化完成。对象在内存里可以粗略理解成对象头、实例数据和对齐填充三部分。至于引用类型，最常见的是强引用，另外还有软引用、弱引用和虚引用。软引用更偏缓存，弱引用在 GC 时更容易被回收，虚引用主要是为了跟踪对象回收过程。

## 19. 最容易答错的点

- 一上来就把问题讲成“对象都在栈里”
- 把构造方法误说成对象创建第一步
- 只知道强引用，不知道软弱虚引用
- 把软引用和弱引用说成一回事
- 把虚引用讲成“为了更强地引用对象”

## 20. 一句话收尾

> 对象创建这条线，核心是讲清对象从无到有的过程；引用类型这条线，核心是讲清不同引用强度对应不同的回收语义。
