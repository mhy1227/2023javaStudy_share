# JVM 模拟面试追问链

## 1. 这份文档怎么用

这一篇不是单纯的问题清单，而是更像一场真实 JVM 面试里的追问链。

建议用法：

1. 先看主问题，自己口头答一遍。
2. 再顺着追问往下答，尽量不要中断。
3. 如果某一段明显答不顺，就回对应主线文档补。

这份文档的目的不是让你背标准答案，而是训练你把 JVM 从“点状知识”讲成“连续对话”。

## 2. 第一组：先用 1 分钟整体讲 JVM

主问题：

> 你先整体讲一下你对 JVM 的理解。

你答完之后，面试官大概率会继续追：

1. 你刚提到内存结构，那 JVM 运行时内存结构有哪些？
2. 堆和栈有什么区别？
3. 为什么面试里老爱问堆、栈、方法区？
4. 对象一般放在哪？

这一组的核心，不是把名词背全，而是先把 JVM 主线立起来：

- 内存结构
- GC
- 类加载
- JMM / `volatile`
- OOM / 排查

## 3. 第二组：内存结构追问链

主问题：

> 你讲一下 JVM 内存结构。

常见追问：

1. 哪些区域是线程私有，哪些是线程共享？
2. 程序计数器是干什么的？
3. 虚拟机栈里主要放什么？
4. 方法区你怎么理解？
5. 为什么堆是 GC 的重点区域？

继续深一层，可能还会问：

1. 栈溢出一般是什么场景？
2. 堆溢出和栈溢出怎么区分？
3. 方法区和元空间是什么关系？

回答重点：

> 先把区域划分答清，再把“堆更偏对象和 GC，栈更偏方法调用和线程执行”讲清。

## 4. 第三组：GC 追问链

主问题：

> 你讲一下 JVM 的垃圾回收。

常见追问：

1. JVM 怎么判断对象是不是垃圾？
2. 为什么不主讲引用计数？
3. 什么是可达性分析？
4. 什么是 GC Roots？
5. 常见垃圾回收算法有哪些？

继续往下追，可能会问：

1. 标记清除、复制、标记整理怎么区分？
2. 为什么要有分代思想？
3. 新生代和老年代怎么理解？
4. 什么是 Minor GC、Full GC？
5. 什么是 Stop-The-World？

这一组最容易卡住的点：

- 说不清“为什么不用引用计数”
- 只会背算法名字，不知道解决什么问题
- 一听到 Full GC 就只会说“加内存”

## 5. 第四组：垃圾回收器追问链

主问题：

> 常见垃圾回收器你知道哪些？

常见追问：

1. CMS 怎么理解？
2. G1 怎么理解？
3. CMS 和 G1 怎么简单区分？
4. 为什么现在更常听到 G1？
5. Full GC 频繁为什么危险？

如果继续追：

1. 低停顿和高吞吐你怎么理解？
2. 如果线上老 Full GC，你第一反应是什么？

这一组的稳法不是背所有回收器，而是：

> 知道 CMS 和 G1 的定位差异，并且能把 Full GC 频繁和线上排查串起来。

## 6. 第五组：类加载追问链

主问题：

> 你讲一下类加载过程。

常见追问：

1. 类加载过程有哪些阶段？
2. 加载、验证、准备、解析、初始化分别做什么？
3. 什么是类加载器？
4. 常见类加载器有哪些？
5. 什么是双亲委派？

继续往下追，常见是：

1. 为什么要有双亲委派？
2. 如果没有双亲委派会怎么样？
3. 双亲委派主要解决什么问题？

回答重点：

> 类加载这条线不只是背五个阶段，更重要的是讲清“类怎么从 `.class` 变成 JVM 可用数据”，以及双亲委派为什么重要。

## 7. 第六组：JMM 与 volatile 追问链

主问题：

> 你讲一下 JMM 和 `volatile`。

常见追问：

1. 什么是 JMM？
2. 为什么要有 JMM？
3. JMM 主要解决什么问题？
4. `volatile` 有什么作用？
5. `volatile` 能不能保证线程安全？

继续往下追，最常见的是：

1. 为什么 `volatile` 不能保证 `i++`？
2. happens-before 是什么？
3. `volatile` 和 `synchronized` 怎么区分？
4. 什么场景下你会用 `volatile`，什么场景下你会用锁？

这一组最重要的不是背术语，而是：

> 把可见性、有序性、原子性三件事分清。

## 8. 第七组：对象创建与引用追问链

主问题：

> 你讲一下 `new` 一个对象发生了什么。

常见追问：

1. 为什么对象创建前要先检查类加载？
2. 对象一般分配在哪？
3. 为什么先分配内存再执行构造方法？
4. 对象在内存里大概怎么布局？
5. 什么是对象头？

继续往下追，经常会接：

1. 强引用、软引用、弱引用、虚引用怎么区分？
2. 软引用适合什么场景？
3. 弱引用和软引用的差别是什么？
4. 虚引用主要是干什么的？

这一组的答法要尽量顺，不要把“对象创建”和“引用类型”拆成完全两张皮。

## 9. 第八组：OOM 与排查追问链

主问题：

> 你讲一下 OOM 和 JVM 排查思路。

常见追问：

1. 什么是 OOM？
2. 什么是内存泄漏？
3. 内存泄漏和 OOM 有什么区别？
4. 常见 OOM 类型有哪些？
5. 堆溢出和栈溢出怎么区分？

继续深问时，通常会问：

1. Full GC 频繁怎么查？
2. OOM 怎么查？
3. CPU 飙高怎么查？
4. 常见 JVM 工具有哪些？
5. JVM 调优为什么不是只改参数？

这一组的关键是分清三条线：

- OOM 更偏内存分配失败
- Full GC 频繁更偏对象存活与回收压力
- CPU 飙高更偏线程执行问题

## 10. 第九组：如果面试官开始问场景题

常见问法：

> 线上服务突然变慢了，你怎么判断是不是 JVM 问题？

可以继续追：

1. 你先看什么指标？
2. 你怎么区分是 GC 问题还是 CPU 问题？
3. 如果是 Full GC 频繁，你下一步做什么？
4. 如果是 CPU 飙高，你会看什么？
5. 如果没有复杂线上经验，你怎么答更稳？

更真实的答法，不是硬吹自己干过特别复杂的大规模调优，而是：

> 我会先看现象和监控，再区分问题类型，然后结合 GC、堆、线程几个维度去排查。

## 11. 一轮比较完整的模拟面试顺序

你可以按下面顺序自己练：

1. 你先整体讲一下 JVM。
2. 你讲一下 JVM 内存结构。
3. JVM 怎么判断对象可回收？
4. CMS 和 G1 怎么区分？
5. 你讲一下类加载和双亲委派。
6. 你讲一下 JMM 和 `volatile`。
7. `volatile` 为什么不能保证 `i++`？
8. 你讲一下 `new` 一个对象发生了什么。
9. 强软弱虚引用怎么区分？
10. OOM、Full GC 频繁、CPU 飙高分别怎么查？

这一轮练下来，基本就能把 JVM 高频面试主线覆盖一遍。

## 12. 你最容易被追崩的地方

- 只会背 JVM 主线，进到某一条就讲不细
- GC 只会说“回收垃圾”，说不清怎么判断垃圾
- `volatile` 只会说“线程安全”
- 类加载只记得双亲委派，不记得五个阶段
- OOM、Full GC、CPU 飙高三种问题混着答
- 一讲排查就只会说“看日志”“调参数”

## 13. 面试前最后怎么练最有效

建议这样练：

1. 先对着 `jvm-interview-high-frequency.md` 过一遍问题。
2. 再对着 `jvm-interview-answers.md` 修表达。
3. 最后用这份追问链做口头模拟，尽量一题接一题往下答。

## 14. 一句话收尾

> JVM 模拟面试最关键的不是单题会不会，而是你能不能从“整体主线”一路被追到“具体细节”，还能保持表达不断线。
