## 2. Gateway 情景题

### 2.1 网关入口与路由

- 如果网关路由配置错了，请求打到错误服务，怎么排查？
- 如果某个服务扩容后，Gateway 还没有转发到新实例，可能是什么原因？
- 如果 Gateway 可以访问服务名，但访问具体接口 404，怎么排查？
- 如果同一个路径前缀需要路由到不同版本服务，怎么设计？
- 如果要做灰度发布，Gateway 可以根据哪些条件分流？

### 2.2 网关鉴权

- 如果用户没有登录就访问接口，Gateway 怎么拦截？
- 如果 token 过期，网关应该返回什么？
- 如果用户伪造 `userId` 请求头，Gateway 怎么防止？
- 网关鉴权通过后，用户信息怎么传给下游服务？
- 哪些权限适合放网关，哪些权限必须放业务服务？

### 2.3 网关限流

- 网关怎么做 IP 限流？
- 如果多个用户共用一个出口 IP，IP 限流误伤怎么办？
- 如果攻击者不断换 IP，IP 限流还有效吗？
- 用户限流和 IP 限流怎么配合？
- 接口维度限流和用户维度限流有什么区别？
- 单机 Gateway 限流和多实例 Gateway 限流有什么区别？
- 如果 Gateway 多实例部署，限流计数怎么共享？
- 限流后应该返回什么状态码和提示？
- 限流规则配置太低误伤用户怎么办？
- 网关限流和 Sentinel 服务内部限流怎么分工？

### 2.4 网关高可用

- Gateway 挂了怎么办？
- Gateway 会不会成为系统瓶颈？
- Gateway 多实例部署后，前面还需要 Nginx 或 SLB 吗？
- Gateway 过滤器逻辑太重会有什么问题？
- 如果 Gateway CPU 飙高，怎么排查？

## 3. OpenFeign / 服务调用情景题

### 3.1 调用失败

- Feign 调用下游服务超时怎么办？
- 下游服务没挂但响应很慢，怎么保护上游？
- 注册中心里有实例，但 Feign 仍然调用失败，怎么排查？
- Feign 调用返回 500，是上游问题还是下游问题，怎么定位？
- 如果调用链中某个服务偶发失败，应该重试吗？

### 3.2 重试与幂等

- Feign 重试导致重复下单怎么办？
- 哪些接口可以重试，哪些接口不能随便重试？
- 读接口重试和写接口重试有什么区别？
- 如果第一次请求下游已经成功，但响应超时，上游重试会有什么问题？
- Feign 重试和 Sentinel 熔断怎么配合？

### 3.3 请求头与上下文

- Feign 调用下游时怎么透传用户信息？
- Feign 调用下游时怎么透传 traceId？
- 如果下游服务拿不到用户上下文，可能是什么原因？
- 请求头全量透传有什么风险？
- 内部服务之间怎么防止伪造用户身份？

## 8. 一句话总结

Spring Cloud 情景题的核心不是背概念，而是沿着“入口、调用、保护、一致性、排查”这条链路思考问题。
