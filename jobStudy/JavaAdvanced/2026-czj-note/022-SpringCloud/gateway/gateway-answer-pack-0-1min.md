# Gateway 0-1 分钟回答包

## 1. Gateway 是什么？

`Spring Cloud Gateway` 是微服务系统里的 API 网关组件，通常作为后端服务的统一入口。客户端请求先进入 Gateway，再由 Gateway 根据路由规则转发到具体微服务。同时它也可以统一做鉴权、限流、日志、跨域、灰度等入口层治理能力。

## 2. 为什么需要 Gateway？

微服务拆分后，后端服务数量变多，如果客户端直接访问各个服务，会导致服务地址暴露、前端调用复杂、鉴权和限流逻辑重复分散。Gateway 提供统一入口，把路由转发和通用治理能力集中处理，降低客户端和后端服务拆分之间的耦合。

## 3. 路由、断言、过滤器是什么？

路由决定请求转发到哪个服务；断言负责判断请求是否匹配某条路由，比如按路径、请求方法、请求头匹配；过滤器负责在请求转发前后做处理，比如鉴权、限流、日志、请求头改写等。简单说，路由决定去哪里，断言决定是否匹配，过滤器决定前后做什么。

## 4. Gateway 和 Nginx 有什么区别？

Nginx 更偏基础设施层，适合做反向代理、静态资源、外部入口负载均衡和 HTTPS 处理；Gateway 更偏微服务网关，适合和注册中心、鉴权、限流、灰度、链路追踪等微服务治理能力结合。两者不是简单替代关系，实际项目里可以是请求先到 Nginx，再进入 Gateway。

## 5. Gateway 怎么做鉴权？

请求进入 Gateway 后，可以先从请求头或 Cookie 中获取 token，校验 token 是否有效。无效请求直接拦截，有效请求解析出用户信息，再把用户 ID、租户 ID 等必要信息通过请求头传给下游服务。但复杂业务权限仍然应该由业务服务自己校验。

## 6. Gateway 会不会成为单点？

如果只部署一个 Gateway，就会有单点风险。实际项目中 Gateway 一般要多实例部署，前面挂 Nginx 或 SLB 做负载均衡，并配合健康检查、监控和限流，避免入口层故障影响整个系统。
