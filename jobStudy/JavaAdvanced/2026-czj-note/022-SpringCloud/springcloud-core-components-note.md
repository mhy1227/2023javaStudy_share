# Spring Cloud 核心组件总览

## 1. 这一份是干什么的

这一份是 `Spring Cloud` 的组件基础篇。

前面两份文档已经把：

- 为什么会引入微服务
- 微服务拆分后会多出什么问题
- `Spring Cloud` 整体在补什么

这些主线讲过了。

这一份继续往下补的是：

1. 这些核心组件分别在干什么
2. 它们为什么会经常一起出现
3. 面试里怎么先把组件作用讲清楚

## 2. 先记住一个总原则

`Spring Cloud` 里的很多组件，不是单点炫技，而是在给微服务补基础设施。

可以先记成一句话：

> 微服务拆分后，服务发现、调用、配置、入口治理、稳定性、事务这些事情都要有人管，Spring Cloud 组件就是干这个的。

所以理解组件时，不要先背名词，而是先问：

- 它在解决什么问题
- 它是站在系统哪一层工作的
- 它和别的组件是什么关系

## 3. `Nacos` 是干什么的

`Nacos` 最常被放在两个位置上讲：

- 注册中心
- 配置中心

### 3.1 作为注册中心

它解决的是服务发现问题。

微服务拆开后，一个服务要调用另一个服务，得先知道对方有哪些可用实例。

`Nacos` 可以帮你管理：

- 服务实例注册
- 服务实例上下线
- 调用方查找可用服务

### 3.2 作为配置中心

它也可以集中管理配置。

这时它解决的是：

- 配置分散
- 环境切换麻烦
- 配置更新不好同步

### 3.3 面试里先怎么说

可以先说：

`Nacos` 在 Spring Cloud 里常被同时用作注册中心和配置中心，前者解决服务发现，后者解决配置集中管理。

## 4. `OpenFeign` 是干什么的

`OpenFeign` 主要解决的是服务之间调用的问题。

在单体里，一个模块调另一个模块，直接方法调用就行。

但微服务拆开后，就变成了网络请求。

`OpenFeign` 的价值在于：

- 让服务调用更像接口调用
- 减少手写 HTTP 模板代码
- 和负载均衡、超时控制等能力更容易组合

### 4.1 先怎么理解

可以把它先理解成“更方便的服务间调用封装”。

它不是替代 HTTP，而是把 HTTP 请求的细节藏起来，让调用方写起来更像普通 Java 接口。

## 5. `LoadBalancer` 是干什么的

当一个服务有多个实例时，请求不能总是打到同一个实例上。

这时就需要负载均衡。

`LoadBalancer` 的作用就是：

- 在多个可用实例之间分配请求
- 避免流量一直堆到某一个实例
- 配合注册中心完成服务调用

### 5.1 先怎么理解

先把它理解成“请求分流器”就行。

服务发现负责告诉你有哪些实例可用，负载均衡负责决定这次请求打到哪一个实例。

## 6. `Spring Cloud Gateway` 是干什么的

`Gateway` 是微服务的统一入口。

它主要解决的是：

- 外部流量怎么统一进入系统
- 路由怎么转发
- 鉴权怎么统一做
- 限流怎么统一做
- 灰度怎么统一做

### 6.1 为什么需要网关

如果没有网关，外部请求可能直接打到各个微服务上。

这样会带来：

- 入口分散
- 鉴权重复
- 路由混乱
- 监控和日志分散

所以网关更像微服务系统的门口。

## 7. `Sentinel` 是干什么的

`Sentinel` 主要解决的是系统稳定性问题。

它关注的关键词通常是：

- 熔断
- 限流
- 降级

### 7.1 先怎么理解

你可以先把它理解成“系统保护器”。

当某个下游很慢、流量很大、或者出现异常时，`Sentinel` 可以帮你把系统行为控制住，不让问题无限扩散。

## 8. `Seata` 是干什么的

`Seata` 主要解决的是分布式事务问题。

当多个微服务一起参与一次业务修改时，单机事务那套就不够用了。

`Seata` 的作用就是：

- 帮你协调跨服务的数据一致性
- 让分布式场景下的事务问题有可落地的处理方式

### 8.1 先怎么理解

先别把它理解成“万能事务框架”。

更准确地说，它是在分布式环境里给你提供一种一致性协调手段。

## 9. 这些组件为什么经常一起出现

因为它们解决的是同一类系统问题链条。

1. 服务拆开以后，先需要注册中心
2. 服务之间要调用，就需要服务调用封装
3. 多实例要分流，就需要负载均衡
4. 配置不能散，就需要配置中心
5. 外部流量不能乱进，就需要网关
6. 下游不稳要保护，就需要熔断限流降级
7. 多服务一起改数据，就需要分布式事务

所以这几个组件通常是一起讲的，不是因为名字相近，而是因为它们对应的是同一套微服务基础设施。

## 10. 面试里先怎么说这一组组件

可以先这样说：

`Spring Cloud` 这一组常见组件，本质上是在给微服务补基础设施。`Nacos` 管注册和配置，`OpenFeign` 管服务调用，`LoadBalancer` 管负载均衡，`Gateway` 管统一入口，`Sentinel` 管熔断限流降级，`Seata` 管分布式事务。

## 11. 当前这一份先记住什么

先记住下面几句：

1. `Nacos` 常同时负责注册和配置
2. `OpenFeign` 让服务调用更像接口调用
3. `LoadBalancer` 负责服务实例之间的分流
4. `Gateway` 负责统一入口治理
5. `Sentinel` 负责稳定性保护
6. `Seata` 负责分布式事务协调

## 12. 一句话总结

`Spring Cloud` 的核心组件，本质上是在微服务拆分后分别解决服务发现、服务调用、负载均衡、配置管理、入口治理、系统保护和分布式事务这些问题。
