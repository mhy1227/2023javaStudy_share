# Spring Cloud 可能会问的问题

## 1. 这一份怎么用

这一份先只整理“可能会问的问题”，暂时不写标准回答。

使用方式很简单：

1. 先看高频主线
2. 再看自己项目里实际用过哪些组件
3. 优先准备和自己项目最相关的问题

## 13. 一句话总结

`Spring Cloud` 题库最核心的主线，不是背组件名，而是把“微服务带来的问题”和“这些组件各自在解决什么问题”对应起来。
