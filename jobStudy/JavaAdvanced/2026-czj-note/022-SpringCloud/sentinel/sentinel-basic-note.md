# Sentinel 基础

## 1. Sentinel 是什么

`Sentinel` 是阿里开源的流量治理和服务保护组件。

在 Spring Cloud 微服务体系里，它常用来解决：

- 限流
- 熔断
- 降级
- 系统保护
- 热点参数保护

可以先这样理解：

> Sentinel 的核心作用，是在流量过高、下游异常或系统资源紧张时，保护服务不被拖垮。

## 2. 为什么需要 Sentinel

微服务拆分后，一个请求可能会经过多个服务。

比如：

```text
订单服务 -> 用户服务 -> 库存服务 -> 支付服务
```

只要其中一个服务变慢或故障，就可能影响上游服务。

如果没有保护机制，可能出现：

- 请求大量堆积
- 线程池被占满
- 数据库连接被耗尽
- 上游服务也被拖慢
- 故障沿调用链扩散

这就是服务稳定性问题。

Sentinel 解决的就是这类问题。

## 3. Sentinel 主要保护什么

Sentinel 主要保护两个方向：

1. 保护当前服务自己
2. 防止下游故障拖垮上游

比如：

- 当前接口 QPS 太高，直接限流
- 下游服务异常率太高，触发熔断
- 某个热点参数访问太集中，单独限制
- 系统负载过高，触发系统保护

## 4. Sentinel 的核心能力

常见能力包括：

- 流量控制
- 熔断降级
- 热点参数限流
- 系统自适应保护
- 实时监控
- 规则动态配置

面试里最常问的是前三个：

- 限流
- 熔断
- 降级

## 5. Sentinel 和网关是什么关系

Gateway 更偏入口层统一治理。

Sentinel 更偏流量控制和服务保护。

两者可以配合：

- Gateway 层做入口限流
- 服务内部用 Sentinel 做接口级限流、熔断、降级
- Feign 调用下游时结合 Sentinel 做失败保护

所以它们不是替代关系。

## 6. Sentinel 和 OpenFeign 是什么关系

OpenFeign 负责远程调用。

Sentinel 可以保护远程调用。

比如：

- 下游服务慢了，触发熔断
- 调用失败率高了，走降级
- 当前接口流量高了，做限流

可以理解为：

> Feign 负责调用，Sentinel 负责在调用不稳定时保护系统。

## 7. Sentinel 是不是只用于微服务

不是。

只要有流量保护需求，单体系统也可以使用 Sentinel。

但它在微服务场景下更常见，因为微服务链路更长，服务之间互相依赖，更容易出现雪崩和级联故障。

## 8. 面试怎么说

可以这样说：

`Sentinel` 是一个流量治理和服务保护组件，常用于 Spring Cloud 微服务体系里做限流、熔断、降级和系统保护。微服务拆分后，一个请求可能会调用多个服务，如果某个下游服务变慢或异常，上游请求就可能堆积，甚至把整个调用链拖垮。Sentinel 的作用就是在流量过高、失败率过高、响应变慢或系统资源紧张时，及时限制请求或触发熔断降级，保护当前服务和整个调用链的稳定性。

## 9. 一句话总结

Sentinel 的核心，是在系统流量异常或依赖不稳定时，帮助服务及时止损，避免故障扩散。
