# Sentinel 0-3 分钟回答包

## 1. 请你介绍一下 Sentinel

`Sentinel` 是一个流量治理和服务保护组件，在 Spring Cloud 微服务体系里常用于限流、熔断、降级、热点参数保护和系统保护。

微服务拆分后，一个请求往往会经过多个服务。如果某个下游服务变慢或异常，上游服务可能会一直等待下游响应，导致线程资源被占用。请求量一大，就会出现请求堆积、线程池耗尽，最终故障沿调用链扩散。Sentinel 的作用就是在流量过高、失败率升高、慢调用增多或系统负载过高时，及时进行限流、熔断或降级，保护当前服务和整体调用链稳定性。

所以 Sentinel 不只是限流工具，它更准确的定位是流量治理和服务保护。

## 2. 什么是服务雪崩？

服务雪崩是指微服务调用链中某个下游服务变慢或故障后，影响依赖它的上游服务，最终故障沿调用链不断扩散，导致多个服务不可用。

比如订单服务调用库存服务，库存服务又调用商品服务。如果商品服务响应很慢，库存服务调用商品服务的线程会长时间等待。库存服务线程被占满后，订单服务调用库存服务也会变慢，最终用户下单接口也可能不可用。

服务雪崩的根本原因是局部故障没有被及时隔离和止损。常见防护手段包括超时、限流、熔断、降级、隔离和监控告警。Sentinel 主要承担限流、熔断、降级和系统保护这部分能力。

## 3. 限流、熔断、降级有什么区别？

限流、熔断和降级都是服务保护手段，但解决的问题不同。

限流关注的是流量太大。比如某个接口 QPS 突然升高，系统承受不住，就需要限制一部分请求进入系统，保护当前服务。

熔断关注的是下游不稳定。比如下游服务异常比例过高，或者慢调用比例过高，继续调用只会让上游线程阻塞，也会继续给下游施加压力，这时可以暂时停止调用下游。

降级关注的是失败后的兜底。比如评论服务失败时，商品详情页可以先不展示评论；推荐服务失败时，可以返回默认推荐。降级的目标是保证核心链路可用。

可以总结为：限流挡住过高流量，熔断停止调用异常依赖，降级返回可接受的替代结果。

## 4. Sentinel 常见规则有哪些？

Sentinel 常见规则包括流控规则、熔断规则、热点参数规则和系统保护规则。

流控规则主要按 QPS 或并发线程数限制请求量，适合秒杀、热点查询、外部开放接口等场景。

熔断规则可以根据慢调用比例、异常比例或异常数触发，适合保护远程调用、第三方接口、数据库依赖等不稳定依赖。

热点参数限流针对的是某个参数值特别热的场景。比如商品详情接口整体流量还可以，但某个爆款商品 ID 被大量访问，就可以只对这个商品 ID 做更细粒度保护。

系统保护规则关注的是整体系统状态，比如 CPU、系统负载、平均响应时间、入口 QPS、并发线程数等。当整体负载过高时，Sentinel 可以触发保护，避免系统被彻底打垮。

## 5. Sentinel 和 Gateway、OpenFeign 怎么配合？

Gateway 更偏统一入口，适合做入口层限流、鉴权、路由等治理。Sentinel 更偏流量控制和服务保护，可以用于网关层，也可以用于服务内部接口。

OpenFeign 负责服务之间的远程调用，而 Sentinel 可以保护这些远程调用。比如一个服务通过 Feign 调用下游服务，如果下游慢调用比例过高或异常比例过高，就可以触发熔断，后续请求快速失败或走降级逻辑，避免调用方线程一直等待。

所以它们的关系不是替代，而是配合：Gateway 管入口，Feign 管调用，Sentinel 管保护。

## 6. 项目里怎么介绍 Sentinel？

可以这样说：

项目里 Sentinel 主要用于接口流量控制和远程调用保护。对于访问量比较高的接口，可以按 QPS 或并发线程数配置限流，避免突发流量直接压垮服务。对于依赖下游服务的接口，可以配置熔断降级规则，比如下游慢调用比例或异常比例超过阈值时，暂时停止继续调用下游，返回一个业务可接受的兜底结果。

同时 Sentinel 规则不能随便拍脑袋配置，要结合压测结果、历史流量和接口重要程度来定。核心链路和非核心链路策略也不一样，比如推荐、评论这类非核心能力可以更早降级，而支付、扣库存这类核心写操作不能随便返回默认成功。

## 7. 一段完整口述版

`Sentinel` 是一个流量治理和服务保护组件，常用于微服务里的限流、熔断、降级和系统保护。微服务拆分后调用链变长，如果某个下游服务变慢或异常，上游服务线程可能被大量阻塞，最后故障沿调用链扩散，形成服务雪崩。

Sentinel 的作用就是在流量异常或依赖不稳定时及时保护系统。限流用于控制进入系统的请求量，防止高流量压垮当前服务；熔断用于下游异常率或慢调用比例过高时，暂时停止继续调用下游；降级用于失败后返回兜底结果，保证核心链路可用。

项目里可以在高流量接口上配置 QPS 或并发限流，也可以在 Feign 远程调用下游服务时配置熔断降级，避免下游异常拖垮上游。规则配置要结合压测和实际流量，不能随便设置阈值，也不能对强一致的核心写操作随便降级成成功。
