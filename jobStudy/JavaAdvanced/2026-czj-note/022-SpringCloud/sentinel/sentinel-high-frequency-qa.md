# Sentinel 高频问题

## 1. Sentinel 是什么？

回答方向：

- 流量治理和服务保护组件
- 常用于限流、熔断、降级、系统保护
- 保护当前服务和调用链稳定性
- 防止流量异常或下游故障拖垮系统

## 2. 为什么需要 Sentinel？

回答方向：

- 微服务调用链变长
- 下游故障可能影响上游
- 高流量可能压垮服务
- 需要限流、熔断、降级保护系统

## 3. 什么是服务雪崩？

回答方向：

- 下游服务故障或变慢
- 上游请求阻塞
- 线程资源被占满
- 故障沿调用链扩散
- 最终多个服务不可用

## 4. 限流、熔断、降级有什么区别？

回答方向：

- 限流：控制进入系统的请求量
- 熔断：下游异常时暂时停止调用
- 降级：失败后返回兜底结果
- 一句话：限流挡流量，熔断断依赖，降级给兜底

## 5. Sentinel 限流一般按什么维度？

回答方向：

- QPS
- 并发线程数
- IP
- 用户
- 接口
- 热点参数

## 6. 熔断一般根据什么触发？

回答方向：

- 慢调用比例
- 异常比例
- 异常数
- 下游超时或失败持续升高

## 7. 什么是热点参数限流？

回答方向：

- 针对某个参数值做限流
- 比如某个商品 ID、用户 ID、文件 ID
- 适合局部热点流量
- 比接口整体限流更细粒度

## 8. Sentinel 和 Gateway 是什么关系？

回答方向：

- Gateway 是统一入口
- Sentinel 是流量治理和服务保护
- Gateway 可以做入口限流
- Sentinel 也可以在服务内部做接口级保护
- 两者可以配合，不是替代关系

## 9. Sentinel 和 OpenFeign 是什么关系？

回答方向：

- Feign 负责远程调用
- Sentinel 可以保护 Feign 调用
- 下游异常时可以熔断降级
- 避免调用方被下游拖垮

## 10. 降级返回结果怎么设计？

回答方向：

- 要业务可接受
- 不影响核心链路
- 非核心数据可以返回默认值
- 强一致写操作不能随便降级成成功

## 11. 如果下游服务慢了怎么办？

回答方向：

- 配置超时
- 慢调用比例过高时熔断
- 返回降级结果
- 避免上游线程一直等待
- 监控告警定位下游问题

## 12. Sentinel 规则怎么配置才合理？

回答方向：

- 结合压测结果
- 结合历史流量
- 区分核心和非核心接口
- 区分读写接口
- 配合监控持续调整

## 13. Sentinel 会不会误伤正常请求？

回答方向：

- 会有可能
- 阈值太低会误伤
- 规则要压测和监控支撑
- 需要灰度配置和动态调整

## 14. Sentinel 只是限流组件吗？

回答方向：

- 不是
- 除了限流，还有熔断降级、热点参数、系统保护、实时监控等能力
- 面试不要只把 Sentinel 说成限流工具

## 15. 项目里怎么介绍 Sentinel？

回答方向：

- 对高流量接口做限流
- 对远程调用做熔断降级
- 对非核心服务失败返回兜底
- 防止下游异常拖垮上游
- 结合监控观察规则效果
