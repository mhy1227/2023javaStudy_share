# Sentinel 0-1 分钟回答包

## 1. Sentinel 是什么？

`Sentinel` 是一个流量治理和服务保护组件，常用于 Spring Cloud 微服务体系里做限流、熔断、降级和系统保护。它的核心作用是在流量过高、下游异常或系统资源紧张时，及时限制请求或触发降级，避免服务被拖垮。

## 2. 为什么需要 Sentinel？

微服务拆分后，一个请求可能会调用多个服务，如果某个下游服务变慢或异常，上游线程可能一直等待，导致请求堆积，最终引发服务雪崩。Sentinel 可以通过限流、熔断和降级及时止损，保护当前服务和整个调用链。

## 3. 什么是服务雪崩？

服务雪崩是指一个下游服务变慢或故障后，上游服务因为等待下游响应而被拖慢，线程资源逐渐被占满，最终故障沿调用链扩散，导致多个服务不可用。它的本质是局部故障没有被及时隔离和止损。

## 4. 限流、熔断、降级有什么区别？

限流是控制进入系统的请求量，防止流量太大压垮当前服务；熔断是下游异常率或慢调用比例过高时，暂时停止继续调用下游，防止故障扩散；降级是调用失败或熔断后返回兜底结果，保证核心链路可用。简单说，限流挡流量，熔断断依赖，降级给兜底。

## 5. Sentinel 和 Gateway 有什么关系？

Gateway 是统一入口，Sentinel 是流量治理和服务保护组件。Gateway 可以在入口层做基础限流，Sentinel 也可以在服务内部对接口、热点参数、远程调用做更细粒度的限流、熔断和降级。两者可以配合，不是替代关系。

## 6. 项目里怎么用 Sentinel？

项目里可以对高流量接口配置 QPS 或并发限流，对调用下游服务的接口配置熔断降级。比如下游服务慢调用比例或异常比例过高时，Sentinel 触发熔断，避免调用方线程一直等待，并返回一个业务可接受的兜底结果。
