# OpenFeign 与 LoadBalancer 0-1 分钟回答包

## 1. OpenFeign 是什么？

`OpenFeign` 是 Spring Cloud 里常用的声明式 HTTP 调用组件。它通过接口和注解描述远程服务调用，底层生成代理对象，把接口方法调用转换成 HTTP 请求。它的价值是减少手写 HTTP 请求的重复代码，让服务间调用更统一、更接近业务接口。

## 2. 为什么要用 OpenFeign？

因为微服务拆分后，服务之间需要大量远程调用。如果每个地方都手写 HTTP 请求，会有很多重复代码，也不方便统一处理超时、异常和日志。OpenFeign 可以把远程调用封装成接口调用，提高可读性，也方便和注册中心、负载均衡、降级组件整合。

## 3. OpenFeign 和 RestTemplate 的区别？

`RestTemplate` 更像手动发 HTTP 请求，需要自己拼 URL、传参数和处理响应；`OpenFeign` 是声明式调用，只需要定义接口和注解，业务代码像调用本地接口一样调用远程服务。微服务内部调用里，OpenFeign 通常更清晰，也更容易和注册中心、负载均衡整合。

## 4. LoadBalancer 是什么？

`LoadBalancer` 是客户端负载均衡组件。它解决的是一个服务有多个实例时，本次请求应该打到哪个实例。它通常会从注册中心拿到服务实例列表，再按轮询等策略选择一个实例进行调用。

## 5. Nacos 和 LoadBalancer 的关系？

`Nacos` 负责服务注册与发现，提供目标服务有哪些实例；`LoadBalancer` 负责从这些实例里选择一个具体实例。简单说，Nacos 告诉你有哪些机器，LoadBalancer 决定这次请求打哪台机器。

## 6. OpenFeign 和 LoadBalancer 的关系？

`OpenFeign` 负责把接口调用转换成远程 HTTP 请求，`LoadBalancer` 负责在多个服务实例之间选择一个目标实例。一次服务调用里，Feign 根据服务名发起调用，LoadBalancer 根据实例列表选出具体 IP 和端口，最后完成请求。

## 7. 客户端负载均衡和服务端负载均衡区别？

客户端负载均衡是在调用方本地选择服务实例，比如 Spring Cloud LoadBalancer；服务端负载均衡是请求先到代理层，再由代理层转发，比如 Nginx、SLB 或网关。内部微服务调用常见客户端负载均衡，外部入口流量常见服务端负载均衡。

## 8. Feign 调用失败怎么办？

Feign 本质是远程调用，所以要配置合理超时，避免线程一直阻塞；重试要谨慎，只适合幂等请求或短暂网络抖动；如果下游不可用，可以结合 Sentinel 做熔断降级，返回兜底结果，避免故障沿调用链扩散。
