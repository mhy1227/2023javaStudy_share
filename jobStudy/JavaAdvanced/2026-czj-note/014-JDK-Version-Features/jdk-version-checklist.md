# JDK 版本题复习清单

## 1. 这份清单怎么用

这一篇不是拿来精读的。

它适合你在面试前 5 到 10 分钟快速过一遍，看自己有没有明显短板。

用法很简单：

- 会讲，打勾
- 讲不顺，回原文档补
- 能讲 30 秒，再尝试讲 1 分钟版本

## 2. JDK 8 清单

- 我能说出 `JDK 8` 最核心的主线是函数式和声明式写法进入 Java
- 我能说出 `Lambda`
- 我能说出函数式接口
- 我能说出 `Stream`
- 我能说出 `Optional`
- 我能说出接口默认方法
- 我能说出新的日期时间 API
- 我能讲 `Lambda` 和匿名内部类的区别
- 我能讲 `Stream` 和普通 `for` 循环怎么选
- 我能讲 `Optional` 为什么不能滥用

## 3. JDK 9 清单

- 我能说出 `JDK 9` 最核心的是模块化系统
- 我能解释为什么模块化重要
- 我能顺手提到 `List.of`、`Set.of`
- 我知道接口私有方法
- 我知道 `JShell`
- 我不会把 `JDK 9` 讲成一堆零碎 API

## 4. JDK 17 清单

- 我能说出 `JDK 17` 是 `LTS`
- 我能说出 `record`
- 我能说出密封类
- 我能说出文本块
- 我能讲清楚 `JDK 17` 为什么越来越常见
- 我能讲清楚 `JDK 17` 不只是语法升级，还是现代工程基线

## 5. 对比清单

- 我能把 `JDK 8 / 9 / 17` 放在一起讲
- 我不会把三个版本讲成杂乱的特性列表
- 我能说出：
- `JDK 8` 看编码风格变化
- `JDK 9` 看模块化
- `JDK 17` 看现代工程基线和 `LTS`

## 6. 项目题清单

- 我能回答“为什么很多项目还在 `JDK 8`”
- 我能回答“为什么很多新项目开始用 `JDK 17`”
- 我能回答“从 `JDK 8` 升级到 `JDK 17` 要关注什么”
- 我能说清楚 JDK 升级和 `Spring Boot 3 / Spring 6` 的关系
- 我不会把版本升级理解成“只换语法”

## 7. 陷阱题清单

- 我不会把 `JDK 8` 只答成 `Lambda`
- 我不会把 `JDK 17` 只答成 `LTS`
- 我不会说不清 `JDK 9` 的主线
- 我不会把“语言特性”和“项目升级”混成一团
- 如果没真正在线上用过 `JDK 17`，我也知道怎么诚实但不显弱地表达

## 8. 最后自测

如果下面 4 题你都能在 30 秒到 1 分钟内讲顺，这一章就基本够用了：

1. `JDK 8` 最重要的新特性有哪些
2. `JDK 9` 最值得记的是什么
3. `JDK 17` 为什么现在越来越常见
4. 为什么有些项目还在 `JDK 8`，有些项目会升级到 `JDK 17`

## 9. 一句话收尾

> 这章复习到最后，目标不是记得更多，而是能讲得更顺、更像真实面试。
