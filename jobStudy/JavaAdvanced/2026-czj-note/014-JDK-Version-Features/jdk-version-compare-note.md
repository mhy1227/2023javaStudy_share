# JDK 8 / 9 / 17 对比笔记

## 1. 这份文档怎么用

这一篇不是单讲某一个版本，而是把：

- JDK 8
- JDK 9
- JDK 17

放在一起看，方便你在面试里回答：

> JDK 8、9、17 各自最值得记的特性是什么？

## 2. 一句话先区分

- JDK 8：函数式编程风格明显引入
- JDK 9：模块化最有代表性
- JDK 17：长期支持版本 + 现代语言特性代表

## 3. JDK 8 最值得记的点

主线是：

- Lambda
- 函数式接口
- Stream
- Optional
- 接口默认方法
- 新时间 API

一句话概括：

> JDK 8 的核心意义是明显改变了 Java 的编码风格。

## 4. JDK 9 最值得记的点

主线是：

- 模块化系统
- 集合工厂方法
- 接口私有方法
- Stream / Optional 增强
- JShell

一句话概括：

> JDK 9 的核心关键词是模块化。

## 5. JDK 17 最值得记的点

主线是：

- LTS 版本意义
- `record`
- 密封类
- 文本块
- 现代语言特性演进

一句话概括：

> JDK 17 的核心意义是既有语言更新，也成了很多项目升级的现实基线。

## 6. 三个版本放一起怎么答最稳

如果面试官问：

> JDK 8、9、17 各自有什么比较有代表性的特性？

你可以这样答：

> JDK 8 我会重点讲 Lambda、Stream、Optional、新时间 API，因为它明显改变了 Java 的编码风格；JDK 9 我会重点讲模块化，这基本是它最有代表性的更新；JDK 17 我会重点讲它作为长期支持版本的重要性，再补 `record`、密封类、文本块这些比较有代表性的现代语言特性。所以这三个版本我一般会分别理解成：8 偏编码风格变化，9 偏模块化，17 偏现代语言能力和工程升级基线。 

## 7. 面试里最容易答错的点

- JDK 9 讲了一堆小点，却忘了模块化
- JDK 17 只会说它是 LTS，不知道任何语言特性
- JDK 8 只会说 Lambda，不知道 Stream / Optional / 新时间 API
- 把不同版本里的预览特性说得太死

## 8. 如果结合项目讲

可以这样答：

> 如果从项目视角看，JDK 8 是现在很多老项目和中型项目的长期基础；JDK 9 更像过渡性理解重点，面试里主要记模块化；JDK 17 则更像新项目和框架升级时越来越常见的现实选择。

## 9. 一句话收尾

> 这三个版本不用背成更新日志，面试里抓住“8 看编码风格、9 看模块化、17 看现代化和 LTS 基线”就够用了。
