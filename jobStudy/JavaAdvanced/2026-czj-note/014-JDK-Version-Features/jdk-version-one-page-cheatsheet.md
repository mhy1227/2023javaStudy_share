# JDK 版本题一页速背

## 1. 这份速背怎么用

这一页只做最后压缩。

适合：

- 面试前快速看一遍
- 记口语主线
- 防止答题时散掉

## 2. 三个版本一句话主线

- `JDK 8`：函数式和声明式写法进入 Java
- `JDK 9`：模块化
- `JDK 17`：现代工程基线 + `LTS`

## 3. JDK 8 必背点

- `Lambda`
- 函数式接口
- `Stream`
- `Optional`
- 接口默认方法
- 新的日期时间 API

一句话答法：

> `JDK 8` 最大的意义不是多了几个类，而是明显改变了 Java 的编码风格。

## 4. JDK 9 必背点

- 模块化系统
- 集合工厂方法：`List.of`、`Set.of`
- 接口私有方法
- `JShell`

一句话答法：

> `JDK 9` 最值得记的是模块化，它让依赖关系和封装边界表达得更清楚。

## 5. JDK 17 必背点

- `LTS`
- `record`
- 密封类
- 文本块

一句话答法：

> `JDK 17` 不只是新特性版本，更像现代 Java 项目的重要基线。

## 6. 对比口径

如果面试官问：

> 你把 `JDK 8 / 9 / 17` 放在一起讲一下

就按这条线答：

> `JDK 8` 代表编码风格变化，`JDK 9` 代表模块化，`JDK 17` 代表现代工程基线和长期支持版本。

## 7. 项目题口径

### 为什么很多项目还在 JDK 8

- 稳定
- 生态成熟
- 历史依赖多
- 升级成本高
- 回归验证成本大

### 为什么新项目更容易选 JDK 17

- `LTS`
- 新框架支持更好
- 技术基线更现代
- 避免刚起步又面临迁移

### 升级时关注什么

- 框架兼容
- 第三方依赖
- 构建与部署环境
- 测试和灰度验证

## 8. 最容易踩的坑

- `JDK 8` 只会说 `Lambda`
- `JDK 17` 只会说 `LTS`
- `JDK 9` 说不出模块化
- 把语言特性和项目升级混在一起

## 9. 最稳的一段总答法

> 我会把 `JDK 8 / 9 / 17` 分成三条主线来记。`JDK 8` 更偏函数式和声明式写法进入 Java，比如 `Lambda`、`Stream`、`Optional`；`JDK 9` 最关键的是模块化；`JDK 17` 则不只是语法更新，它已经是很多现代 Java 项目和新框架升级时的重要基线。所以版本题不只是背新特性，更要能结合项目去讲为什么选这个版本、为什么升级。

## 10. 一句话收尾

> 面试里真正值钱的，不是背得多，而是主线清楚、答得像做过项目的人。
