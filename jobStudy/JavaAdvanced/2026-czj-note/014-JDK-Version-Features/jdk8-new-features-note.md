# JDK 8 新特性笔记

## 1. 这份文档怎么用

JDK 8 是 Java 面试里最常被问的新特性版本之一。

你不用背所有更新，只要把最常见的几条线讲顺：

- Lambda
- 函数式接口
- Stream
- Optional
- 接口默认方法
- 新时间 API

## 2. 先记住最核心的 6 句

1. JDK 8 是 Java 版本演进里非常重要的一次更新。
2. 它把函数式编程风格正式带进了 Java。
3. Lambda 和 Stream 是最常被问的两条线。
4. `Optional` 是为了让空值处理更显式。
5. 新时间 API 是为了替代旧日期 API 不好用的问题。
6. 面试里讲 JDK 8，不是背名词，而是要说清它改变了什么编码方式。

## 3. Lambda 表达式

最稳的回答：

> Lambda 表达式让我们可以把函数式行为以更简洁的方式写出来，尤其适合配合函数式接口和 Stream API 使用。

更口语一点：

> 原来很多匿名内部类能做的事，JDK 8 以后很多场景都可以用 Lambda 写得更短。

## 4. 函数式接口

可以这样答：

> 函数式接口指的是只包含一个抽象方法的接口，JDK 8 里 Lambda 能和这类接口很好地配合。像 `Runnable`、`Comparator` 都可以从这个角度去理解。

一句话记忆：

> Lambda 通常要落在函数式接口上。

## 5. Stream API

这是 JDK 8 最高频题之一。

最稳的回答：

> Stream API 主要是为了让集合数据处理更偏声明式，比如过滤、映射、排序、聚合这类操作可以通过链式方式表达出来。

一句话版：

> Stream 让“对集合做什么”比“怎么写循环”更突出。

## 6. Stream 和传统循环的区别

可以这样答：

> 传统循环更偏命令式写法，强调一步步怎么遍历；Stream 更偏声明式写法，强调我要做过滤、映射还是聚合。它让代码在数据处理意图上更清晰。

## 7. Optional

面试里也经常会顺带问到。

可以这样答：

> `Optional` 是 JDK 8 提供的一个容器类，用来更显式地表达某个值可能为空，目的是减少空指针处理时的混乱感。

一句话记忆：

> `Optional` 不是彻底消灭空指针，而是让空值处理更显式。

## 8. 接口默认方法

可以这样答：

> JDK 8 允许接口里定义默认方法，这样接口在扩展能力时不一定要把所有实现类都改一遍。

这题值钱的地方是：

> 它提升了接口演进时的兼容性。

## 9. 新时间 API

这也是高频点。

可以这样答：

> JDK 8 引入了 `LocalDate`、`LocalTime`、`LocalDateTime` 这些新的日期时间 API，主要是为了替代旧的 `Date`、`Calendar` 在易用性和线程安全等方面的问题。

## 10. 并行流

有时也会被顺带问。

可以这样答：

> JDK 8 的 Stream 还提供了并行流能力，但面试里更稳的表达不是盲目吹并行流性能，而是知道它适合某些数据处理场景，不是所有场景都应该直接开并行。

## 11. 一版适合面试的完整回答

> JDK 8 最值得讲的几条线，一是 Lambda 和函数式接口，让 Java 可以更自然地表达函数式行为；二是 Stream API，让集合处理从偏命令式写法转成更偏声明式写法；三是 `Optional`，让空值处理更显式；四是接口默认方法，让接口演进时更容易兼容；五是新的日期时间 API，用来替代旧的 `Date`、`Calendar` 这类不太友好的用法。所以 JDK 8 的核心意义，不只是加了几个新类，而是明显改变了 Java 的编码风格。 

## 12. 一句话收尾

> JDK 8 真正要记住的，不是功能点列表，而是它把 Java 从传统写法往函数式和声明式风格推进了一大步。
