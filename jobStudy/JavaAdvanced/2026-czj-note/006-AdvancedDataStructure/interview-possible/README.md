# 下一层数据结构面试专题说明

## 1. 为什么要单独开这个目录

前面的：

- `005-DataStructure`
- `006-AdvancedDataStructure`

已经把基础结构和下一层主题的主目录搭起来了。

但从面试准备角度看，还需要把问题再分成两层：

1. 高频手撕
2. 不一定高频手撕，但面试可能问及

因为数据结构这块确实经常是：

- 代码手撕占主导

但也并不只是手撕。

面试官还会继续问：

- 这个结构适合什么场景
- 为什么不是另外一个结构
- 时间复杂度大概怎样
- 和 Java 集合有没有对应关系

所以这里单独开一个“面试可能问及”目录。

## 2. 这一组文档怎么读

建议按下面这个顺序读：

1. `advanced-data-structure-interview-overview.md`
2. `graph-interview-possible.md`
3. `union-find-interview-possible.md`
4. `trie-interview-possible.md`

## 3. 这一组文档重点解决什么

重点不是替代正式笔记，而是解决：

1. 哪些值得先补
2. 哪些是高频手撕
3. 哪些更偏口述追问
4. 该怎么安排复习优先级

## 4. 一句话总结

这一组文档相当于给 `Trie / 图 / 并查集` 单独补一个“面试视角”的入口，把高频手撕和口述追问拆开看。
