# 图面试可能问及

## 1. 图这块为什么值得优先补

因为图题既可能直接出现，也可能“换个壳子”出现。

很多题表面不是在说图，但本质上是在考：

- 节点
- 边
- 遍历
- 连通性

所以图是很值得优先补的一章。

## 2. 图这块更偏什么

图这块在面试里通常既有：

- 手撕

也有：

- 口述追问

但整体上还是：

- 手撕占主导

## 3. 图的高频手撕方向

最值得优先准备的是：

1. DFS
2. BFS
3. 岛屿数量 / 连通块
4. 拓扑排序
5. 最短路径的基础认知

这里面最常先出的通常是：

- DFS / BFS
- 岛屿类
- 拓扑排序

## 4. 图的口述追问方向

比较常见的追问有：

1. 图和树有什么区别
2. 邻接表和邻接矩阵有什么区别
3. 什么时候用 BFS，什么时候用 DFS
4. 为什么课程安排这题能看成图
5. Dijkstra 是解决什么问题的
6. 图里为什么要关心环

## 5. 最值得优先练的题型

如果只刷一轮，建议优先：

1. 岛屿数量
2. 课程表
3. 克隆图
4. 省份数量
5. 单词接龙

这些题覆盖的主线比较完整：

- 遍历
- 连通性
- 拓扑
- 最短路径感知

## 6. 面试里图题最容易暴露的问题

1. 没有真正理解“节点和边”
2. visited 使用不稳
3. BFS 和 DFS 的使用场景说不清
4. 拓扑排序不知道为什么能判环
5. 看到网格题没有意识到它本质上也是图

## 7. 当前阶段最该记住的结论

1. 图题高频而且很泛化
2. BFS / DFS 是图题主入口
3. 岛屿类和课程表类非常值得优先练
4. 图题不只是写代码，还会追问建模思路

## 8. 一句话总结

图这块是下一层数据结构里最该先补的，因为它既高频，又容易在不同题壳子里反复出现。
