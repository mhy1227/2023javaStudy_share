# String 相关的栈、堆、常量池

## 1. 为什么这个知识点重要

学 `String` 时，很多人会卡在这些问题上：

- 为什么 `s1 == s2` 有时候是 `true`
- 为什么 `new String("abc")` 和 `"abc"` 不一样
- 为什么比较字符串内容要用 `equals()`

这些问题背后，核心就是：

- 栈
- 堆
- 字符串常量池

## 2. 先建立一个最基础的认识

在 Java 里可以先这样理解：

- 栈里主要放局部变量和引用
- 堆里主要放对象
- 字符串常量池里放可复用的字符串字面量

这不是完整的 JVM 内存模型课，但对当前 String 复习已经够用了。

## 3. 什么是栈

栈中通常保存：

- 方法里的局部变量
- 引用变量本身

例如：

```java
String s = "abc";
```

这里：

- `s` 这个变量在栈里
- `s` 保存的是一个地址

注意：

- 栈里不是直接放整个字符串内容
- 栈里放的是“指向对象的引用”

## 4. 什么是堆

堆中通常保存：

- `new` 出来的对象
- 普通对象实例

例如：

```java
String s = new String("abc");
```

这里 `new String("abc")` 创建出来的对象在堆里。

所以可以先记：

- 栈里放变量
- 堆里放对象

## 5. 什么是字符串常量池

字符串常量池可以理解为：

- 专门放字符串字面量的一块可复用区域

例如：

```java
String s1 = "abc";
String s2 = "abc";
```

这里的 `"abc"` 不会因为写了两次就一定创建两个独立对象，通常会复用同一个常量池对象。

所以常见现象是：

```java
System.out.println(s1 == s2); // true
```

原因不是因为 `==` 能比较内容，而是：

- `s1` 和 `s2` 指向的是同一个常量池对象

## 6. 一个最经典的例子

```java
String s1 = "abc";
String s2 = "abc";
String s3 = new String("abc");
```

这时可以这样理解：

### `s1`

- `s1` 在栈里
- 指向常量池中的 `"abc"`

### `s2`

- `s2` 在栈里
- 也指向常量池中的 `"abc"`

### `s3`

- `s3` 在栈里
- 指向堆中通过 `new` 创建的对象

所以：

```java
System.out.println(s1 == s2);      // true
System.out.println(s1 == s3);      // false
System.out.println(s1.equals(s3)); // true
```

解释：

- `s1 == s2`：地址相同
- `s1 == s3`：地址不同
- `equals()`：内容相同

## 7. 字面量和 `new` 的区别

### 直接写字面量

```java
String s = "hello";
```

特点：

- 优先使用常量池中的对象
- 相同字面量可以复用

### 使用 `new`

```java
String s = new String("hello");
```

特点：

- 一定会创建新的堆对象

所以复习时先抓住一句话：

- 字面量偏向复用常量池
- `new` 一定创建新的对象

## 8. 为什么不能用 `==` 比较字符串内容

因为 `==` 比较的是：

- 两个引用是否指向同一个对象

而不是：

- 字符串内容是否一样

例如：

```java
String s1 = "java";
String s2 = new String("java");

System.out.println(s1 == s2);      // false
System.out.println(s1.equals(s2)); // true
```

所以平时判断字符串内容，一定优先：

```java
equals()
```

## 9. 为什么 String 不可变和常量池有关

因为字符串常量池要复用对象。

如果字符串内容可以随便改，就会出现问题：

```java
String s1 = "abc";
String s2 = "abc";
```

如果 `s1` 改了内容，那么 `s2` 也会被影响，这显然不合理。

正因为 `String` 不可变，常量池中的字符串对象才能安全复用。

所以这两个点是连在一起的：

- `String` 不可变
- 字符串常量池可以复用对象

## 10. 常见面试题：`new String("abc")` 到底创建了几个对象

这个题容易越抠越细。

当前阶段建议这样回答：

- `"abc"` 这个字面量可能对应常量池中的对象
- `new String("abc")` 一定会在堆里创建一个新的 `String` 对象

所以你至少可以确定：

- 堆里有一个新对象

至于某些版本细节，不是当前基础复习的重点，不要陷进去太久。

## 11. 一个更容易记住的图景

可以把它理解成下面这样：

### 栈

- 放变量名
- 放引用

### 堆

- 放 `new` 出来的对象

### 字符串常量池

- 放被复用的字符串字面量对象

一句话记忆：

- 栈管“引用”
- 堆管“对象”
- 常量池管“复用”

## 12. 当前阶段最该掌握的结论

复习 String 时，你至少要把这几个结论记牢：

1. `String` 是不可变的
2. 栈里放引用，堆里放对象
3. 字符串字面量通常会进入常量池
4. 相同字面量可能复用同一个常量池对象
5. `new String()` 会创建新的堆对象
6. 比较字符串内容要用 `equals()`

## 13. Java 里需要手动释放 String 吗

不需要。

这是 Java 和 C/C++ 很不一样的一个点：

- Java 没有让你手动 `free` 一个字符串对象的操作
- `String` 对象的回收主要交给 GC

例如：

```java
String s = new String("abc");
s = null;
```

这里的 `s = null` 不是“立刻释放内存”，它只是表示：

- 当前这个引用不再指向原来的对象了
- 如果这个对象也没有其他引用，就有机会在后续被 GC 回收

所以复习时要记住：

- Java 里一般不讲“手动释放 String”
- 更准确的说法是“对象是否还可达，以及是否会被 GC 回收”

## 14. `intern()` 是什么

`intern()` 和字符串常量池直接相关。

你可以先这样理解：

- `intern()` 会返回字符串常量池中对应内容的那个引用

例如：

```java
String s1 = new String("abc");
String s2 = s1.intern();
String s3 = "abc";

System.out.println(s1 == s3); // false
System.out.println(s2 == s3); // true
```

这里：

- `s1` 指向堆里的新对象
- `s2` 指向常量池里的 `"abc"`
- `s3` 也指向常量池里的 `"abc"`

所以：

- `s1 == s3` 为 `false`
- `s2 == s3` 为 `true`

当前阶段只需要记住：

1. `intern()` 和常量池有关
2. 它返回的是常量池中的引用
3. 它常用于理解和验证字符串常量池行为

## 15. 一句话总结

String 相关的栈、堆、常量池，核心不是背 JVM 术语，而是搞清楚：

- 变量在哪
- 对象在哪
- 为什么有的字符串能复用
- 为什么 `==` 和 `equals()` 结果不同
