# String 复习检查清单

## 1. 基础理解

复习完后，检查自己能不能直接说清：

- 什么是 `String`
- 为什么 `String` 不可变
- `String` 和普通引用类型有什么使用上的不同感受

如果这些还说不顺，说明基础还没稳。

## 2. 基础方法清单

你应该能熟练使用这些方法：

- `length()`
- `charAt()`
- `isEmpty()`
- `toLowerCase()`
- `toUpperCase()`
- `trim()`
- `substring()`
- `concat()`
- `contains()`
- `startsWith()`
- `endsWith()`
- `indexOf()`
- `lastIndexOf()`

自测标准：

- 看到需求时，能立刻想到对应方法
- 不需要临时查文档

## 3. 比较相关

你应该能说清：

- `==` 比较的是地址
- `equals()` 比较的是内容
- `equalsIgnoreCase()` 是忽略大小写比较内容
- `compareTo()` 常用于排序

如果这些还容易混，说明字符串比较这一块还没过关。

## 4. 截取与查找

你应该能熟练处理：

- 截取前半段
- 截取后半段
- 截取某个标志位之前或之后的内容
- 查找某个子串第一次出现的位置
- 查找最后一次出现的位置

典型场景：

- 从邮箱中截用户名
- 从路径中截文件名

## 5. 替换相关

你应该能明确区分：

- `replace()`
- `replaceAll()`

自测问题：

1. 普通字符替换该用哪个
2. 普通字符串替换该用哪个
3. 按正则规则替换该用哪个

如果你看到正则还在用 `replace()`，说明这一块还没理顺。

## 6. 正则相关方法

你应该掌握这三个高频方法：

- `replaceAll()`
- `matches()`
- `split()`

并且能说清：

- `replaceAll()` 用来替换
- `matches()` 用来做整串校验
- `split()` 用来按规则拆分

## 7. 正则中的基础符号

当前阶段至少要认识：

- `\\d`
- `\\d+`
- `^`
- `$`
- `{7,8}`
- `|`
- `\\.`
- `\\|`

不用要求自己会写复杂正则，但至少看代码时不能完全陌生。

## 8. 常见易错点

检查自己是否已经避开这些坑：

- 用 `==` 比较字符串内容
- 把 `null` 当成空串处理
- 误以为 `trim()` 会去掉中间空格
- 忘了 `substring()` 左闭右开
- 把单引号和双引号混用
- 在 `split(".")`、`split("|")` 里忘记转义
- 把 `matches()` 当成“包含匹配”

## 9. 输出能力检查

只会看懂还不够，你还应该能自己表达：

### 你能不能回答这些问题

1. String 为什么不可变
2. `==` 和 `equals()` 区别是什么
3. String 为什么适合做 `HashMap` 的 key
4. String、StringBuffer、StringBuilder 区别是什么
5. `replace` 和 `replaceAll` 区别是什么
6. `matches()` 和 `contains()` 有什么不同

如果这 6 个问题答不顺，建议回看笔记。

## 10. 实战能力检查

你应该能独立写出以下小功能：

1. 判断字符串是否全为数字
2. 按 `.` 或 `|` 拆分字符串
3. 替换掉某段固定文本
4. 去掉字符串首尾空格
5. 统计某个字符出现次数
6. 截取邮箱用户名

## 11. 当前阶段的达标标准

String 这一章复习达标，不是要求你会所有面试题，而是达到这三个标准：

1. 常用方法会用
2. 高频易错点能避开
3. 常见小题能独立写出来

## 12. 一句话结论

如果你已经能把“基础方法 + 比较区别 + 正则三件套 + 常见小题”说顺并写出来，String 这一块就算复习到位了。
